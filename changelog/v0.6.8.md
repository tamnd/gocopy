# v0.6.8 — codegen module-level: docstring

Date: 2026-04-29
Spec: notes/Spec/1500/1517_gocopy_v068_codegen_docstring.md
CPython parity: Python/codegen.c (3.14.0): module-level
`compiler_mod` for bodies with a leading string literal.

The v0.6.8 release teaches the codegen package the next module-
level shape after the v0.6.7 no-ops surface: a body whose first
statement is a string literal (the docstring), optionally
followed by zero or more no-op statements. The classifier still
owns every other shape.

The 246/246 oracle stays byte-identical. The parallel-output gate
(`TestCodegenParity`) automatically lights up every fixture whose
body matches the new shape; today that adds 8 fixtures to the
codegen path.

## Note on roadmap deviation

Spec 1508 originally allocated v0.6.8 to the `optimize_cfg` pass.
With codegen owning only empty + no-ops shapes after v0.6.7,
`optimize_cfg` would have nothing to optimize — no jumps, no dead
blocks, no redundant branches. The optimizer slides into a later
sub-release once codegen owns shapes whose CFGs are non-trivial.
Spec slots from 1518 onward retain their roadmap allocations.

## Added

- `compiler/codegen/visit_module.go` — `classifyDocstringModule` and
  `buildDocstringModule`. The builder emits
  `RESUME 0; LOAD_CONST 0 (text); STORE_NAME 0 (__doc__); (N-1)x NOP;
  LOAD_CONST 1 (None); RETURN_VALUE`. Both LOAD_CONST/STORE_NAME
  share the docstring's Loc; the trailing LOAD_CONST/RETURN_VALUE
  pair shares the last stmt's Loc (the docstring itself when there
  is no tail). The PEP 626 encoder run-merges these into entries
  byte-identical to `bytecode.DocstringBytecode` and
  `bytecode.DocstringLineTable`.

- `compiler/codegen/source.go` — `findTripleQuoteEnd` (ported from
  the classifier) for multi-line docstring end-position lookup,
  plus `isPlainAsciiNoEscape` and `splitOnNewline` for the same
  ASCII-only constraint the classifier enforces.

## Changed

- `compiler/codegen/codegen.go` — `Build` recognizes the docstring
  shape after the empty/no-ops shapes and ahead of the
  ErrUnsupported fallback.

- `compiler/codegen_parity_test.go` — `compileViaClassifier` picks
  up the modDocstring classifier path so the parity gate validates
  every newly-codegen-owned fixture against the classifier
  reference.

## Tests

- 246/246 fixtures byte-identical (`tests/run.sh`).
- `compiler/codegen/codegen_test.go` covers a single docstring,
  a docstring with a two-stmt no-op tail, a multi-line triple-
  quoted docstring, and the non-ASCII rejection case.
- `compiler/codegen_parity_test.go` automatically validates 8
  docstring fixtures (`019`, `020`, `021`, `024`, `025`, `026`,
  `027`, `028`) plus the prior empty/no-ops fixtures.

## Out of scope

- Module-level assignments (modAssign and friends) — next sub-release.
- Function definitions, closures, classes, comprehensions — later.
- Optimizer passes (`optimize_cfg`, `optimize_load_fast`,
  superinstructions, fold) — later.
