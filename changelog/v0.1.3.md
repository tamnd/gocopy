# v0.1.3 - arithmetic and unary (2026-04-27)

Adds support for module-level assignments where the right-hand side is a
binary arithmetic or bitwise expression between two names, or a unary
operator applied to a name.

## Supported patterns

Binary operators (all 12 non-inplace operators):

```python
x = a + b     # NB_ADD (0)
x = a & b     # NB_AND (1)
x = a // b    # NB_FLOOR_DIVIDE (2)
x = a << b    # NB_LSHIFT (3)
x = a * b     # NB_MULTIPLY (5)
x = a % b     # NB_REMAINDER (6)
x = a | b     # NB_OR (7)
x = a ** b    # NB_POWER (8)
x = a >> b    # NB_RSHIFT (9)
x = a - b     # NB_SUBTRACT (10)
x = a / b     # NB_TRUE_DIVIDE (11)
x = a ^ b     # NB_XOR (12)
```

Unary operators:

```python
x = -a        # UNARY_NEGATIVE
x = ~a        # UNARY_INVERT
x = not a     # TO_BOOL + UNARY_NOT
```

## Opcodes added

| Opcode | Number | Cache words |
|--------|--------|-------------|
| TO_BOOL | 39 | 3 |
| UNARY_INVERT | 40 | 0 |
| UNARY_NEGATIVE | 41 | 0 |
| UNARY_NOT | 42 | 0 |
| CALL_INTRINSIC_1 | 53 | 0 |

## Out of scope for this version

- Constant folding (`x = 1 + 2`)
- Mixed operands (`x = a + 1`)
- Unary positive (`x = +a`) - uses CALL_INTRINSIC_1
- Nested expressions (`x = (a + b) * c`)
- Matrix multiply (`x = a @ b`) - added to oparg table but no fixture yet
