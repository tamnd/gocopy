# v0.6.19 — codegen module-level: subscript + attribute store

Date: 2026-04-29
Spec: notes/Spec/1500/1529_gocopy_v0619_codegen_subscript_attr_store.md
Roadmap: notes/Spec/1500/1523_gocopy_v06x_part2_roadmap.md (band A)
CPython parity: Python/codegen.c (3.14.0): module-level
`compiler_subscript` / `compiler_attribute` for the store
form, driven by a top-level `Assign` whose target is a
`Subscript` or `Attribute`.

The v0.6.19 release is the sixth sub-release in band A of
the v0.6.x part 2 roadmap. It teaches the codegen package
the store siblings of v0.6.18:

- **modSubscriptStore** — `<obj>[<key>] = <val>` where obj,
  key, and val are Names of 1..15 ASCII chars on a single
  source line. First codegen path emitting `STORE_SUBSCR`
  with 1 inline-cache word.
- **modAttrStore** — `<obj>.<attr> = <val>` where obj and
  val are Names and attr is a 1..15 ASCII char identifier
  on a single source line. First codegen path emitting
  `STORE_ATTR` with 4 inline-cache words.

The 246/246 oracle stays byte-identical. The parallel-output
gate (`TestCodegenParity`) automatically lights up fixtures
125 (`a[b] = x`) and 127 (`a.b = x`).

## Added

- `compiler/codegen/visit_subscript_store.go` —
  `subscriptStoreModule` type plus
  `classifySubscriptStoreModule` +
  `buildSubscriptStoreModule`. Single-statement only so the
  parity gate stays byte-equal with the classifier's
  `compileSubscriptStore`.

- `compiler/codegen/visit_attr_store.go` —
  `attrStoreModule` type plus `classifyAttrStoreModule` +
  `buildAttrStoreModule`. Single-statement only so the
  parity gate stays byte-equal with the classifier's
  `compileAttrStore`.

## Changed

- `compiler/codegen/codegen.go` — `Build` recognizes
  modSubscriptStore and modAttrStore after modAttrLoad,
  ahead of the ErrUnsupported fallback.

- `compiler/codegen_parity_test.go` — `compileViaClassifier`
  picks up the modSubscriptStore and modAttrStore
  classifier paths so the parity gate validates the new
  codegen-owned fixtures against the classifier reference.

## Tests

- 246/246 fixtures byte-identical (`tests/run.sh`).
- `compiler/codegen/codegen_test.go` adds:
  `TestBuildSubscriptStore`,
  `TestBuildSubscriptStoreMultiCharNames`,
  `TestBuildSubscriptStoreWithTailFallsThrough`,
  `TestBuildAttrStore`,
  `TestBuildAttrStoreMultiCharNames`,
  `TestBuildAttrStoreWithTailFallsThrough`.
- `compiler/codegen_parity_test.go` automatically validates
  fixtures 125 and 127 via the new arms.

## Out of scope

- Augmented stores (`a[b] += x`, `a.b += x`).
- Slices (`a[b:c] = x`), nested attrs (`a.b.c = x`),
  mixed targets — classifier rejects too.
- Multi-line variants — classifier rejects too.
- modCallAssign — v0.6.20.
- modGenExpr — v0.6.21.
- Function definitions, classes, comprehensions, try/except
  — bands B and C of the part 2 roadmap.
- Optimizer passes, classifier funeral, perf — closing
  band.
