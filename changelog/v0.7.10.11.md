# v0.7.10.11 — raise + assert + global + AnnAssign in funcbody

Date: 2026-04-30
Spec: notes/Spec/1500/1569_gocopy_v07111_raise.md
Roadmap: notes/Spec/1500/1558_gocopy_v0710x_release_roadmap.md

v0.7.10.11 opens `visitFuncBodyDef` to four of the five stmt
shapes spec 1558 §v0.7.10.11 bundled: `raise`, `assert`,
`global`, and `AnnAssign`. `nonlocal` defers to v0.7.10.12
because it needs the cell/free-var closure pipeline (cellvar
mark on outer scope, STORE_DEREF/LOAD_DEREF on inner).

  - **raise** — bare / `raise X` / `raise X from Y`
  - **global** — `global x; x = 1` → STORE_GLOBAL,
    `co_varnames=()`
  - **AnnAssign** — `x: int = 5` emits as Assign;
    `x: int` (no value) emits no instructions per PEP 526
    function-scope rules
  - **assert** — `assert X` and `assert X, "msg"` →
    TO_BOOL + POP_JUMP_IF_TRUE + LOAD_COMMON_CONSTANT 0 +
    (optional msg load + CALL 0) + RAISE_VARARGS 1 +
    implicit LOAD_CONST None + RETURN_VALUE

## CFG terminator fix

`compiler/flowgraph/cfg.go::isTerminator` was missing
`RAISE_VARARGS`. CPython 3.14 `Python/flowgraph.c` classifies it
under `IS_SCOPE_EXIT_OPCODE`, so a basic block that ends with
RAISE_VARARGS must not get a fall-through edge. Without this,
`assert X` produced `co_stacksize=2` instead of CPython's 1
because Block 1 (which ends in RAISE_VARARGS) propagated
depth=1 into the end block via a spurious fall-through, and the
end block's LOAD_CONST None then peaked at 2.

`isScopeExit` in `optimize_load_fast.go` extended in lockstep
to keep the two predicates aligned.

## Symtable: global wins over local

`compiler/symtable/build.go::analyze` now strips `SymLocal` from
any name that also has `SymGlobal` set. This mirrors
CPython's `symtable.c::analyze_block` rule "global wins over
local": for `global x; x = 1`, `visitTarget` had unconditionally
added `SymLocal` for the assignment target, leaving `x` in
`co_varnames`. Stripping `SymLocal` when `SymGlobal` is set
forces the analyze pass to keep `x` in `co_names` (and out of
`co_varnames`) — the shape CPython emits.

`*ast.Assert` was also added to `visitStmt` (was rejected as
unsupported); it visits Test and Msg into the current scope.

## Stack size minimum

Bare `raise` emits only `RESUME 0` and `RAISE_VARARGS 0`,
both zero-effect, so `StackDepth` reports 0. CPython's
`Objects/codeobject.c::_PyCode_Validate` (lines 517-519)
unconditionally bumps every code object's stacksize to ≥ 1
before serialization:

```c
if (con->stacksize == 0) {
    con->stacksize = 1;
}
```

Ported 1:1 in `compiler/assemble/assemble.go` after the
`StackDepth(cfg)` call. Fixture 040 (bare raise) is the
trigger; without this bump it would ship `co_stacksize=0`
while CPython's reference `.pyc` carries `co_stacksize=1`.

## Assert codegen

Mirrors CPython's `codegen_assert` (Python/codegen.c:2932):

```
test block:    visit Test + TO_BOOL + POP_JUMP_IF_TRUE → end
raise block:   NOT_TAKEN + LOAD_COMMON_CONSTANT 0 [+ Msg load + CALL 0] + RAISE_VARARGS 1
end block:     LOAD_CONST None + RETURN_VALUE
```

All raise-block instructions emit at `LOC(test)` per
`flowgraph.c::propagate_line_numbers` (NO_LOCATION inherits
prev_location). `LOAD_COMMON_CONSTANT 0` is the 3.14
replacement for 3.13's `LOAD_ASSERTION_ERROR` — oparg 0 is
`CONSTANT_ASSERTIONERROR`.

## Added opcodes

  - `bytecode.RAISE_VARARGS = 104` — HasArg, ArgRaw,
    OpTerminator, StackVar (oparg-dependent pop).
  - `bytecode.LOAD_COMMON_CONSTANT = 81` — HasArg, ArgRaw,
    StackEff +1.

## Changed

  - `bytecode/opcode.go` — add RAISE_VARARGS, LOAD_COMMON_CONSTANT.
  - `bytecode/opmeta.go` — register both opcodes.
  - `compiler/assemble/assemble.go` — port
    `_PyCode_Validate` minimum-stacksize-1 bump.
  - `compiler/flowgraph/cfg.go` — extend `isTerminator` with
    RAISE_VARARGS so a raise-ending block has no fall-through.
  - `compiler/flowgraph/optimize_load_fast.go` — extend
    `isScopeExit` with RAISE_VARARGS to match
    CPython's `IS_SCOPE_EXIT_OPCODE`.
  - `compiler/symtable/build.go` — accept `*ast.Assert`;
    strip `SymLocal` from globals in `analyze` so
    `global x; x = 1` excludes `x` from `co_varnames`.
  - `compiler/codegen/visit_funcbody_stmt.go` — extend the
    last-stmt classifier with `*ast.Raise` and `*ast.Assert`;
    extend the non-last-stmt classifier with `*ast.AnnAssign`,
    `*ast.Global`, `*ast.Nonlocal`. Add validators
    (`validateFuncBodyRaise`, `validateFuncBodyAssert`,
    `validateFuncBodyAnnAssign`, `validateFuncBodyGlobalNonlocal`)
    and emitters (`emitFuncBodyRaise`, `emitFuncBodyAssert`,
    `emitFuncBodyAnnAssign`). Global / Nonlocal are no-ops at
    emit time. `emitFuncBodyAssign` now routes through
    `emitNameStore` so `global x; x = 1` resolves to
    STORE_GLOBAL.
  - `CHANGELOG.md` — v0.7.10.11 entry added.

## Out of scope (deferred)

  - **nonlocal** — needs cell/free-var closure pipeline
    (cellvar mark on outer + STORE_DEREF/LOAD_DEREF on inner).
    Rolls into v0.7.10.12 alongside while/for/nested-def.
  - **Mid-body raise / assert** — needs
    `eliminate_unreachable_code` flowgraph pass to drop the
    trailing implicit RETURN_NONE. Stays on the legacy
    `visitFunctionDef` path.
  - **raise of richer expression** — `raise Exception("msg")`,
    `raise mod.Err`. Same gate-out — only bare-Name accepted.
  - **assert of richer expression** — `assert x == 1`,
    `assert all(...)`. Only bare-Name Test accepted.
  - **AnnAssign with non-Name annotation or value** —
    `x: List[int] = ...` and friends. Only Name-annotated,
    Name-or-Constant-valued AnnAssigns accepted.
