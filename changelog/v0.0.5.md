## v0.0.5

`gocopy compile` accepts a leading single-line ASCII string literal
as the module's docstring, plus string and bytes literals as no-op
statements anywhere else. Output still matches `python3.14 -m
py_compile` byte-for-byte.

CPython lowers a leading string literal to:

    RESUME 0
    LOAD_CONST <docstring>
    STORE_NAME __doc__
    [trailing-tail body]
    LOAD_CONST None
    RETURN_VALUE

The `consts` tuple becomes `(docstring, None)` and `names` becomes
`('__doc__',)`. The docstring's line table entry covers length 4
when the docstring is the only statement and length 2 otherwise; the
trailing tail of t no-op statements adds `max(0, t-1)` NOPs because
the last tail statement's line entry absorbs the implicit
`LOAD_CONST None / RETURN_VALUE` pair.

Bytes literals (`b"hi"`, `b'x'`) and any non-leading string literal
are no-ops: CPython drops the value and the body collapses to the
v0.0.4 no-op shape.

### Added

- `bytecode.STORE_NAME` opcode (CPython 3.14 opcode 116).
- `bytecode.DocstringBytecode(t)` and `bytecode.DocstringLineTable(
  docLine, docCol, tail)` for the new module shape.
- A string-literal scanner in `compiler` covering single, double,
  and triple-quoted ASCII literals with no backslash escapes and
  no embedded matching quote, plus the same shapes prefixed with
  `b` or `B` for bytes.
- `marshal.emitObject` learns the `string` case (encoded as
  `TYPE_SHORT_ASCII_INTERNED | FLAG_REF`).
- Seven new fixtures: `019_docstring.py` through
  `025_docstring_two_tail.py`.

### Changed

- `compiler.classify` returns three module shapes (`modEmpty`,
  `modNoOps`, `modDocstring`); the docstring shape carries the
  docstring text and the no-op tail.
- `compiler.module` accepts the consts and names tuples as
  parameters so the docstring shape can supply its own
  `(docstring, None)` and `('__doc__',)`.

### Out of scope (deferred)

- Backslash escapes inside string literals.
- Triple-quoted docstrings spanning multiple source lines (would
  require `end_line_delta != 0` in the LONG entry).
- Raw, f-, and t-strings, plus the rest of the prefix combinations.
- Non-ASCII docstring contents (`TYPE_UNICODE` marshal path).
- Strings longer than 255 bytes (`TYPE_ASCII` marshal path).
- Wiring gopapy as the parser; still waiting on a gopapy v1.0.0.
