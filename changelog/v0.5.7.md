# v0.5.7 — mixed module (colorsys.py) (2026-04-28)

## What changed

### New classification: modMixed

Added support for the module shape `[docstring?] [constLitColl?] [foldedBinOp assigns*] [funcBodyExprs+]`,
the pattern found in CPython's `colorsys.py`. The `stmtsToClassification` function now detects this shape
and returns `modMixed`. The detection block is placed before the `stmtString` early-return so that a module
starting with a docstring followed by non-no-op statements doesn't incorrectly fall through.

### compileMixed

New `compiler/mixed.go` with `compileMixed` function. It compiles each function body using
`compileFuncBodyCore`, builds `co_consts` in CPython order
(`[docstr?, func_codes..., None, clcTuple?, foldedVals...]`), builds `co_names` as
`[__doc__?, clc_target?, assignNames..., funcNames...]`, assembles module bytecode, and calls
`MixedModuleLineTable` for the linetable.

### MixedModuleLineTable

New function `bytecode.MixedModuleLineTable` (in `bytecode/funcdef.go`) that emits the PEP 626
module linetable for the mixed-module shape: prologue, optional docstring span, optional CLC span,
assign entries, then function-def entries (3 CU each except the last which gets 5 CU).

### Multi-line constLitColl fix: spansLines check

`extractConstLitColl` previously only delegated to `extractMultiLineSmallList` when the first
element was on a different line than the opening bracket. The `__all__` list in colorsys.py starts
its first element on the same line as `[` but overflows to the next line. Fixed by scanning all
elements and delegating when any element is on a different line.

### extractMultiLineSmallList closeLine scan fix

The `]` scan started at `lastLine` (1-indexed) used as 0-indexed, which skipped the line where
the last element and the closing `]` both appear. Fixed by starting at `lastLine - 1` (0-indexed)
and scanning backwards from the right end of each line to find `]`.

### compileFuncBodyCore endCol varint fix

The endCol+1 varint was decoded with a single-byte read, silently truncating values ≥ 64 (columns
≥ 63). `hls_to_rgb`'s last return statement ends at column 76, so endCol+1=77 requires a two-byte
varint. Fixed by reading the full varint with a loop.

## New fixture

- `242_colorsys.py` — full CPython `colorsys.py`: module docstring, `__all__` list, three BinOp
  constant assignments (`ONE_THIRD`, `ONE_SIXTH`, `TWO_THIRD`), and seven function definitions.
