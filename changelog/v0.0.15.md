# v0.0.15

`gocopy compile` accepts a two-statement augmented assignment at module
level: `name = initVal` followed by `name += augVal`, where both values
are non-negative integers.

CPython's bytecode pattern:

    RESUME 0
    LOAD initVal          (LOAD_SMALL_INT if 0..255, else LOAD_CONST 0)
    STORE_NAME 0
    LOAD_NAME 0
    LOAD augVal           (LOAD_SMALL_INT if 0..255, else LOAD_CONST N)
    BINARY_OP 13          (NB_INPLACE_ADD)
    <10 zero bytes>       (5 inline-cache words)
    STORE_NAME 0
    LOAD_CONST noneIdx
    RETURN_VALUE

The consts tuple: `initVal` is always at index 0 (a phantom slot when
small); `augVal` is at index 1 only when it exceeds 255; `None` is always
last. `co_stacksize` is 2.

`BINARY_OP` carries 5 inline-cache words (10 zero bytes) in the raw
bytecode stream. `CacheSize[44] = 5` records this in `bytecode/opcode.go`.

## Added

- `bytecode.AugAssignBytecode` and `bytecode.AugAssignLineTable` in
  `bytecode/augassign.go` (new file).
- `bytecode.BINARY_OP` (44), `bytecode.LOAD_NAME` (93), and
  `bytecode.NbInplaceAdd` (13) in `bytecode/opcode.go`.
- `CacheSize[44] = 5` for BINARY_OP's 5 inline-cache words.
- `modAugAssign` classifier kind; `stmtAugAssign`; `tryParseAugAssign`
  in `compiler/classify.go`.
- `compileAugAssign` in `compiler/compiler.go`.
- Four new fixtures: `074`-`077` (small/large init x small/large aug).
- `TestAugAssign` in `compiler/compiler_test.go`.

## Deferred

- Augmented assignment with other operators (`-=`, `*=`, etc.).
- `name += floatVal` or `name += strVal` (non-integer RHS).
- `name += name2` (variable RHS).
- Annotated assignment (`x: int = 1`).
- Wiring gopapy as the parser.
