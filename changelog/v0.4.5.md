# v0.4.5 — `frozenset(name).__contains__` assignments

## What's new

Handle module-level assignments of the form `target = frozenset(name).__contains__`
where `name` is any simple identifier. This is the pattern CPython's `keyword.py`
uses to create fast membership-test callables.

### Bytecode shape

```
RESUME
LOAD_NAME 'frozenset'
PUSH_NULL
LOAD_NAME 'arg'
CALL 1             (+ 3 inline-cache words)
LOAD_ATTR 4        ('__contains__', non-method: name_idx=2 × 2 = 4)
                   (+ 9 inline-cache words)
STORE_NAME 3       (target)
LOAD_CONST 0       (None)
RETURN_VALUE
```

co_consts: `(None,)` — no string constants needed.  
co_names: `('frozenset', arg, '__contains__', target)`.  
stacksize: 3.

### Linetable

Five source-position groups, all on the same line:
1. LOAD_NAME + PUSH_NULL (2 CUs) at `[frozensetCol, frozensetCol+9)`
2. LOAD_NAME arg (1 CU) at `[argCol, argCol+len(arg))`
3. CALL + 3 cache (4 CUs) at `[frozensetCol, argCol+len(arg)+1)`
4. LOAD_ATTR + 9 cache (10 CUs, split 8+2) at `[frozensetCol, attrEnd)`
5. STORE_NAME + None + RETURN_VALUE (3 CUs) at `[0, len(target))`
