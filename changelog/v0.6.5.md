# v0.6.5 — assembler with PEP 626 line-table encoder and stack-depth analysis

Date: 2026-04-29
Spec: notes/Spec/1500/1514_gocopy_v065_assemble.md
CPython parity: Python/assemble.c (3.14.0):
`assemble_emit_location`, `write_location_info_entry`,
`stackdepth_walk`.

The v0.6.5 release lands the assembler that turns an `InstrSeq`
into a fully-formed `*bytecode.CodeObject`. It replaces the
v0.6.3/v0.6.4 passthrough of `OrigLineTable` and `OrigExcTable`
with generic encoders, and computes `StackSize` from CFG-walking
abstract interpretation instead of taking it from the source
CodeObject.

No compiler shape routes through Assemble at v0.6.5; the parity
gate is the round-trip
(Decode → Build → Linearize → Assemble → byte-equal) applied to
every fixture's module CodeObject and every nested function
CodeObject. v0.6.6 starts wiring shapes through.

The 246/246 oracle stays byte-identical. No CodeObject field
changes. No CLI changes. Pure additive.

## Added

- `compiler/assemble/assemble.go` — `Assemble(*ir.InstrSeq, Options)
  (*bytecode.CodeObject, error)`. Recovers the CFG via
  `flowgraph.Build`, emits bytecode via `ir.Encode`, line table via
  `EncodeLineTable`, exception table via `EncodeExcTable`, and
  computes `StackSize` via `StackDepth`. `Options.Source` carries
  metadata fields (filename, name, qualname, flags, argcount tuple,
  consts, names, locals) that the IR does not yet hold; v0.6.6+
  populates these from the symtable scope.

- `compiler/assemble/linetable.go` — PEP 626 encoder mirroring
  CPython's `write_location_info_entry`. Emits NONE for
  zero-valued Locs (synthetic MAKE_CELL/COPY_FREE_VARS), SHORT_FORM
  for `line_delta == 0` with a tight column window, ONE_LINE_X for
  `line_delta in [0,3)` with `col,end_col < 128`, NO_COLUMNS when
  the column is missing on a one-line entry, and LONG otherwise.
  Consecutive instructions sharing the same Loc merge into one
  entry whose length covers all of them, capped at 8 code units
  per entry.

- `compiler/assemble/exctable.go` — PEP 657 exception-table encoder
  type and stub. v0.6.5 only handles the empty-table case (no
  fixture exercises exception handling); the `ExcRegion` type is
  in place so v0.6.10's try/except work plugs into a stable
  interface. Non-empty regions panic until v0.6.10.

- `compiler/assemble/stackdepth.go` — CFG-walking abstract
  interpretation mirroring CPython 3.14
  `Python/assemble.c::stackdepth_walk`. Each block gets an incoming
  depth from its predecessors; instructions advance the depth via
  `OpMeta.StackEff` for fixed-effect ops or a per-opcode formula
  for variable-effect ops (`BUILD_{LIST,TUPLE,SET,MAP}`, `CALL n`,
  `LOAD_GLOBAL` with the push-NULL flag, `LOAD_ATTR` with the
  method-style flag). Conditional branches propagate the
  branch-arm depth to jump targets and the fallthrough depth
  forward; FOR_ITER's branch arm is treated as a no-op (the
  iterator stays on the stack until END_FOR pops it).

## Tests

- 246/246 fixtures byte-identical (`tests/run.sh`).
- `compiler/assemble/roundtrip_test.go` extends the v0.6.4 round-
  trip with the assembler: `Decode → Build → Linearize → Assemble`
  must reproduce the bytecode, line table, exception table, and
  stack size for every fixture's module CodeObject and every
  nested function CodeObject.
- `compiler/assemble/linetable_test.go` covers the five entry
  kinds: empty seq, NONE, SHORT_FORM, ONE_LINE_1, LONG (synthetic
  prologue), run merging, and run-split-over-8 boundary case.
- `compiler/assemble/stackdepth_test.go` covers a linear chain,
  CALL with variable arg count, `LOAD_GLOBAL` with the push-NULL
  flag, and BUILD_LIST.
- `compiler/assemble/exctable_test.go` covers the empty-table case
  and the panic gate for non-empty regions.

## Out of scope

- Routing shapes through Assemble. v0.6.6 starts module-level.
- Generators / yield interactions in stackdepth. v0.7.x.
- Exception-region encoding from real try/except IR. v0.6.10.
- Optimizer passes between flowgraph and assembler. v0.6.8 / v0.6.9.
