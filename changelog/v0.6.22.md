# v0.6.22 — codegen module-level: if/elif/else (band B start)

Date: 2026-04-29
Spec: notes/Spec/1500/1532_gocopy_v0622_codegen_if_else.md
Roadmap: notes/Spec/1500/1523_gocopy_v06x_part2_roadmap.md (band B)
CPython parity: Python/codegen.c (3.14.0): module-level
`compiler_if` lowering of an `If` chain whose branch bodies are
single small-int assignments.

The v0.6.22 release opens band B of the v0.6.x part 2 roadmap.
It teaches the codegen package the modIfElse shape:

- A single top-level `if cond: name = val` chain, optionally
  extended with any number of `elif cond: name = val` clauses
  and a terminal `else: name = val`. Every condition is a
  1..15-char ASCII Name on the if/elif source line; every body
  is a single `name = small_int` (0..255) assignment.
  Multi-statement modules and bodies with anything other than a
  single small-int assign fall through.

This is the first codegen path that emits a multi-branch
forward-jump pattern. It generalises modBoolOp / modTernary's
two-block jump patterns to N branches, plus a no-else
implicit-return-None tail that requires a LONG line-table entry
referring back to the first condition's source position. The
jump opargs are constant (`POP_JUMP_IF_FALSE 5` for every
branch), so codegen still emits one straight-line block of IR
without any new label-resolution infrastructure — the
"multi-block CFG" risk from the band-B roadmap doesn't
materialize for this shape. The `EncodeLineTable` LONG fallback
path picks up the negative-lineDelta tail entry automatically
when given `Loc{Line: firstCondLine, Col: firstCondCol, EndCol:
firstCondEnd}`, byte-equal with the classifier's hand-written
LONG entry.

Notable byte-equality details, all mirroring the classifier's
`compileIfElse` byte-for-byte:

- **Per branch** (8-cu condition run + 1-cu LOAD_SMALL_INT +
  3-cu STORE/LOAD_CONST/RETURN run): `LOAD_NAME condIdx` /
  `TO_BOOL 0` (+3 caches) / `POP_JUMP_IF_FALSE 5` (+1 cache) /
  `NOT_TAKEN 0` at L_cond, then `LOAD_SMALL_INT bodyVal` at
  L_val, then `STORE_NAME varIdx` / `LOAD_CONST noneIdx` /
  `RETURN_VALUE 0` at L_var.
- **Terminal else**: `LOAD_SMALL_INT elseVal` at L_elseVal,
  then `STORE_NAME elseVarIdx` / `LOAD_CONST noneIdx` /
  `RETURN_VALUE 0` at L_elseVar.
- **No-else implicit return None**: 2-cu `LOAD_CONST noneIdx`
  + `RETURN_VALUE 0` attributed back to the first condition's
  source position; the encoder picks LONG because lineDelta is
  negative.
- **co_consts**: `[int64(branches[0].BodyVal), nil]` — the
  "first branch value as phantom" mirrors the classifier's
  bytecode-table layout, even when subsequent branches use
  different small ints (those reach `LOAD_SMALL_INT` directly
  via oparg and never enter co_consts).
- **co_names**: insertion order, deduped — for each branch,
  add condName then varName; then else varName when present.
  If the same identifier appears as both a condition and a
  body target, the dedupe collapses it to a single entry.
- **StackSize**: 1 (single-int constant on the stack at any
  given moment).

The 246/246 oracle stays byte-identical. The parallel-output
gate (`TestCodegenParity`) automatically lights up fixtures
131-133:

- 131 `if_simple.py`     (`if a: x = 1`)
- 132 `if_else.py`       (`if a: x = 1` / `else: x = 2`)
- 133 `if_elif_else.py`  (`if a: x = 1` / `elif b: x = 2` /
                          `else: x = 3`)

Band B begins. modWhile (v0.6.23) introduces the first
backward jump and the first label-resolved control flow.
modFor (v0.6.24) closes band B.

## Added

- `compiler/codegen/visit_if_else.go` — `ifElseModuleBranch`
  and `ifElseModule` types plus `classifyIfElseModule` +
  `buildIfElseModule`. Mirrors `compiler/compiler.go::
  compileIfElse` and `compiler/classify.go::ifElseClassify`
  byte-for-byte. Single-statement only so the parity gate
  stays byte-equal with the classifier.

## Changed

- `compiler/codegen/codegen.go` — `Build` recognizes
  modIfElse after modGenExpr, ahead of the ErrUnsupported
  fallback. Doc comment updated to record band B start.

- `compiler/codegen_parity_test.go` — `compileViaClassifier`
  picks up the modIfElse classifier path so the parity gate
  validates the new codegen-owned fixtures against the
  classifier reference.

## Out of scope

- `if not a: ...` and other non-Name conditions — classifier
  rejects.
- Multi-statement bodies, non-int values, nested if-else
  inside the body or elif chain bodies.
- BoolOp / Compare / Call inside the condition — those land
  in modBoolOp / modCmpAssign / modCallAssign at module scope
  and modFuncBodyExpr at function scope.
- Function-body if-else (already covered by the classifier's
  modFuncBodyExpr from band C territory; codegen reaches it
  later in band B/C).
- Optimizer passes, classifier funeral, perf — bands C and D.
