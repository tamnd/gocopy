# v0.3.0 - if-return in function bodies (2026-04-28)

## What changed

Added support for `if <compare>: return <expr>` as a non-final statement in
function bodies compiled by `compileFuncBodyExpr`.

## Pattern

```python
def f(a, b):
    if a < b:
        return a
    return b
```

The compiler emits COMPARE_OP in conditional form (oparg + 16),
POP_JUMP_IF_FALSE with a forward offset, NOT_TAKEN, the then-branch
LOAD + RETURN_VALUE, then falls through to the fallback return.

## Bytecode sequence

```
RESUME 0
LOAD_FAST_BORROW_LOAD_FAST_BORROW (a, b)   # or general load for non-Name operands
COMPARE_OP cmpBase+16                       # conditional form
[cache]
POP_JUMP_IF_FALSE <offset>                  # forward jump to fallthrough
[cache]
NOT_TAKEN
LOAD_FAST_BORROW / LOAD_SMALL_INT / LOAD_CONST   # then-branch value
RETURN_VALUE
<L1: fallthrough return>
```

## Linetable

- LFLBLFLB (if used): ONE_LINE, 1 CU, [condLeft.col, condLeft.end)
- 5-CU condition block: SAME_LINE, 5 CUs, [condLeft.col, condRight.end)
- Then-branch load: ONE_LINE, 1 CU, [thenVal.col, thenVal.end)
- Then-branch RETURN_VALUE: SAME_LINE, 1 CU, [retKw.col, thenVal.end)

## Limitations

Only single Compare conditions are supported. Boolean combinations
(and/or) and non-compare truth tests are not yet handled.
