# v0.4.6 ŌĆö CPython keyword.py compilation (2026-04-28)

Extends `modConstLitSeq` to compile the real CPython stdlib `keyword.py`
byte-identically. Three gaps closed to make it work.

## What's new

### Multi-line small list literals (n < 31)

`softkwlist = ['_', 'case', 'match', 'type']` spans source lines 56ŌĆō61
with each element on its own line. `extractConstLitColl` now delegates to
`extractLargeStringList` when the first element is not on the opening
bracket's line, regardless of element count. The linetable gains a LONG
entry spanning the full line range (BUILD_LIST + LOAD_CONST + LIST_EXTEND
= 3 CUs), matching CPython's output for multi-line small lists.

### TYPE_ASCII marshal for long strings (len > 255)

The 388-byte `keyword.py` docstring exceeds the 255-byte
`TYPE_SHORT_ASCII` limit. The marshaller now emits `TYPE_ASCII` (0x61)
with a 4-byte LE length for strings longer than 255 bytes. FLAG_REF
applies only when the same string appears more than once.

### Docstring end-line from source scan (`findTripleQuoteEnd`)

`classifyAST` previously computed the docstring's closing line from the
number of `\n` in the string VALUE. Backslash-newline escape sequences
(`\<newline>`) remove both the backslash and the newline from the value,
so the value newline count is lower than the source line count. A new
`findTripleQuoteEnd` scanner walks raw source bytes from the opening
triple-quote to the closing one, returning the correct source line and
column regardless of escape sequences.
