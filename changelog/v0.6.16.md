# v0.6.16 — codegen module-level: ternary assignment

Date: 2026-04-29
Spec: notes/Spec/1500/1526_gocopy_v0616_codegen_ternary_assign.md
Roadmap: notes/Spec/1500/1523_gocopy_v06x_part2_roadmap.md (band A)
CPython parity: Python/codegen.c (3.14.0): module-level
`compiler_ifexp`, driven by a top-level `Assign`.

The v0.6.16 release is the third sub-release in band A of the
v0.6.x part 2 roadmap. It teaches the codegen package the
modTernary shape: a single
`<target> = <trueVal> if <cond> else <falseVal>` where
target, cond, trueVal, and falseVal are all Names of 1..15
ASCII chars.

This is the second jump-bearing codegen path and the first
with two return-bearing branches. The bytecode is RESUME,
LOAD_NAME cond, TO_BOOL (with 3 cache words),
POP_JUMP_IF_FALSE 5 (with 1 cache word), NOT_TAKEN,
LOAD_NAME trueVal, STORE_NAME target, LOAD_CONST None,
RETURN_VALUE, LOAD_NAME falseVal, STORE_NAME target,
LOAD_CONST None, RETURN_VALUE. The forward jump oparg is
resolved at codegen time: 5 code units past the cache lands
on the false-branch LOAD_NAME, mirroring the classifier
helper byte-for-byte. co_consts = [None];
co_names = [cond, trueVal, falseVal, target];
co.StackSize = 1.

The 8-unit cond span (LOAD_NAME + TO_BOOL + 3 caches +
POP_JUMP + 1 cache + NOT_TAKEN) is exactly one chunk in the
encoder's flush() loop, so the assembler emits one
ONE_LINE_X entry of length 8 — byte-identical with
`bytecode.TernaryLineTable`.

The 246/246 oracle stays byte-identical. The parallel-output
gate (`TestCodegenParity`) automatically lights up
fixture 116.

## Added

- `compiler/codegen/visit_ternary_assign.go` —
  `ternaryAssignModule` type plus
  `classifyTernaryAssignModule` +
  `buildTernaryAssignModule`. Single-statement only so the
  parity gate stays byte-equal with the classifier's
  `compileTernary`.

## Changed

- `compiler/codegen/codegen.go` — `Build` recognizes the
  modTernary shape after modBoolOp, ahead of the
  ErrUnsupported fallback.

- `compiler/codegen_parity_test.go` — `compileViaClassifier`
  picks up the modTernary classifier path so the parity gate
  validates the new codegen-owned fixture against the
  classifier reference.

## Tests

- 246/246 fixtures byte-identical (`tests/run.sh`).
- `compiler/codegen/codegen_test.go` adds:
  `TestBuildTernaryAssignBasic`,
  `TestBuildTernaryAssignMultiCharNames`,
  `TestBuildTernaryAssignWithTailFallsThrough`.
- `compiler/codegen_parity_test.go` automatically validates
  fixture 116 via the new arm.

## Out of scope

- Ternary with constants in any slot — the v0.6.9 fold path
  collapses constant-only conditions before reaching
  modTernary.
- Nested ternaries — fall through to a richer codegen path
  in band B.
- modCollection — v0.6.17 (next sub-release).
- Function definitions, classes, comprehensions, try/except —
  bands B and C of the part 2 roadmap.
- Optimizer passes, classifier funeral, perf — closing band.
