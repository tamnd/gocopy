# v0.3.9 — string constants in function bodies (2026-04-28)

Adds support for string literal constants inside function bodies, enabling
patterns like `return "ok"`, `return 'error'`, and multi-branch returns
with string values.

## What changed

- **`compiler/classify_ast.go`** — `isFuncBodyExpr` now accepts `Constant`
  nodes with kind `"str"` when `col <= 255`.

- **`compiler/func_body.go`** — `loadConst` emits `LOAD_CONST` for strings
  (strings are always stored in `co_consts`, never via `LOAD_SMALL_STR`).
  `exprEndCol` handles strings via the new `scanStringEnd` helper.

- **`scanStringEnd`** — new method on `funcState` that scans the source line
  starting at the opening `"` or `'`, handling single-quoted, double-quoted,
  and triple-quoted literals with backslash escape sequences. Returns the
  column past the closing quote for accurate linetable end-column emission.

## Fixtures

Four new fixtures (199–202):

| Fixture | Source |
|---------|--------|
| `199_funcbody_return_str_double.py` | `def f(x): return "ok"` |
| `200_funcbody_return_str_single.py` | `def f(x): return 'error'` |
| `201_funcbody_str_return_branch.py` | if-branch returning `"one"` or `"other"` |
| `202_funcbody_str_assign.py` | assign `"val"` to local, then return it |

## Test results

202/202 fixtures byte-identical.
