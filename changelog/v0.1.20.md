# v0.1.20 - not operator in function bodies (2026-04-28)

Adds `not expr` in function body return expressions. Two code paths:
`not <compare>` avoids TO_BOOL by using COMPARE_OP with oparg+16 and a
trailing UNARY_NOT. `not <name>` uses TO_BOOL (3 cache words) followed
by UNARY_NOT.

## Supported patterns (new)

```python
def f(a):
    return not a

def f(a, b):
    return not (a < b)
```

## Architecture

`isFuncBodyExpr` already accepted `*parser2.UnaryOp` for USub and
Invert. This release adds the Not case: the operand must itself be a
valid func body expression (Name, Constant, Compare, BinOp, or another
UnaryOp passing isFuncBodyExpr).

`walkExpr` in func_body.go gains a "Not" branch inside the UnaryOp case.
If the operand is a Compare node, the comparison is folded into a single
COMPARE_OP(oparg+16) + UNARY_NOT sequence with no TO_BOOL, matching what
CPython's specializing adaptive interpreter expects. Otherwise TO_BOOL
is emitted (with its 3 cache words) followed by UNARY_NOT.

The linetable for `not (compare)` spans from the `not` column through the
closing `)`. `scanEndCol` scans the source line forward from the last
AST-computed column to absorb any trailing `)` character, so the span
correctly ends one past the `)`.

## New code

- `compiler/classify_ast.go`: Not case in UnaryOp branch of isFuncBodyExpr
- `compiler/func_body.go`: Not branch in walkExpr; scanEndCol helper
- 2 new fixtures: 160, 161
