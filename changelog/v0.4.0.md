# v0.4.0 — module-level import statements (2026-04-28)

Adds support for modules whose body consists of import and from-import
statements (with optional blank and comment lines between them).

## What changed

- **`bytecode/opcode.go`** — added `IMPORT_NAME` (73) and `IMPORT_FROM` (72).

- **`bytecode/import.go`** — new file: `ImportEntry`, `ImportAlias`,
  `ImportNameRef`, `AliasNameRef` types; `ImportBytecode` and
  `ImportLineTable` builders.

  The linetable builder handles CPython's SHORT linetable entry (kind=0,
  1 payload byte) used for same-line continuation entries when a single
  source line generates more than 8 code units. SHORT is used when
  `startCol ≤ 15` and `endCol - startCol ≤ 15`; otherwise ONE_LINE0 is
  used. Both produce byte-identical output against CPython 3.14.

- **`compiler/classify.go`** — new `rawStmtKind` values `stmtImport` and
  `stmtFromImport`; new `modKind` value `modImports`; new `importAlias` and
  `importEntry` types; `importAsgn importEntry` field on `rawStmt`;
  `imports []importEntry` field on `classification`.

- **`compiler/classify_ast.go`** — handles `*parser2.Import` and
  `*parser2.ImportFrom` nodes, building `stmtImport` / `stmtFromImport`
  entries. Dotted imports with alias (different bytecode) are rejected.

- **`compiler/compiler.go`** — new `compileImports` function; dispatches
  `modImports` classification.

## Supported import forms

- `import X` — simple absolute import
- `import X as Y` — with local alias
- `import X.Y` — dotted module, binds top-level name
- `import X, Y, Z` — multiple modules on one line
- `from X import Y` — single name
- `from X import Y, Z` — multiple names
- `from X import Y as Z` — with alias

## co_consts layout

`co_consts[0]` = integer level (0 for absolute). Simple imports share one
`None` entry; each from-import adds its own `ConstTuple` of name strings.
