# v0.6.2 — gocopy AST surface, lowering boundary, __future__ flags

Date: 2026-04-29
Spec: notes/Spec/1500/1511_gocopy_v062_ast.md
CPython parity: Python/Python-ast.c, Python/future.c (3.14.0).

The compiler stops reaching into the gopapy parser package
directly. Three new packages establish the boundary, and every
existing compiler file flips its import from `gopapy/parser` to
`gocopy/compiler/ast`.

The 246/246 oracle stays byte-identical; the wire format is
unchanged; the CLI surface (`gocopy compile`) is unchanged. The
release is purely a refactor.

## Added

- `compiler/ast/` — gocopy-owned AST surface. Every node type the
  compiler touches is a Go type alias for the corresponding
  parser type. Identity is preserved end-to-end; later releases
  flip individual aliases to concrete gocopy types when codegen
  needs derived-data slots (`Symbol`, `ConstRef`) on nodes.
- `compiler/lower/lower.go` — `Lower(*parser.Module) (*ast.Module,
  error)`. The single translation point between the parser and
  the compiler. Identity at v0.6.2 (alias means same pointer);
  the signature is the contract every later release keeps.
- `compiler/future/future.go` — `Collect(*ast.Module) (Flags,
  error)`. Walks the module's leading statements, finds
  `from __future__ import …` directives, validates placement
  against CPython's rules, and returns the OR of recognized
  feature bits. The `Annotations` flag is the only one codegen
  will eventually consume; the rest are recognized for parity.

## Changed

- `compiler.Compile` orchestration is now Parse → Lower → Collect
  futures → Build symtable → classify → emit. Lower and Collect
  are eager; misplaced future imports surface as
  `ErrUnsupportedSource`.
- `compiler/symtable/build.go` consumes `*ast.Module` instead of
  `*parser.Module`. Same body — alias means types match.
- `compiler/classify.go`, `compiler/classify_ast.go`,
  `compiler/func_body.go`, `compiler/gen_expr.go` swap their
  `parser2 "github.com/tamnd/gopapy/parser"` import for
  `"github.com/tamnd/gocopy/compiler/ast"` and rename `parser2.X`
  to `ast.X`. No semantic change.

## Invariant

- New `TestParserImportBoundary` (compiler package) scans the
  compiler tree and fails CI if any production file outside
  `compiler/compiler.go`, `compiler/lower/lower.go`, and
  `compiler/ast/ast.go` imports `gopapy/parser`.

## Tests

- 246/246 fixtures byte-identical (`tests/run.sh`).
- `compiler/ast` — alias-identity smoke test.
- `compiler/lower` — identity round-trip, nil-input safety.
- `compiler/future` — empty module, single name, multiple names,
  docstring-then-future, misplaced future, `*` import,
  unknown-name, plain `import __future__`.
- `compiler/symtable` continues to pass (parity against
  python3.14's `symtable.symtable`).

## Out of scope

- Concrete (non-alias) AST types. Aliases are sufficient until
  v0.6.7 when codegen needs derived-data slots on nodes.
- `astdiff` printer. Defer to whichever release first needs it
  for triage.
- Promoting `parser.ParseFile` into the lower package. Today
  `compiler.Compile` is the ingestion root and parsing
  orchestration belongs there; revisit at v0.6.13 driver
  consolidation.
