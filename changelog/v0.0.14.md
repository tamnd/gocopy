## v0.0.14

`gocopy compile` accepts a chained assignment at module level, e.g.
`x = y = 1` or `a = b = c = d = 5`. Any literal type already supported
in v0.0.7-v0.0.13 (None, True, False, `...`, string, bytes, integer,
float, complex, negative) is accepted as the assigned value.

The consts tuple and bytecode follow CPython's exact layout:

- Small integers (0..255) use LOAD_SMALL_INT; the int value is still
  placed at consts[0] as a phantom slot.
- None values use consts = (None,) with LOAD_CONST 0.
- Negative literals use consts = (pos, None, neg) with LOAD_CONST 2.
- All other values use consts = (value, None) with LOAD_CONST 0.
- co_stacksize is 2 (COPY pushes a second item onto the stack).

The CPython bytecode pattern for N targets is: LOAD value, then
[COPY 1, STORE_NAME t[i]] for each of the first N-1 targets, then
STORE_NAME t[N-1].
