# v0.2.2 - subscript and attribute load in function bodies (2026-04-28)

Adds `a[b]` and `a.attr` as valid function-body expressions.

## Supported patterns (new)

```python
def f(a, b):
    return a[b]

def f(a):
    return a.real
```

## Architecture

`isFuncBodyExpr` gains two new cases:

- `*parser2.Attribute`: accepted when the receiver is a known
  local/param Name. Used for simple attribute loads like `a.real`.
- `*parser2.Subscript`: accepted when both `n.Value` and `n.Slice`
  are valid func body expressions.

`walkExpr` gains two new cases:

**Subscript** (`a[b]`): compiles to BINARY_OP NbGetItem (oparg=26) plus
5 cache words, identical to a BinOp. The LFLBLFLB optimisation applies
when both operands are Name nodes with slots 0-15. The closing `]` is
found with `scanSubscriptEnd` to produce the correct linetable column
span.

**Attribute load** (`a.attr`): emits LOAD_FAST_BORROW for the receiver,
then LOAD_ATTR(name_index<<1) with no method-call bit (oparg low bit=0).
LOAD_ATTR has 9 cache words = 10 code units total. Since the SHORT/
ONE_LINE linetable formats cap at 8 CUs per entry, the span is split
into 8-CU + 2-CU entries at the same column range, matching CPython.

The `scanCallEnd`/`scanSubscriptEnd` helpers are now implemented on top
of a shared `scanChar(startCol, ch byte)` method that looks for any
given character.

## New code

- `compiler/classify_ast.go`: Attribute and Subscript cases in
  isFuncBodyExpr
- `compiler/func_body.go`: Attribute and Subscript cases in walkExpr;
  scanChar method; scanSubscriptEnd method
- 2 new fixtures: 172-173
