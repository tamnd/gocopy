# v0.6.23 — codegen module-level: while loop

Date: 2026-04-29
Spec: notes/Spec/1500/1533_gocopy_v0623_codegen_while.md
Roadmap: notes/Spec/1500/1523_gocopy_v06x_part2_roadmap.md (band B)
CPython parity: Python/codegen.c (3.14.0): module-level
`compiler_while` lowering of a `While` whose body is a single
small-int assignment.

The v0.6.23 release continues band B of the v0.6.x part 2
roadmap. It teaches the codegen package the modWhile shape:

- A single top-level `while cond: name = val` loop with no
  break/continue/else, where cond is a 1..15-char ASCII Name
  on the while-line and val is a small int (0..255).

This is the first codegen path that emits a backward jump
(`JUMP_BACKWARD 12`). The jump oparg is constant for this
exact shape (the cu distance back to the loop-top
LOAD_NAME), so codegen still emits one straight-line block of
IR without any new label-resolution infrastructure — just
like modIfElse hand-coded `POP_JUMP_IF_FALSE 5`. Real label
resolution arrives in v0.6.24 modFor.

Notable byte-equality details, all mirroring the classifier's
`compileWhile` byte-for-byte:

- **Loop body** (8-cu condition run + 1-cu LOAD_SMALL_INT +
  3-cu STORE_NAME / JUMP_BACKWARD / cache run): `LOAD_NAME
  condIdx` / `TO_BOOL 0` (+3 caches) / `POP_JUMP_IF_FALSE 5`
  (+1 cache) / `NOT_TAKEN 0` at L_cond, then `LOAD_SMALL_INT
  bodyVal` at L_val, then `STORE_NAME varIdx` /
  `JUMP_BACKWARD 12` (+1 cache) at L_var.
- **Implicit return None**: 2-cu `LOAD_CONST noneIdx` +
  `RETURN_VALUE 0` attributed back to the condition's source
  position; the encoder picks LONG because lineDelta is
  negative, byte-equal with the classifier's hand-written
  LONG entry in `bytecode.WhileAssignLineTable`.
- **co_consts**: `[int64(BodyVal), nil]`.
- **co_names**: insertion-order, deduped — condName then
  varName, with same-name dedupe collapsing both to one
  entry when condName == varName (mirrors the classifier's
  one-shot dedupe in `compileWhile`).
- **StackSize**: 1.

The 246/246 oracle stays byte-identical. The parallel-output
gate (`TestCodegenParity`) automatically lights up fixture
134 `while_simple.py`.

modFor (v0.6.24) closes band B with the first label-resolved
control flow (FOR_ITER + JUMP_BACKWARD as a labelled pair).

## Added

- `compiler/codegen/visit_while.go` — `whileModule` type plus
  `classifyWhileModule` + `buildWhileModule`. Mirrors
  `compiler/compiler.go::compileWhile` and
  `compiler/classify_ast.go::extractWhileAssign` byte-for-
  byte. Single-statement only so the parity gate stays
  byte-equal with the classifier.

## Changed

- `compiler/codegen/codegen.go` — `Build` recognizes modWhile
  after modIfElse, ahead of the ErrUnsupported fallback. Doc
  comment updated to record the new backward-jump emitter.

- `compiler/codegen_parity_test.go` — `compileViaClassifier`
  picks up the modWhile classifier path so the parity gate
  validates the new codegen-owned fixture against the
  classifier reference.

## Tests

- 246/246 fixtures byte-identical (`tests/run.sh`).
- `compiler/codegen/codegen_test.go` adds:
  `TestBuildWhileSimple`,
  `TestBuildWhileSameNameDedupes`,
  `TestBuildWhileMultiVarFallsThrough`,
  `TestBuildWhileWithElseFallsThrough`,
  `TestBuildWhileNonNameCondFallsThrough`.
- `compiler/codegen_parity_test.go` automatically validates
  fixture 134 via the new arm.

## Out of scope

- `break` / `continue` / `else:` clauses.
- Multi-statement body, non-int values, nested loops.
- Compound conditions (BoolOp / Compare / Call / UnaryOp).
- Function-body while loops — band C territory.
- Optimizer passes, classifier funeral, perf — bands C and D.
