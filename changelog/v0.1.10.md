# v0.1.10 - while loops (2026-04-27)

Adds support for a simple `while cond: name = val` loop at module level.
The condition must be a name. The body must be a single `name = small_int`
assignment (value 0-255). No `break`, `continue`, or `else` clause.

## Supported pattern

```python
while cond:
    x = 1
```

The loop condition uses the same `TO_BOOL` + `POP_JUMP_IF_FALSE` sequence as
if/elif. `JUMP_BACKWARD` at the end of the body jumps back to the `TO_BOOL`
instruction; CPython uses a negative offset counted from the instruction after
the jump.

## Opcodes added

| Opcode | Number | Cache words |
|--------|--------|-------------|
| JUMP_BACKWARD | 75 | 1 |

## New code

- `bytecode/while.go`: `WhileAssignBytecode`, `WhileAssignLineTable`.
- `compiler/classify.go`: `modWhile`, `stmtWhile`, `whileAssign`.
- `compiler/classify_ast.go`: handles `*parser2.While` with a single-assign body
  and a name condition.
- `compiler/compiler.go`: `compileWhile`.
- 1 new fixture: 134.

## Out of scope for this version

- Multi-statement loop bodies.
- `break` and `continue`.
- `else` clause on while.
- Conditions that are not plain names.
