# v0.6.3 — instruction sequence IR with decode/encode round-trip

Date: 2026-04-29
Spec: notes/Spec/1500/1512_gocopy_v063_ir.md
CPython parity: Python/instruction_sequence.c, Python/locations.md
(3.14.0).

The v0.6.3 release lands the pseudo-bytecode IR that every later
compiler pass consumes. The IR is dormant infrastructure — the
classifier still drives codegen — but it is fully validated
against every existing fixture by a decode/encode round trip.

The 246/246 oracle stays byte-identical. No CodeObject field
changes. No CLI changes. Pure additive.

## Added

- `compiler/ir/instr.go` — `Instr{Op, Arg, Loc}` plus
  `Instr.Bytes()` which emits the on-disk form, including
  `EXTENDED_ARG` chains (up to 4 bytes of oparg) and zero-filled
  inline cache words sized by `bytecode.CacheSize`.
- `compiler/ir/label.go` — `LabelID` type plus the `NoLabel`
  sentinel. Symbolic labels are dormant at v0.6.3; v0.6.4's CFG
  introduces them.
- `compiler/ir/block.go` — `Block{ID, Label, Instrs, Succ}` plus
  `Block.AddOp`. `Succ` stays empty at v0.6.3.
- `compiler/ir/instrseq.go` — `InstrSeq` container with the
  builder API (`AddBlock`, `AllocLabel`, `BindLabel`, `AddJump`,
  `Patch`). Holds passthrough `OrigLineTable` and `OrigExcTable`
  for byte-identical re-emission.
- `compiler/ir/decode.go` — `Decode(*CodeObject) (*InstrSeq, error)`.
  Walks bytecode, collapses `EXTENDED_ARG` chains, skips inline
  cache words, attaches per-instruction `Loc` from the line
  table.
- `compiler/ir/encode.go` — `Encode(*InstrSeq) (bytecode []byte,
  linetable []byte, error)`. Emits each instruction's bytes;
  returns `OrigLineTable` verbatim. The generic linetable
  encoder over `(Instr, Loc)` runs lands at v0.6.5.
- `compiler/ir/linetable.go` — PEP 626 line-table decoder
  mirroring the format documented in `Python/locations.md`.
  Handles SHORT_FORM, ONE_LINE_{0,1,2}, NO_COLUMNS, LONG, and
  NONE entries.

## Tests

- 246/246 fixtures byte-identical (`tests/run.sh`).
- `compiler/ir/roundtrip_test.go` round-trips every fixture's
  module CodeObject *and* every nested function CodeObject in its
  Consts pool. Decode → Encode must produce byte-identical
  bytecode and line table.
- Unit tests on `Instr.Bytes()` for 8-bit, 16-bit, 24-bit, and
  32-bit oparg widths and for cached opcodes.
- Unit tests on the builder API (`AllocLabel` density, `AddBlock`
  numbering, `AddOp` ordering).

## Out of scope

- CFG construction. v0.6.4.
- Stack-depth analysis and exception-table encoding. v0.6.5.
- Generic linetable encoder over `Loc` runs. v0.6.5.
- Codegen consuming the IR. v0.6.6.
