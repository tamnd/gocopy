# v0.7.5 — visitExpr dispatcher + collection / subscript / attribute / call (closes phase A)

Date: 2026-04-29
Spec: notes/Spec/1500/1547_gocopy_v075_visit_collections_subscript_attr_call.md
Architecture: notes/Spec/1500/1541_gocopy_v07_architecture.md
Roadmap: notes/Spec/1500/1542_gocopy_v07x_roadmap.md
CPython parity: Python/codegen.c (3.14.0): `codegen_visit_expr` (the
recursive expression dispatcher), `codegen_visit_expr_list`,
`codegen_visit_expr_tuple`, `codegen_visit_expr_set`,
`codegen_visit_expr_dict`, `codegen_visit_expr_subscript`,
`codegen_visit_expr_attribute`, `codegen_visit_expr_call`,
`codegen_visit_stmt_assign` (Subscript / Attribute target branches);
Python/compile.c (3.14.0): `compiler_add_const` const-pool
dedup-by-equality with implicit-None ordering (carries the empty-tuple
post-None slot through `emitDeferredConstLoad`).

The fifth v0.7.x feature release. v0.7.4 closed the branching
expression cluster and landed the first two flowgraph optimizer
passes; v0.7.5 closes phase A by promoting the seven remaining
classifier shapes the v0.6 cluster owned: `modCollection` (list /
tuple / set / dict literals, including the empty forms),
`modSubscriptLoad`, `modSubscriptStore`, `modAttrLoad`,
`modAttrStore`, `modCallAssign` (positional-only `<t> = f(args)`),
and `modGenExpr` (recursive Name / SmallInt / BinOp / UnaryOp). The
release introduces the central recursive `visitExpr` dispatcher — the
analogue of CPython 3.14 `Python/codegen.c::codegen_visit_expr` — and
is the first time the visitor actually recurses through nested
expressions rather than handling closed-form one- or two-operand
cases. After v0.7.5 every expression-RHS shape the v0.6 classifier
owned has been promoted; only the statement-level shapes remain in
the classifier surface. The 246/246 oracle stays byte-identical and
the visitor's parity count climbs from 114/246 to 135/246.

## Added

- `compiler/codegen/visit_expr.go` — `visitExpr`, the central
  recursive expression dispatcher. Returns the `(startCol, endCol)`
  span the caller can use to anchor a wrapping op's Loc. Cases:
  `*ast.Constant`, `*ast.Name`, `*ast.BinOp`, `*ast.UnaryOp`,
  `*ast.Compare`, `*ast.BoolOp`, `*ast.IfExp`, `*ast.List`,
  `*ast.Tuple`, `*ast.Set`, `*ast.Dict`, `*ast.Subscript`,
  `*ast.Attribute`, `*ast.Call`. The file also hosts
  `visitConstantExpr` (recursive context — int 0..255 with
  `constEndCol`-derived span, distinct from the simple-Constant arm
  in `emitAssignValue` that uses lineEnd), `visitNameExpr` (LOAD_NAME
  with the v0.6 1..15-char + line-match + col≤255 envelope),
  `visitBinOpExpr` (recurses both operands, emits BINARY_OP with the
  span Loc), and `visitUnaryOpExpr` (USub / Invert / Not — Not emits
  TO_BOOL + UNARY_NOT). Mirrors `Python/codegen.c::codegen_visit_expr`.
- `compiler/codegen/visit_subscript.go` — `visitSubscriptExpr`
  (Load) recurses obj then slice and emits `BINARY_OP NbGetItem` (1
  cache word). `emitSubscriptStore` recurses the RHS first, then obj,
  then slice, and emits `STORE_SUBSCR 0` (1 cache word). Both anchor
  the op Loc on `objCol → sliceEnd+1` to span past the closing `]`.
- `compiler/codegen/visit_attribute.go` — `visitAttributeExpr`
  (Load) recurses obj and emits `LOAD_ATTR (idx<<1)` (9 cache words,
  split 8+2 by the line-table encoder). `emitAttributeStore` recurses
  the RHS first, then obj, and emits `STORE_ATTR idx` (4 cache
  words). Both anchor the op Loc on `objCol → objEnd+1+len(attr)`.
  The 1..15-char attribute restriction matches the classifier surface.
- `compiler/codegen/visit_call.go` — `visitCallExpr` recurses the
  callee, emits `PUSH_NULL` (with the callee's own funcLoc — not a
  zero-width Loc — to match the classifier byte layout), recurses
  every positional arg in order, and emits `CALL N` (3 cache words).
  Keyword args / `*args` / `**kwargs` return `ErrNotImplemented`.
  `callCloseEnd` scans the source line forward from the last-arg-end
  column tracking `()`,`[]`,`{}` nesting depth so a nested call inside
  an outer attribute (e.g. `frozenset(x).__contains__`) still anchors
  the CALL Loc on the matching `)` rather than the line end.

## Changed

- `compiler/codegen/visit_collection.go` — replaced the v0.6
  classifier file with the visitor handlers `visitListExpr` /
  `visitTupleExpr` / `visitSetExpr` / `visitDictExpr`. Empty forms:
  `[]` → `BUILD_LIST 0`; `()` → `emitDeferredConstLoad` of
  `bytecode.ConstTuple{}` (lands post-None at co_consts[1]); `{}` →
  `BUILD_MAP 0`; empty `Set` has no literal form and returns
  `ErrNotImplemented`. Non-empty forms recurse every element via
  `visitExpr` and emit `BUILD_LIST N` / `BUILD_TUPLE N` /
  `BUILD_SET N` / `BUILD_MAP N` (where N is the number of pairs;
  `**other` unpacking via a nil Key returns `ErrNotImplemented`).
  `collectionSpan` returns the `(openCol, lineEnd)` span the build op
  anchors on.
- `compiler/codegen/visit_assign_stmt.go::visitAssignStmt` — the
  single-target path gains two new arms before the Name loop:
  `*ast.Subscript` target dispatches to `emitSubscriptStore`,
  `*ast.Attribute` target dispatches to `emitAttributeStore`. Both
  arms are gated to `len(a.Targets) == 1`; multi-target chained
  assignments with non-Name targets return `ErrNotImplemented` and
  fall through to the legacy classifier path. The trailing
  `LOAD_CONST None` / `RETURN_VALUE` anchors on the store op's own
  Loc, lifted byte-for-byte from the v0.6 classifier.
- `compiler/codegen/visit_assign_stmt.go::emitAssignValue` — the
  closed-form `*ast.BinOp` and `*ast.UnaryOp` Name+Name arms are
  replaced with calls into the new `visitBinOpExpr` /
  `visitUnaryOpExpr` recursive paths (both delegate through
  `visitExpr`, so a BinOp whose operand is itself a BinOp now lowers
  end-to-end through the visitor). A new arm covers `*ast.List`,
  `*ast.Tuple`, `*ast.Set`, `*ast.Dict`, `*ast.Subscript`,
  `*ast.Attribute`, `*ast.Call`, and `*ast.Name` by routing straight
  to `visitExpr`. The `*ast.Constant` fold short-circuit and the
  `BinOp(Const,op,Const)` / `UnaryOp(USub,Const)` fold paths are
  preserved verbatim — they live until v0.7.16's `fold_constants`
  pass takes over.
- `compiler/codegen/codegen.go::Build` — drops the seven dispatch
  arms (`classifyCollectionModule`, `classifySubscriptLoadModule`,
  `classifySubscriptStoreModule`, `classifyAttrLoadModule`,
  `classifyAttrStoreModule`, `classifyCallAssignModule`,
  `classifyGenExprModule`) plus the matching package doc-comment
  paragraphs. `Build` now starts at `classifyIfElseModule`; only the
  six statement-level shapes (modIfElse, modWhile, modFor, modFuncDef,
  modClosureDef) remain on the classifier surface.

## Out of scope

- Multi-target chained Subscript / Attribute assignment
  (`a[b] = c = d`). The new target arms are gated to a single
  target; multi-target shapes still ship through the classifier and
  land at v0.7.10.
- Non-Name elements in collection literals beyond what `visitExpr`
  already handles (e.g. nested calls inside list elements). Lifts
  naturally as the recursion surface widens at v0.7.10+.
- Keyword args, `*args`, `**kwargs`, slice expressions in subscript.
  Deferred to v0.7.8 (function definitions) / v0.7.10 (general call
  shapes).
- Statement-level shapes (`If`, `While`, `For`, `FuncDef`, `Closure`).
  Phase B opens at v0.7.6 with `visit_If`.
- The full CPython optimizer chain. v0.7.5 ships no new flowgraph
  passes — the v0.7.4 chain (`inline_small_exit_blocks` +
  `resolve_jumps`) is a no-op on the linear IR every v0.7.5 visitor
  handler emits.
- gopapy parser changes.
