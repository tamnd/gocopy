# v0.6.15 — codegen module-level: boolop assignment

Date: 2026-04-29
Spec: notes/Spec/1500/1525_gocopy_v0615_codegen_boolop_assign.md
Roadmap: notes/Spec/1500/1523_gocopy_v06x_part2_roadmap.md (band A)
CPython parity: Python/codegen.c (3.14.0): module-level
`compiler_visit_expr1` / `compiler_boolop`, driven by a top-level
`Assign`.

The v0.6.15 release is the second sub-release in band A of the
v0.6.x part 2 roadmap. It teaches the codegen package the
modBoolOp shape: a single `<target> = <left> and <right>` or
`<target> = <left> or <right>` where target, left, and right
are all Names of 1..15 ASCII chars.

This is the first codegen path that emits a forward jump.
The bytecode is RESUME, LOAD_NAME left, COPY 1, TO_BOOL (with
3 cache words), POP_JUMP_IF_FALSE 3 (or POP_JUMP_IF_TRUE for
`or`, with 1 cache word), NOT_TAKEN, POP_TOP, LOAD_NAME right,
STORE_NAME target, LOAD_CONST None, RETURN_VALUE. The forward
jump oparg is resolved at codegen time: 3 code units past the
cache lands on STORE_NAME, mirroring the classifier helper
byte-for-byte. co_consts = [None]; co_names = [left, right,
target]; co.StackSize = 2.

The 9-unit same-Loc run (COPY..NOT_TAKEN..POP_TOP) splits at
the encoder's 8-unit chunk boundary into two SHORT entries
(8 + 1), matching `bytecode.BoolAndOrLineTable`'s hand-written
split.

The 246/246 oracle stays byte-identical. The parallel-output
gate (`TestCodegenParity`) automatically lights up fixtures
114 and 115 (2 boolop fixtures spanning both operators).

## Added

- `compiler/codegen/visit_boolop_assign.go` —
  `boolOpAssignModule` type plus `classifyBoolOpAssignModule`
  + `buildBoolOpAssignModule`. Single-statement only so the
  parity gate stays byte-equal with the classifier's
  `compileBoolOp`.

## Changed

- `compiler/codegen/codegen.go` — `Build` recognizes the
  modBoolOp shape after modCmpAssign, ahead of the
  ErrUnsupported fallback.

- `compiler/codegen_parity_test.go` — `compileViaClassifier`
  picks up the modBoolOp classifier path so the parity gate
  validates the new codegen-owned fixtures against the
  classifier reference.

## Tests

- 246/246 fixtures byte-identical (`tests/run.sh`).
- `compiler/codegen/codegen_test.go` adds:
  `TestBuildBoolOpAssignAnd`,
  `TestBuildBoolOpAssignOr`,
  `TestBuildBoolOpAssignMultiCharNames`,
  `TestBuildBoolOpAssignWithTailFallsThrough`.
- `compiler/codegen_parity_test.go` automatically validates
  fixtures 114 and 115 via the new arm.

## Out of scope

- Chained bool ops (`a and b and c`) — modGenExpr (band A
  v0.6.21).
- BoolOp with constants on either side — folded.
- modTernary — v0.6.16 (next sub-release).
- Function definitions, classes, comprehensions, try/except —
  bands B and C of the part 2 roadmap.
- Optimizer passes, classifier funeral, perf — closing band.
