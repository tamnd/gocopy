CPython parity: Python/codegen.c (3.14.0) `codegen_visit_stmt_for`
(the for-loop statement visitor, ported as `visitForStmt`);
Python/flowgraph.c (3.14.0) `resolve_unconditional_jumps` and
the relocation half of Python/assemble.c (3.14.0)
`_PyAssemble_ResolveJumps` (both reused verbatim from v0.7.7).

A patch-level continuation of v0.7.7. The original v0.7.7 row in
the roadmap bundled `visit_While + visit_For`; v0.7.7 shipped
visit_While only. v0.7.7.1 ships visit_For — same row, second
half — keeping subsequent roadmap slots in place (v0.7.8 stays
`compiler_enter_scope + visit_FunctionDef`, v0.7.9 stays
closures, etc.).

The v0.6.24 modFor codegen classifier file and Build dispatch
arm are gone; the legacy `compiler/compiler.go::compileFor`
fallback stays alive for shapes the visitor still rejects (else
clause, multi-stmt body, non-Name iter/target, non-small-int
RHS) until later phase B widens the surface.

visitForStmt is the visitor's first emit of the iterator
protocol opcodes (`FOR_ITER`, `GET_ITER`, `END_FOR`, `POP_ITER`)
and the first whose CFG has both a forward (FOR_ITER → exit)
and a backward (JUMP_BACKWARD → top) labelled edge sharing one
body block.

The 246/246 oracle stays byte-identical and the visitor's parity
count climbs from 139/246 to 140/246 (135_for_simple.py).

## Added

- `compiler/codegen/visit_for_stmt.go` — `visitForStmt`, the
  for-loop statement visitor. Lands the iterator-protocol prologue
  (`LOAD_NAME iter + GET_ITER`) into the entry block (the one
  holding RESUME), allocates `topLabel` (loop top, target of
  `JUMP_BACKWARD`) and `exitLabel` (loop exit, target of
  `FOR_ITER`'s exhausted-iter edge), adds a labelled topBlock
  carrying just `FOR_ITER → exitLabel`, an unlabelled body block
  carrying `STORE_NAME loopVar + emitIfBranchAssign body +
  JUMP_BACKWARD → topLabel`, and a labelled exit block carrying
  `END_FOR + POP_ITER + LOAD_CONST None + RETURN_VALUE` at
  iterLoc. Reuses `emitIfBranchAssign` from v0.7.6 verbatim.
  Sets `u.tailEmitted = true` so visitModule skips its own
  trailing terminator emit.

## Changed

- `compiler/codegen/visit_stmt.go::visitStmt` — adds an
  `*ast.For` arm that dispatches to `visitForStmt`.
  `isLast=false` returns `ErrNotImplemented` (the v0.7.7.1
  surface covers only top-level single-stmt-body for loops with
  no trailing stmts).
- `compiler/codegen/codegen.go::Build` — drops the
  `classifyForModule` / `buildForModule` dispatch arm. `Build`
  now starts at `classifyFuncDefModule`; only two statement-level
  shapes (modFuncDef, modClosureDef) remain on the codegen
  classifier surface. Package doc updated to reflect v0.7.7.1
  promotion.

## Removed

- `compiler/codegen/visit_for.go` — the v0.6.24 modFor classifier
  file (~199 LOC: `forModule`, `classifyForModule`,
  `buildForModule`).
- `compiler/codegen/codegen_test.go` — the `TestBuildForSimple`,
  `TestBuildForSameNamesDedupe`,
  `TestBuildForMultiVarFallsThrough`,
  `TestBuildForWithElseFallsThrough`, and
  `TestBuildForNonNameIterFallsThrough` tests that called
  `Build` directly to assert specific bytecode / fall-through for
  the deleted classifier shape. Oracle parity, visitor parity,
  and `TestCodegenParity` still cover the build-side behaviour
  end to end.

## Out of scope

- `else` clause on `for`. The v0.6 classifier rejected it too;
  later phase B generalises.
- `break` / `continue`. Same — v0.6 rejected; later phase B widens.
- Non-Name iter (e.g., `for x in [1, 2, 3]:`) and non-Name
  target (e.g., `for x, y in pairs:`). v0.6 rejected; later
  generalisation lands once visitExpr's collection / unpacking
  surface widens.
- Multi-stmt body. The v0.6 modFor classifier didn't accept
  these either; later phase B generalises.
- gopapy parser changes.
