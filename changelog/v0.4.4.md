# v0.4.4 ‚Äî multi-statement constant-literal collection sequences

## What's new

Handle module-level bodies consisting of an optional docstring followed by
two or more (or one with docstring) constant-literal collection assignments.
This enables keyword.py-style modules with a docstring, a small list, a large
list, and another small list ‚Äî all in one pass.

New classification `modConstLitSeq` fires when:
- `stmtString` + ‚â•1 `stmtConstLitColl` statements, or
- ‚â•2 `stmtConstLitColl` statements with no docstring.

### Co-consts ordering (matches CPython)

CPython places constants in three phases:
1. **Atomic strings**: docstring (if any), then each large-list (n‚â•31) element in source order.
2. **None** sentinel.
3. **Tuples** in source order: one `ConstTuple` per LIST_EXTEND or tuple statement.

Statements with n‚â•31 contribute their elements to phase 1; LIST_EXTEND (n=3..30)
and tuple statements contribute their tuple to phase 3. This matches the ordering
CPython produces for keyword.py (`co_consts = (docstring, elem0..elemN-1, None, tuple0, tuple1)`).

### Linetable encoding

Each statement produces entries exactly as in the standalone case, but:
- The docstring covers 2 CUs (LOAD_CONST + STORE_NAME `__doc__`) in one entry at `[0, docEndCol)`.
- Intermediate STORE_NAME uses SHORT0 n=1.
- The final statement's STORE_NAME uses SHORT0 n=3 (merged with LOAD_CONST None + RETURN_VALUE).
