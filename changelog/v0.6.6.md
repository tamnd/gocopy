# v0.6.6 — codegen module-level (foundation): empty-module shape

Date: 2026-04-29
Spec: notes/Spec/1500/1515_gocopy_v066_codegen_module.md
CPython parity: Python/codegen.c (3.14.0): module-level
`compiler_mod` for an empty body.

The v0.6.6 release introduces the `compiler/codegen` package and
routes its first shape — the empty module — through the v0.6.5
assembler. Everything else still falls back to the classifier.
The substantive validation is a parallel-output gate: every fixture
is compiled by the classifier path AND, when codegen owns the
shape, by the codegen path; the two CodeObjects must be
byte-identical.

This is the foundation step of the v0.6.x classifier funeral.
v0.6.7 grows the codegen shape set; v0.6.13 deletes the
classifier.

The 246/246 oracle stays byte-identical. No CodeObject field
changes. No CLI changes. Pure additive.

## Added

- `compiler/codegen/codegen.go` — `codegen.Build(*ast.Module,
  *symtable.Scope, codegen.Options) (*bytecode.CodeObject, error)`.
  Returns `ErrUnsupported` for shapes the codegen package does not
  yet own. v0.6.6 owns one shape: `len(mod.Body) == 0`. The
  emitted IR is the synthetic prologue
  `RESUME 0 / LOAD_CONST None / RETURN_VALUE`, all carrying the
  synthetic prologue Loc that the PEP 626 encoder turns into a
  single LONG entry covering three code units.

- `compiler/assemble/assemble.go` — `Options` extended with direct
  metadata fields (Filename, Name, QualName, Flags, ArgCount tuple,
  Consts, Names, LocalsPlus*) used when `Source == nil`. The
  `Source != nil` round-trip path is unchanged.

## Changed

- `compiler/compiler.go` — `Compile` tries `codegen.Build` before
  the classifier. On `ErrUnsupported`, falls back. Other codegen
  errors propagate (a malformed IR is a bug, not "this shape is
  not yet wired").

## Tests

- 246/246 fixtures byte-identical (`tests/run.sh`).
- `compiler/codegen/codegen_test.go` — direct emission test for the
  empty module: bytecode equals `bytecode.NoOpBytecode(1)`,
  linetable equals `bytecode.LineTableEmpty()`, stack size = 1,
  consts = `[None]`. Plus negative cases (non-empty module returns
  `ErrUnsupported`; nil module returns a distinct error).
- `compiler/codegen_parity_test.go` — parallel-output gate. For
  every fixture, runs codegen and the classifier; whenever codegen
  owns the shape (no `ErrUnsupported`), the two CodeObjects must
  be byte-identical. Today only `001_empty.py` exercises codegen;
  later releases light up additional fixtures automatically.
- `compiler/assemble/assemble_test.go` — covers the `Source == nil`
  path: explicit-metadata Options produce a complete CodeObject.
