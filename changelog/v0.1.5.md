# v0.1.5 - boolean ops and ternary (2026-04-27)

Adds support for `x = a and b`, `x = a or b`, and `x = a if c else b` at
module level, where all operands are names.

## Supported patterns

```python
x = a and b      # BoolOp: short-circuit AND
x = a or b       # BoolOp: short-circuit OR
x = a if c else b  # conditional expression (ternary)
```

CPython lowers boolean ops and ternary expressions using conditional jumps.
`and` jumps forward on a false-y left operand; `or` jumps on a truthy one.
The ternary evaluates the condition first and jumps over one of the two value
branches.

## Opcodes added

| Opcode | Number | Cache words |
|--------|--------|-------------|
| NOT_TAKEN | 28 | 0 |
| POP_TOP | 31 | 0 |
| JUMP_FORWARD | 77 | 0 |
| POP_JUMP_IF_FALSE | 100 | 1 |
| POP_JUMP_IF_TRUE | 103 | 1 |

## New code

- `bytecode/bool.go`: `BoolAndBytecode`, `BoolOrBytecode`, `BoolAndOrLineTable`,
  `TernaryBytecode`, `TernaryLineTable`, `appendFirstLineEntry`.
- `compiler/classify.go`: `modBoolOp`, `modTernary`, `stmtBoolOp`, `stmtTernary`,
  `boolAssign`, `ternaryAssign` types.
- `compiler/classify_ast.go`: handles `*parser2.BoolOp` (two name values) and
  `*parser2.IfExp` (name test, name body, name orelse).
- `compiler/compiler.go`: `compileBoolOp`, `compileTernary`.
- 3 new fixtures: 114-116.

## Out of scope for this version

- Boolean ops with more than two operands (`a and b and c`).
- Nested boolean expressions.
- Operands that are not plain names.
