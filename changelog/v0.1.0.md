# v0.1.0 (2026-04-26)

Initial release. `gocopy compile` produces a CPython 3.14 `.pyc`
that is byte-for-byte identical to `python3.14 -m py_compile` for
an empty Python source file.

The fixture set is one file: `tests/fixtures/empty.py` (zero bytes).
The CI oracle job byte-diffs gocopy's output against
`python3.14 -m py_compile`'s output of the same fixture and fails
on any difference.

Pipeline pieces wired up here so future versions are a localised
extension rather than a re-bootstrap:

- gopapy is the AST source. gocopy depends on it as
  `github.com/tamnd/gopapy/v1`.
- `bytecode` package: opcode constants, inline-cache size table,
  CodeObject struct.
- `compiler.Compile`: AST module → CodeObject (empty module only;
  any non-empty body returns `compiler: v0.1.0 only supports
  empty modules`).
- `marshal.Marshal`: two-pass writer that mirrors CPython's
  `Py_REFCNT > 1` heuristic for FLAG_REF.
- `pyc.WriteFile`: 16-byte header + marshal stream. Implements
  timestamp, checked-hash, and unchecked-hash modes. Hash mode
  uses SipHash-1-3 with the magic-number-derived key.
- CLI: `gocopy compile FILE.py [-o OUT.pyc]
  [--mode MODE] [--source-date-epoch N]`.

Voice carried from gopapy / goipy: human, small focused commits,
no AI register, no em-dashes.
