# v0.4.1 — constant folding for module-level BinOp(Constant, op, Constant)

## What's new

Handle module-level assignments where the RHS is a constant-folded binary
operation: `x = 1.0/3.0`, `x = 2**8`, `x = 2*3` (single), and sequences
like `ONE_THIRD = 1.0/3.0\nONE_SIXTH = 1.0/6.0` (multi-assign).

CPython folds `BinOp(Constant, op, Constant)` at compile time, emitting
`LOAD_SMALL_INT` when the result fits 0–255 or `LOAD_CONST` otherwise.
The left operand is kept as a phantom at `co_consts[0]` for the first
assignment; subsequent assignments' left operands are not stored.

Supported operators: Add, Sub, Mult, Div (→ float64), FloorDiv, Mod, Pow.
Operand types: int64, float64, or mixed.

## New fixtures (209–213)

- 209 `x = 1.0/3.0` — float division, result stored in co_consts
- 210 `x = 2**8` — integer power, result 256 stored
- 211 `x = 2*3` — integer multiply, result 6 as LOAD_SMALL_INT
- 212 `ONE_THIRD = 1.0/3.0\nONE_SIXTH = 1.0/6.0` — multi float div
- 213 `a = 2**8\nb = 3**4` — multi int pow, 256 stored + 81 as LOAD_SMALL_INT

## Files changed

- `compiler/classify.go` — added `foldedBinOp` type
- `compiler/classify_ast.go` — extended `extractValue` for BinOp(Constant, op, Constant); added `numConstVal` and `foldBinOp` helpers
- `compiler/compiler.go` — `modAssign` handles `foldedBinOp`; `compileMultiAssign` defers folded results after None in co_consts
