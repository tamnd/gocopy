# v0.6.4 — flow graph (CFG) with read-only construction and dominators

Date: 2026-04-29
Spec: notes/Spec/1500/1513_gocopy_v064_flowgraph.md
CPython parity: Python/flowgraph.c (cfg-construction half:
_PyCfgBuilder, cfg_builder_use_label, cfg_builder_check) (3.14.0).

The v0.6.4 release lands a control-flow graph over the v0.6.3
instruction-sequence IR. CFG construction is **read-only and
shape-preserving**: blocks are split at jump-target boundaries and
after every terminator, successors are recorded, but no instruction
is rewritten and `Linearize` reproduces the original byte stream.

Mutation (block reordering, jump rewriting, peephole) belongs to the
optimizer and lands at v0.6.8. The dominator tree ships now so the
optimizer release does not have to debug both at once.

The 246/246 oracle stays byte-identical. The extended round-trip
(Decode → Build → Linearize → Encode) is the substantive parity
gate. No CodeObject field changes. No CLI changes. Pure additive.

## Added

- `compiler/flowgraph/cfg.go` — CFG construction over an `InstrSeq`.
  - `EdgeKind` (`EdgeFallthrough`, `EdgeJump`) and `Edge{Kind, Dest}`.
  - `CFG{Seq, Blocks, Entry, Exit, Edges, startCodeUnit}`.
  - `Build(seq) (*CFG, error)`: 4-pass construction.
    - Pass 1 assigns each instruction a code-unit offset (matching
      `Decode`'s view of the byte stream).
    - Pass 2 collects jump-target offsets, including the offset
      immediately after every jump so successor analysis is uniform.
    - Pass 3 emits blocks in boundary order with monotonically
      increasing IDs, recording each block's starting code-unit.
    - Pass 4 walks each block's last instruction and records jump,
      fallthrough, or no edges accordingly.
  - `Linearize(g) *InstrSeq`: identity in shape — concatenates
    `Blocks` in original layout order, preserves names/consts/sym/
    line-table/exception-table passthrough.
  - `instructionUnits(instr)`: computes per-instruction on-disk size
    in code units, including `EXTENDED_ARG` chain words and trailing
    inline cache words.
  - Opcode predicates `isJump`, `isConditionalJump`, `isTerminator`
    cover `JUMP_FORWARD`, `JUMP_BACKWARD`, `POP_JUMP_IF_{FALSE,TRUE,
    NONE,NOT_NONE}`, `FOR_ITER`, and `RETURN_VALUE`.
  - `JUMP_BACKWARD` subtracts oparg from the next-instruction offset;
    every other jump adds oparg.

- `compiler/flowgraph/dom.go` — Cooper-Harvey-Kennedy iterative
  dominator algorithm.
  - `DomTree{Idom, PostOrder}` plus `Dominators(g) *DomTree`.
  - DFS post-order from `Entry`, reverse-post-order traversal for
    fixed-point iteration, idom-chain `intersect` walk on rpoIndex.
  - Skips predecessors that are unreachable from `Entry` so unreached
    blocks (e.g., dead code after `JUMP_FORWARD`) do not derail the
    intersect walk.

## Tests

- 246/246 fixtures byte-identical (`tests/run.sh`).
- `compiler/flowgraph/roundtrip_test.go` extends the v0.6.3
  round-trip with a CFG pass: `Decode → Build → Linearize → Encode`
  must reproduce both the byte stream and the line table for every
  fixture, including nested function CodeObjects.
- `compiler/flowgraph/cfg_test.go` covers `Build` shapes:
  empty sequence, single-RETURN, JUMP_FORWARD over NOP,
  POP_JUMP_IF_FALSE with both jump and fallthrough edges, and
  `Linearize` idempotence.
- `compiler/flowgraph/dom_test.go` covers `Dominators`:
  linear chain (Idom of join block is the entry) and diamond
  (Idom of join block is the diamond head, not either branch).

## Out of scope

- Jump rewriting and block reordering. v0.6.8 optimizer.
- Stack-depth analysis and exception-table encoding. v0.6.5.
- Generic linetable encoder over `Loc` runs. v0.6.5.
- Codegen consuming the CFG. v0.6.6.
