# v0.2.1 - method calls in function bodies (2026-04-28)

Adds `return obj.method(args...)` inside function bodies where the
receiver is a known local/param and every argument is a valid func-body
expression.

## Supported patterns (new)

```python
def f(a):
    return a.upper()

def f(a, b):
    return a.find(b)

def f(a, b):
    return a.replace(b, b)
```

## Architecture

`isFuncBodyExpr` gains an Attribute branch inside the Call case. A
method call qualifies when the receiver (`fn.Value`) is a known
local/param Name, there are no keyword arguments, and every positional
argument passes `isFuncBodyExpr`.

`walkExpr` in func_body.go now checks whether `n.Func` is a
`*parser2.Attribute`. When it is, the method-call path runs:

1. LOAD_FAST_BORROW(slot) for the receiver object.
2. LOAD_ATTR oparg = (name_index << 1) | 1 (the low bit requests the
   NULL|self pair that CPython 3.14 merges into LOAD_ATTR for method
   calls); 9 cache words (18 bytes = 10 code units).
3. The 10-CU LOAD_ATTR span must be split into two linetable entries
   because the SHORTn/ONE_LINE formats can encode at most 8 code units
   each. The split is 8 CUs + 2 CUs at the same column range, matching
   CPython's output exactly.
4. Arguments and CALL follow the same path as global function calls
   (LFLBLFLB where applicable, ambient-depth tracking for nested cases).

The receiver-object loading uses `emit` (first-on-line) rather than
`emitSame`, so the delta from the def line is encoded correctly.
LOAD_ATTR and the args all use `emitSame`.
