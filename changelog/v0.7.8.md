# v0.7.8 — compiler_enter_scope + visit_FunctionDef (band C opener)

Date: 2026-04-29
Spec: notes/Spec/1500/1551_gocopy_v078_visit_funcdef.md
Architecture: notes/Spec/1500/1541_gocopy_v07_architecture.md
Roadmap: notes/Spec/1500/1542_gocopy_v07x_roadmap.md
CPython parity: Python/codegen.c (3.14.0)
`codegen_visit_stmt_function_def` (the function-def statement
visitor, ported as `visitFunctionDef`); Python/compile.c
(3.14.0) `compiler_enter_scope` and `compiler_exit_scope` (the
scope push/pop pair, ported as
`compileUnit.pushChildUnit` / `popChildUnit`); Python/assemble.c
(3.14.0) `_PyAssemble_MakeCodeObject` (exercised on a non-module
unit for the first time).

Phase B's loop / branching shapes closed at v0.7.7.1. v0.7.8
opens band C by promoting modFuncDef — a top-level
`def f(arg): return arg` definition where `f` and `arg` are
1..15-char identifiers, the body is a single
`return <argName>` statement, and there are no decorators,
annotations, type-params, defaults, vararg, or kwarg — to the
visitor pipeline as `visitFunctionDef`. The v0.6.25 codegen
classifier file (`visit_funcdef.go`) and Build dispatch arm
are gone; the legacy `compiler/compiler.go::compileFuncDef`
fallback stays alive for shapes outside the v0.7.8 surface
(decorators, multiple args, multi-stmt body, defaults,
vararg/kwarg, return-of-non-Name) until v0.7.9 (closures) and
v0.7.10 (general body) widen the surface.

`visitFunctionDef` is the visitor's first **nested compile
unit**: it pushes a child `compileUnit` linked to the parent,
emits the body's IR into the child, then pops back and routes
the child through `optimize.Run + assemble.Assemble`. The
resulting inner `*bytecode.CodeObject` is appended to the
parent's const pool at index 0; `MAKE_FUNCTION` then wraps it
into a function object at module scope. It is also the first
visitor that emits `LOAD_FAST_BORROW` and `MAKE_FUNCTION`.

The 246/246 oracle stays byte-identical and the visitor's
parity count climbs from 140/246 to 141/246
(136_funcdef_simple.py).

## Added

- `compiler/codegen/scope.go` — two new methods on
  `*compileUnit`: `pushChildUnit(name, qualName, firstLineNo)`
  allocates a fresh child unit linked to the parent and
  inherits `Source` / `Filename`; `popChildUnit(child, opts)`
  finalizes the child by running `optimize.Run` on its Seq and
  `assemble.Assemble` with the supplied options (overwriting
  `opts.Consts` and `opts.Names` with the child's unit-level
  tables). Mirrors CPython 3.14
  `Python/compile.c::compiler_enter_scope` and
  `compiler_exit_scope` for the no-closure case.
- `compiler/codegen/visit_functiondef_stmt.go` —
  `visitFunctionDef`, the function-def statement visitor.
  Pushes a child unit at `defLine`, emits `RESUME 0` at
  `Loc{defLine, defLine, 0, 0}` (so EncodeLineTable produces
  the same SHORT_FORM `[0x80, 0x00]` prologue the classifier's
  `FuncReturnArgLineTable` plants via `appendSameLine(1, 0, 0)`),
  emits `LOAD_FAST_BORROW 0` at the body's Name expression
  (ONE_LINE_1), emits `RETURN_VALUE 0` at the Return statement
  (SHORT_FORM), adds `nil` to the child's Consts pool so
  `co_consts[0] = None`, then pops the child through
  `popChildUnit` with `ArgCount=1`, `Flags=CO_OPTIMIZED|
  CO_NEWLOCALS` (`0x3`), `LocalsPlusKinds=[LocalsKindArg]`.
  In the parent unit, adds the resulting `funcCode` as
  `co_consts[0]`, adds `funcName` as `co_names[0]`, and emits
  `LOAD_CONST funcCode + MAKE_FUNCTION + STORE_NAME funcName`
  into the entry block. Does NOT set `u.tailEmitted` —
  visitModule's trailing `LOAD_CONST None + RETURN_VALUE`
  shares the same outerLoc so the 5-CU run merges into one
  LONG line-table entry byte-identical to
  `bytecode.FuncDefModuleLineTable`.

## Changed

- `compiler/codegen/generate.go::compileUnit` — adds a
  `Filename string` field. `Generate` populates it from
  `opts.Filename`; `pushChildUnit` propagates it to the
  child so assembled non-module CodeObjects inherit the
  parent's filename.
- `compiler/codegen/visit_stmt.go::visitStmt` — adds an
  `*ast.FunctionDef` arm that dispatches to `visitFunctionDef`.
  `isLast=false` returns `ErrNotImplemented` (the v0.7.8
  surface covers only top-level single-stmt funcdefs).
- `compiler/codegen/codegen.go::Build` — drops the
  `classifyFuncDefModule` / `buildFuncDefModule` dispatch arm.
  `Build` now starts at `classifyClosureDefModule`; only one
  statement-level shape (modClosureDef) remains on the codegen
  classifier surface. Package doc updated to reflect band C
  opening and the visitor pipeline's first nested-CodeObject
  emit.

## Removed

- `compiler/codegen/visit_funcdef.go` — the v0.6.25 modFuncDef
  classifier file (151 LOC: `funcDefModule`,
  `classifyFuncDefModule`, `buildFuncDefModule`).
- `compiler/codegen/codegen_test.go` — the
  `TestBuildFuncDefSimple`, `TestBuildFuncDefDifferentNames`,
  `TestBuildFuncDefMismatchedReturnFallsThrough`,
  `TestBuildFuncDefMultipleArgsFallsThrough`, and
  `TestBuildFuncDefDecoratorFallsThrough` tests that called
  `Build` directly to assert specific bytecode / fall-through
  for the deleted classifier shape. Oracle parity, visitor
  parity, and `TestCodegenParity` still cover the build-side
  behaviour end to end.

## Out of scope

- Closures (modClosureDef). v0.7.9.
- General function bodies — multi-stmt body with assigns +
  return + nested expressions (modFuncBodyExpr). v0.7.10.
- Decorators, return annotations, type-params.
- Multiple positional/kw-only arguments, defaults, vararg/kwarg.
- Multi-statement bodies, return of expression other than the
  single arg-Name.
- Symtable consumption inside the child unit. v0.7.8 leaves
  `child.Scope = nil` because the no-closure single-arg shape
  doesn't need free / cell flags. v0.7.9 wires the symtable
  child scope through `pushChildUnit` when closures arrive.
- gopapy parser changes.
