# v0.6.1 — symbol table (2026-04-29)

This release lands the symbol-table pass that mirrors CPython's
`Python/symtable.c`. The classifier still drives compilation; the
symtable runs after parse and before classify, walks the AST, and
returns a `*Scope` tree. v0.6.2 starts threading it through codegen;
v0.6.6 makes it the single source of truth and v0.6.7 deletes the
classifier.

The 246/246 oracle stays green. No marshal change. No CodeObject
field change. The release is purely additive — the symtable runs as
a shadow on every compile.

## What changed

### New package `compiler/symtable/`

- **`compiler/symtable/scope.go`** — `Scope`, `Symbol`, `ScopeKind`,
  `SymbolKind`. The kind enum mirrors CPython's `_Py_block_ty`
  (Module / Function / Class / Comprehension); class and
  comprehension panic on construction with a clear "not yet
  supported" message — the v0.6.11 / v0.6.12 slots will land them.
  `SymbolKind` is the bitset matching `Include/internal/pycore_symtable.h`'s
  `DEF_*` and `USE` macros: `SymParam`, `SymLocal`, `SymCell`,
  `SymFree`, `SymGlobal`, `SymGlobalImplicit`, `SymBuiltin`,
  `SymAssigned`, `SymUsed`, `SymAnnotated`, `SymStarImport`. The
  scope exposes `Resolve(name)` for free-variable lookup and
  `LocalsPlusNames()` / `LocalsPlusKinds()` for the slot tables
  CodeObject needs — params first, then plain locals in
  declaration order, then cells, then frees.

- **`compiler/symtable/build.go`** — `Build(*parser.Module) (*Scope, error)`
  with two passes mirroring `_PySymtable_Build`:

  1. **Definition pass.** Walk every node. Each `FunctionDef`
     pushes a child scope; assignment targets, params, `def` /
     `class` names, `for`-loop targets, with/except/import `as`
     clauses, and `global` / `nonlocal` declarations all record
     definitions on the current scope. Every `Name` load records
     a use.
  2. **Analysis pass.** For each function scope, classify every
     symbol: `global` declared → `SymGlobal`; `nonlocal` declared
     or pulled up by a child → `SymFree` and the enclosing
     binding flips to `SymCell`; assigned in this scope →
     `SymLocal`; used but not bound here and bound in some
     enclosing function → `SymFree`; otherwise →
     `SymGlobalImplicit`. Cell-promotion walks the parent chain
     in source order so `co_freevars` ordering matches CPython.

  Unsupported scope shapes (class def, lambda, comprehensions,
  async, match, with, try) return `UnsupportedScopeError` naming
  the offending node so a future fixture that hits one fails
  loudly rather than silently miscompiling.

- **`compiler/symtable/qualname.go`** — `computeQualname` walks
  parent → child in pre-order: top-level functions get
  `qualname == name`; nested functions get
  `parent.qualname + ".<locals>." + name`. Mirrors
  `compiler_set_qualname` in `Python/codegen.c`.

### `compiler.Compile` integration

`Compile` now calls `symtable.Build` immediately after parse and
before `classifyAST`. The result is held but not yet consumed —
v0.6.2 starts threading `*Scope` into the AST nodes; v0.6.6 starts
consuming it in codegen. If `Build` errors on a source that the
classifier accepts, `Compile` surfaces the symtable error so any
new fixture introducing an unsupported shape fails immediately
instead of silently miscompiling.

### Test layers

- **`compiler/symtable/scope_test.go`** — Symbol/Scope construction,
  flag combination, `LocalsPlusNames` / `LocalsPlusKinds` for
  hand-built scopes covering module, simple function, function
  with multiple params + body locals, single-level closure (cell
  + free).
- **`compiler/symtable/build_test.go`** — `Build` on hand-written
  source snippets: empty module, docstring + assign, funcdef with
  one arg, funcdef with multiple params + locals, closure with
  cell promotion, star import, plain import, global declaration.
  Class def and lambda assert `UnsupportedScopeError`.
- **`compiler/symtable/qualname_test.go`** — qualname for module,
  top-level function, deeply nested function chain.
- **`compiler/symtable/fixture_parity_test.go`** — `Build` on
  every fixture under `tests/fixtures/`. Asserts no error across
  the 246-fixture set; any divergence catches scope shapes the
  symtable does not yet handle.
- **`compiler/symtable/cpython_parity_test.go`** — for each
  fixture, shells out to `python3.14 -c "import symtable; ..."`
  and compares the gocopy scope tree against
  `symtable.symtable(src, '<test>', 'exec')`, after stripping the
  PEP 649 `__annotate__` scopes that CPython 3.14 inserts on every
  block. All 246 fixtures match. The test calls `t.Skip` when
  `python3.14` is not on PATH so the basic CI test job stays
  green; the oracle CI job has Python and runs them.

## What did not change

- No marshal wire-format change.
- No CodeObject field changes.
- No classifier change. `compileFuncBodyExpr`, `compileFuncDefInner`,
  `compileClosureDef`, and `compileMixed` still build their slot
  maps the same way.
- No CLI surface change. `gocopy compile` produces byte-identical
  output to v0.6.0.

The symtable is a shadow today. The next release wires it into
codegen.
