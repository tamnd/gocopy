# v0.6.18 — codegen module-level: subscript + attribute load

Date: 2026-04-29
Spec: notes/Spec/1500/1528_gocopy_v0618_codegen_subscript_attr_load.md
Roadmap: notes/Spec/1500/1523_gocopy_v06x_part2_roadmap.md (band A)
CPython parity: Python/codegen.c (3.14.0): module-level
`compiler_subscript` / `compiler_attribute` for the load
form, driven by a top-level `Assign` whose value is a
`Subscript` or `Attribute`.

The v0.6.18 release is the fifth sub-release in band A of
the v0.6.x part 2 roadmap. It teaches the codegen package
two adjacent shapes:

- **modSubscriptLoad** — `<target> = <obj>[<key>]` where
  target, obj, and key are all Names of 1..15 ASCII chars
  on a single source line.
- **modAttrLoad** — `<target> = <obj>.<attr>` where target
  and obj are Names of 1..15 ASCII chars, attr is a 1..15
  ASCII char identifier, all on a single source line.

modSubscriptLoad is the first codegen path to emit
`BINARY_OP NbGetItem` with its 5 inline-cache words. The
6-code-unit run (1 op + 5 caches) lands in a single
line-table entry under the encoder's 8-unit cap.

modAttrLoad is the first codegen path to emit `LOAD_ATTR`
with its 9 inline-cache words. The 10-code-unit run is
split by the line-table encoder into two 8+2 entries with
the same Loc, matching the `bytecode.AttrLoadLineTable`
helper byte-for-byte.

The 246/246 oracle stays byte-identical. The parallel-output
gate (`TestCodegenParity`) automatically lights up fixtures
124 (`x = a[b]`) and 126 (`x = a.b`).

## Added

- `compiler/codegen/visit_subscript_load.go` —
  `subscriptLoadModule` type plus
  `classifySubscriptLoadModule` +
  `buildSubscriptLoadModule`. Single-statement only so the
  parity gate stays byte-equal with the classifier's
  `compileSubscriptLoad`.

- `compiler/codegen/visit_attr_load.go` —
  `attrLoadModule` type plus `classifyAttrLoadModule` +
  `buildAttrLoadModule`. Single-statement only so the
  parity gate stays byte-equal with the classifier's
  `compileAttrLoad`.

## Changed

- `compiler/codegen/codegen.go` — `Build` recognizes
  modSubscriptLoad and modAttrLoad after modCollection,
  ahead of the ErrUnsupported fallback.

- `compiler/codegen_parity_test.go` — `compileViaClassifier`
  picks up the modSubscriptLoad and modAttrLoad classifier
  paths so the parity gate validates the new codegen-owned
  fixtures against the classifier reference.

## Out of scope

- Subscript/attribute *store* shapes (`a[b] = x`,
  `a.b = x`) — fixtures 125 and 127, planned for v0.6.19.
- Nested subscripts (`a[b][c]`), nested attrs (`a.b.c`),
  mixed (`a.b[c]`, `a[b].c`) — classifier rejects too.
- Slices, tuple keys, computed keys — classifier rejects.
- Multi-line variants — classifier rejects too.
- modCallAssign — v0.6.20.
- modGenExpr — v0.6.21.
- Function definitions, classes, comprehensions, try/except
  — bands B and C of the part 2 roadmap.
- Optimizer passes, classifier funeral, perf — closing
  band.
