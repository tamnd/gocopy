# v0.5.10 — from-star-import in modMixed (2026-04-28)

## What changed

### Star import at module level

The modMixed classifier now accepts `from X import *` as an optional statement
between the docstring and the constLitColl. The import must be absolute (level=0)
and have a single `*` alias.

Bytecode emitted (5 CUs on one source line):
```
LOAD_SMALL_INT 0
LOAD_CONST     starImportIdx  # ('*',) tuple
IMPORT_NAME    moduleNameIdx
CALL_INTRINSIC_1 2             # INTRINSIC_IMPORT_STAR
POP_TOP
```

The `('*',)` tuple is inserted into co_consts right after the docstring (before
function code objects), matching CPython's constant ordering. The module name is
inserted into co_names right after `__doc__`.

### stmtFuncDef in modMixed

The modMixed funcdef loop now accepts both `stmtFuncBodyExpr` and `stmtFuncDef`
(simple `def f(arg): return arg` functions). A new `mixedFunc` union type carries
either kind; a `compileFuncDefInner` helper generates the inner code object for
`stmtFuncDef` entries without reusing the module-wrapper logic.

This was required because the classify_ast dispatcher tries `extractFuncDef`
before `extractFuncBodyExpr`, so single-arg return-arg functions always land as
`stmtFuncDef` even inside multi-statement modules.

## New fixture

- `245_star_import.py` — docstring + `from os import *` + 4-element `__all__`
