# v0.4.2 — constant literal collections (string list and tuple assignments)

## What's new

Handle module-level assignments where the RHS is a list or tuple whose
elements are all plain ASCII string constants. This covers `__all__` and
similar patterns.

CPython emits different bytecode sequences based on element count:

- **Tuple (any size ≥1)**: `LOAD_CONST full-tuple` + STORE_NAME + RETURN
  `co_consts = (first_elem, None, (all, elems, as, tuple))`
- **List 1 element**: `LOAD_CONST elem + BUILD_LIST 1` + STORE_NAME + RETURN
  `co_consts = (elem, None)`
- **List 2 elements**: `LOAD_CONST e0 + LOAD_CONST e1 + BUILD_LIST 2` + ...
  `co_consts = (e0, e1, None)`
- **List 3+ elements**: `BUILD_LIST 0 + LOAD_CONST tuple + LIST_EXTEND 1` + ...
  `co_consts = (first_elem, None, (all, elems, as, tuple))`

The 3+ list case uses LIST_EXTEND (new opcode 79) and covers `__all__`.

## New fixtures (214–220)

- 214 `x = ("a", "b", "c")` — string tuple, 3 elements
- 215 `x = ("hello",)` — string tuple, 1 element
- 216 `x = ["only"]` — string list, 1 element
- 217 `x = ["alpha", "beta"]` — string list, 2 elements
- 218 `__all__ = ["iskeyword", "kwlist", "issoftkeyword"]` — 3-elem __all__
- 219 `__all__ = ["iskeyword", "kwlist", "issoftkeyword", "softkwlist"]` — 4-elem __all__
- 220 `x = ("a", "b")` — string tuple, 2 elements
