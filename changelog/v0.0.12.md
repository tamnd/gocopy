## v0.0.12

`gocopy compile` accepts a pure-imaginary complex literal on the
right-hand side of a `name = literal` assignment: `x = 1j`,
`x = 0j`, `x = 0.5j`, `x = 1e2j`.

CPython lowers this the same way as float: `LOAD_CONST 0`, with
`consts = (complex_val, None)` and `noneIdx = 1`. No separate
folding artifact — the complex constant is at const index 0 directly.

Marshal emits `TYPE_BINARY_COMPLEX` (0x79): 16 bytes, real part as
IEEE 754 float64 little-endian followed by imaginary part as the same.
For pure-imaginary literals the real part is always `0.0`. Complex
is not immortal in CPython 3.14, so `FLAG_REF` is set only when the
same value appears more than once in the walk.

### Added

- `marshal.emitObject` handles `complex128` → `TYPE_BINARY_COMPLEX`
  (16-byte real+imag LE doubles). No FLAG_REF unless count > 1.
- `complexKeyType` and `complexKey()` in `marshal/writer.go`; `tupleKey`
  emits `'Y'` + 16 bytes; `refCounter.tuple` bumps `complexKey(x)`.
- `parseComplexLiteral` in `compiler/classify.go`: strips `j`/`J`
  suffix, parses the body as float64, returns `complex(0, f)`.
- Three new fixtures: `058_assign_complex_one.py`,
  `059_assign_complex_zero.py`, `060_assign_complex_half.py`.

### Out of scope (deferred)

- Negative complex literals (`x = -1j`).
- The `a+bj` form (requires expression parsing).
- Multiple sequential assignments (`x = 1\ny = 2`).
- Wiring gopapy as the parser.
