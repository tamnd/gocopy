# v0.1.4 - comparisons (2026-04-27)

Adds support for module-level assignments where the right-hand side is a
comparison between two names. All ten comparison operators are covered.

## Supported patterns

```python
x = a < b       # COMPARE_OP LT
x = a <= b      # COMPARE_OP LTE
x = a == b      # COMPARE_OP EQ
x = a != b      # COMPARE_OP NE
x = a > b       # COMPARE_OP GT
x = a >= b      # COMPARE_OP GTE
x = a is b      # IS_OP 0
x = a is not b  # IS_OP 1
x = a in b      # CONTAINS_OP 0
x = a not in b  # CONTAINS_OP 1
```

## Opcodes added

| Opcode | Number | Cache words |
|--------|--------|-------------|
| COMPARE_OP | 56 | 1 |
| IS_OP | 74 | 0 |
| CONTAINS_OP | 57 | 1 |

## New code

- `bytecode/cmp.go`: `CmpAssignBytecode`, `CmpAssignLineTable`.
- `CmpLt`, `CmpLtE`, `CmpEq`, `CmpNotEq`, `CmpGt`, `CmpGtE` oparg constants.
- `compiler/classify.go`: `modCmpAssign`, `stmtCmpAssign`, `cmpAssign` type.
- `compiler/classify_ast.go`: `cmpOpFromOp`; handles `*parser2.Compare` with a single
  comparison and two name operands.
- `compiler/compiler.go`: `compileCmpAssign`.
- 10 new fixtures: 104-113.

## Out of scope for this version

- Chained comparisons (`a < b < c`).
- Comparison with a literal on either side.
