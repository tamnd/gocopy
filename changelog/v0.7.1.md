# v0.7.1 — visit_Constant + visit_Name + visit_Module (empty / no-ops / docstring)

Date: 2026-04-29
Spec: notes/Spec/1500/1544_gocopy_v071_visit_constant_name.md
Architecture: notes/Spec/1500/1541_gocopy_v07_architecture.md
Roadmap: notes/Spec/1500/1542_gocopy_v07x_roadmap.md
CPython parity: Python/codegen.c (3.14.0): `codegen_body`,
`codegen_visit_stmt`, `codegen_visit_stmt_pass`,
`codegen_visit_stmt_expr`, `codegen_visit_expr_constant`,
`codegen_visit_expr_name`, `codegen_unit_finalize`.

The first v0.7.x feature release. v0.7.0 wired the visitor pipeline
but every node returned `ErrNotImplemented`; v0.7.1 lights up the
first three AST nodes (`Constant`, `Name`-Load, and the `Module`
driver itself) and **promotes** the three classifier shapes those
nodes cover (`modEmpty`, `modNoOps`, `modDocstring`) onto the new
path. After v0.7.1 the visitor is the authoritative producer for
every fixture today recognised as one of those shapes; the
classifier helpers, the dispatch arms, and the related test scaffolds
are deleted. The 246/246 oracle stays byte-identical because the
visitor reproduces CPython's post-optimizer output exactly.

## Added

- `compiler/codegen/visit_const.go` — `visitConstant` emits
  `LOAD_CONST` for an `*ast.Constant` of every supported kind
  (None, True, False, Ellipsis, int, float, complex, bytes, str)
  via `compileUnit.addConst`. Mirrors CPython's
  `codegen_visit_expr_constant`. Scaffolded for v0.7.2's
  `visit_Assign`.
- `compiler/codegen/visit_name.go` — `visitName` emits
  `LOAD_NAME` for a Load-context `*ast.Name` at module scope via
  `compileUnit.addName`. Load context only. Mirrors CPython's
  `codegen_visit_expr_name`. Scaffolded for v0.7.2's
  `visit_Assign`.
- `compiler/codegen/visit_stmt.go` — three predicates:
  `isModuleDocstring(stmt, source)` recognises a leading
  `*ast.ExprStmt` whose value is a plain-ASCII `str` Constant and
  resolves its multi-line Loc via `findTripleQuoteEnd`;
  `stmtNopLoc(stmt, source)` returns the Loc the visitor attaches
  to a no-op NOP for `*ast.Pass`, a non-string simple Constant, or
  a non-leading `str` Constant (via `tailStringLoc`);
  `singleLineLoc` and `tailStringLoc` are the underlying Loc
  builders. Mirrors CPython's `codegen_visit_stmt_pass` plus the
  constant branch of `codegen_visit_stmt_expr`.
- `compileUnit.addConst(any) uint32` and
  `addName(string) uint32` — append-or-reuse helpers that match
  CPython's `compiler_add_const_o` semantics: nil dedups by
  identity, other values dedup by `==` equality. Strings dedup by
  value, other consts by reference / equality.
- `GenerateOptions.Source []byte` and `compileUnit.Source` —
  source bytes threaded through the pipeline so the visitor can
  resolve end-of-line columns the same way the classifier does
  (`splitLines` + `lineEndCol`).
- `codegen.Generate` now returns the populated
  `(seq, consts, names, err)` tuple so `runVisitorShadow` can
  hand the `Consts` / `Names` tables to `assemble.Assemble`.
- `compiler/codegen/generate_test.go` — `TestGenerateEmptyModule`
  asserts the visitor emits the canonical three-instruction empty
  module; `TestAddConstDedupNil`,
  `TestAddConstDistinctValues`, and `TestAddNameDedup` cover the
  new dedup helpers.

## Changed

- `compiler/codegen/generate.go::visitModule` — replaces the
  v0.7.0 stub with a structural walk over `mod.Body`. Emits
  `RESUME 0` at the synthetic prologue Loc; for an empty body
  emits `LOAD_CONST None` + `RETURN_VALUE` at the same synthetic
  Loc; for a leading docstring emits `LOAD_CONST text` +
  `STORE_NAME __doc__` at the docstring's Loc; emits a `NOP` per
  non-last stmt in the tail; emits `LOAD_CONST None` +
  `RETURN_VALUE` at the last stmt's Loc (or the docstring's Loc
  when the tail is empty). This bakes the v0.7.13 trailing-NOP
  fold into the visitor so v0.7.1 can ship byte-identical output
  while `optimize.Run` is still identity.
- `compiler/compiler.go::Compile` — when `runVisitorShadow`
  returns a non-nil `*bytecode.CodeObject`, that result is
  authoritative; the classifier path is bypassed. The classifier
  remains as a fallback for shapes the visitor does not yet know.
  The `case modEmpty / modNoOps / modDocstring` switch arms are
  deleted.
- `compiler/visitor.go` — `runVisitorShadow` accepts source bytes
  and threads `GenerateOptions.Source` and the assembler's
  `Consts` / `Names` tables.

## Removed

- `compiler/classify.go` — the `modEmpty`, `modNoOps`, and
  `modDocstring` `kind` constants and their construction branches
  in `stmtsToClassification`. The `docLine` / `docEndLine` /
  `docCol` / `docText` fields on the classification struct are
  removed.
- `compiler/codegen/codegen.go::Build` — dispatch for the three
  promoted shapes. `Build` is reserved for shapes the visitor has
  not yet adopted.
- `compiler/codegen/visit_module.go` — `buildEmptyModule`,
  `buildNoOpsModule`, `buildDocstringModule`,
  `classifyNoOpModule`, `classifyDocstringModule`, and the
  `docstringModule` struct. The `noOpStmt` type and `locOf`
  helper stay — the assign / multi-assign / chained-assign /
  aug-assign tail-noop helpers still consume them, and they retire
  with their last consumer shape.
- `compiler/codegen_parity_test.go` — the
  `case modEmpty / modNoOps / modDocstring` switch arms.
- `compiler/codegen/codegen_test.go` — `TestBuildEmptyModule`,
  `TestBuildSinglePass`, `TestBuildThreePass`,
  `TestBuildSingleDocstring`, `TestBuildDocstringWithTail`,
  `TestBuildMultiLineDocstring`, and
  `TestBuildDocstringNonAsciiRejected`. `codegen.Build` no longer
  owns these shapes.

## Out of scope

- Any AST node beyond `Constant`, `Name`-Load, `Pass`, `ExprStmt`,
  and `Module`. `Assign`, `BinOp`, `UnaryOp`, `If`, `FunctionDef`,
  etc. land at v0.7.2+.
- `Name` Store / Del contexts. Load only.
- Any optimizer pass. `optimize.Run` is still identity; the
  visitor emits the post-optimization pattern directly for these
  three shapes.
- Promotion of `modAssign` or any later classifier shape.
- Rename / deletion of `codegen.Build`, `codegen.Options`,
  `codegen.ErrUnsupported`, or any v0.6 `visit_*.go` shape file
  the visitor does not yet subsume.
- gopapy parser changes.
