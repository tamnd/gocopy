# v0.6.17 — codegen module-level: collection literals

Date: 2026-04-29
Spec: notes/Spec/1500/1527_gocopy_v0617_codegen_collection.md
Roadmap: notes/Spec/1500/1523_gocopy_v06x_part2_roadmap.md (band A)
CPython parity: Python/codegen.c (3.14.0): module-level
`compiler_list` / `compiler_tuple` / `compiler_set` /
`compiler_dict`, driven by a top-level `Assign`.

The v0.6.17 release is the fourth sub-release in band A of
the v0.6.x part 2 roadmap. It teaches the codegen package
the modCollection shape: a single
`<target> = [...]`, `<target> = (...)`,
`<target> = {...}` (set), or `<target> = {k: v, ...}` (dict)
where every element (or key/value pair) is a Name on the
same source line as the assignment, plus the empty `[]`,
`()`, and `{}` (dict) cases. The target is a Name of 1..15
ASCII chars.

This is the first codegen path with a variable number of
LOAD_NAME instructions before a BUILD_LIST/TUPLE/SET/MAP,
and the first that emits the empty-tuple constant sentinel
(`LOAD_CONST 1`, with `co_consts = [None, ConstTuple{}]`).

For the non-empty case, co_consts = [None] and co_names is
[elt0, elt1, ..., target] in insertion order — for dicts
the keys and values alternate (`a, b, ...` for `{a: b, ...}`).
co.StackSize equals N for N >= 2 elements.

The 246/246 oracle stays byte-identical. The parallel-output
gate (`TestCodegenParity`) automatically lights up fixtures
117–123 (seven collection fixtures spanning empty list/tuple/
dict and non-empty list/tuple/set/dict).

## Added

- `compiler/codegen/visit_collection.go` —
  `collectionModule` type plus `classifyCollectionModule` +
  `buildCollectionModule`. Single-statement only so the
  parity gate stays byte-equal with the classifier's
  `compileCollection`.

## Changed

- `compiler/codegen/codegen.go` — `Build` recognizes the
  modCollection shape after modTernary, ahead of the
  ErrUnsupported fallback.

- `compiler/codegen_parity_test.go` — `compileViaClassifier`
  picks up the modCollection classifier path so the parity
  gate validates the new codegen-owned fixtures against the
  classifier reference.

## Tests

- 246/246 fixtures byte-identical (`tests/run.sh`).
- `compiler/codegen/codegen_test.go` adds:
  `TestBuildCollectionEmptyList`,
  `TestBuildCollectionEmptyTuple`,
  `TestBuildCollectionEmptyDict`,
  `TestBuildCollectionListNames`,
  `TestBuildCollectionDictNames`,
  `TestBuildCollectionWithTailFallsThrough`.
- `compiler/codegen_parity_test.go` automatically validates
  fixtures 117–123 via the new arm.

## Out of scope

- Multi-line collection literals — classifier rejects them
  too.
- Constant-literal collections (`x = ["a", "b"]`,
  `x = (1, 2)`) — handled by modConstLitColl, distinct
  shape with constant folding.
- Comprehensions — modGenExpr (band A v0.6.21).
- modSubscriptLoad / modAttrLoad — v0.6.18 (next
  sub-release).
- Function definitions, classes, comprehensions, try/except —
  bands B and C of the part 2 roadmap.
- Optimizer passes, classifier funeral, perf — closing band.
