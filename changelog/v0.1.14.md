# v0.1.14 - gopapy import fix (2026-04-27)

Updates the parser dependency from the old `gopapy/v2` path to the
current `gopapy` module at v0.2.11.

## What changed

The compiler was importing `github.com/tamnd/gopapy/v2/parser2` with
version v2.0.0, which was a leftover from before the gopapy module-path
cleanup. The correct import is now `github.com/tamnd/gopapy/parser` at
v0.2.11.

Two files updated:

- `compiler/compiler.go`: import alias `parser2` now points to
  `github.com/tamnd/gopapy/parser`
- `compiler/classify_ast.go`: same import fix

The alias `parser2` is kept so the call sites do not need to change.
`go.mod` drops the old `/v2` entry after `go mod tidy`.

## No behaviour change

All 137 existing fixtures still pass. The generated bytecode is
identical before and after.

## Why this was wrong

The gopapy module dropped the `/v2` major-version suffix and renamed the
`parser2` package to `parser` as part of a module-path cleanup. The
gocopy dependency was never updated after that cleanup.
