## v0.0.1

First public cut. `gocopy compile FILE.py` produces a CPython 3.14
`.pyc` that is byte-for-byte identical to
`python3.14 -m py_compile FILE.py` for an empty Python source file.

The point of v0.0.1 is the plumbing, not the language coverage. One
fixture (`tests/fixtures/001_empty.py`, zero bytes), one oracle diff,
every package wired end to end. Every feature the roadmap lifts after
this is a localised change rather than a re-bootstrap.

### Pipeline

```
source.py
  ├── compiler.Compile     →  bytecode.CodeObject
  ├── marshal.Marshal      →  []byte (CPython marshal stream)
  └── pyc.WriteFile        →  source.pyc (16-byte header + marshal stream)
```

gopapy is the AST source for v0.0.2 onward; v0.0.1 only needs to
recognise "this file is empty", so it short-circuits with a
whitespace-and-comments scan and emits the canonical empty-module
Code object directly.

### Added

- `gocopy compile FILE.py [-o OUT.pyc]` CLI. Defaults the output
  path to `__pycache__/FILE.cpython-314.pyc` to match `py_compile`.
  `--mode timestamp` (default), `--mode hash`, and
  `--mode unchecked-hash` cover the three validation-field
  variants. `--source-date-epoch N` and the `SOURCE_DATE_EPOCH`
  env var override the source mtime baked into timestamp mode.
- `gocopy version` and `gocopy help`.
- `bytecode` package with the three opcodes the empty module needs
  (`RESUME`, `LOAD_CONST`, `RETURN_VALUE`) and the full 256-entry
  inline-cache table sourced from goipy's `op` package. goipy is
  the canonical opcode source in this ecosystem; gocopy follows it
  rather than redefining the constants.
- `compiler.Compile` for empty modules. Whitespace and `#` comments
  count as empty; anything else returns
  `compiler: v0.0.1 only supports empty modules`.
- `marshal.Marshal` covering exactly the type tags an empty Code
  object emits: `TYPE_CODE`, `TYPE_NONE`, `TYPE_INT`, `TYPE_STRING`,
  `TYPE_SMALL_TUPLE`, `TYPE_SHORT_ASCII_INTERNED`, `TYPE_REF`. The
  writer is two-pass: first walk counts how many times each object
  appears, second walk emits with `FLAG_REF` and back-references.
  Same byte sequence as CPython's `Py_REFCNT > 1` heuristic, just
  reached without refcounts.
- `pyc.WriteFile` with all three invalidation modes. Hash mode uses
  SipHash-1-3 with `k0 = magic_number_token`, `k1 = 0`, matching
  `_imp.source_hash`. Vectors locked against Python 3.14.4 in
  `pyc/siphash_test.go`.
- CI: `go test ./...`, the oracle byte-diff, an informational
  stdlib-compile counter, and a Windows `go vet` / `go build` job.
- Release CI: tag-driven cross-platform builds for
  linux / darwin / windows × amd64 / arm64.

### Fixture

`tests/fixtures/001_empty.py` (zero bytes). The oracle harness in
`tests/run.sh` byte-diffs gocopy's output against
`python3.14 -m py_compile`'s output for the same input under a
fixed `SOURCE_DATE_EPOCH` so timestamps don't flake CI.

### Deferred

Anything that isn't an empty module. v0.0.2 wires in the gopapy
AST and starts adding real top-level statements one rung at a time.
