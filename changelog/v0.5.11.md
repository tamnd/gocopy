# v0.5.11 — call statements and function docstrings in modMixed

## What changed

Added support for two new constructs in `funcBodyInfo` functions inside `modMixed` modules:

1. **Function docstrings** — a string literal as the first statement of a function body.
   - `co_consts[0]` is the docstring; `None` is not added to co_consts.
   - `CO_HAS_DOCSTRING` flag (0x4000000) is set on the code object.

2. **Bare call statements** — `func(args)` expressions used as statements (result discarded).
   - Compiles to the same CALL sequence as an assigned call, followed by `POP_TOP`.
   - `POP_TOP` is merged into the preceding CALL's linetable entry (same source span, 5 CUs total).

## Supporting fixes

- `extendLastEntry(n)` helper in `funcState` increments the last linetable entry's CU count by n.
- Module-level `StackSize` in `compileMixed` corrected: starts at 1 (not 2); bumps to 2 for star import or CLC; bumps to `max(2, K)` for K Name-defaults.

## New fixture

`tests/fixtures/246_call_stmts.py` — a `modMixed` module with:
