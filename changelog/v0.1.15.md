# v0.1.15 - general expression compiler (2026-04-28)

Adds a recursive expression compiler for module-level assignments where
the right-hand side is composed of Name, small-int Constant (0-255),
BinOp, and UnaryOp (USub/Invert) nodes in any nesting.

## Supported patterns (new)

```python
x = a + b + c        # nested left-associative BinOp
x = a * b + c * d    # multiple BinOps with different operators
x = a + b * c        # BinOp with operator precedence
x = a + 1            # BinOp with small int literal on right
x = 1 + a            # BinOp with small int literal on left
x = -a + b           # UnaryOp (negation) nested in BinOp
```

The key limitation of the previous code was that each BinOp required
both operands to be plain Name nodes. The new recursive compiler handles
any nesting depth.

## Architecture

New classification kind `modGenExpr` routes single-target assignments
to `compileGenExpr` when the existing per-pattern handlers return false.
The recursive `genState.walk` compiles expression trees bottom-up,
emitting LOAD_NAME, LOAD_SMALL_INT, BINARY_OP, or UNARY_NEGATIVE/INVERT
instructions and the corresponding PEP 626 linetable entries.

Linetable pattern for module-level expressions:
- LONG(1) prologue for RESUME
- ONE_LINE1 for the first LOAD
- SHORTn entries (delta=0) for all subsequent loads and operations
- BINARY_OP entries cover [left_startCol, right_endCol) using 6 CU
  (1 instruction + 5 inline-cache words)
- SHORT0(3)[0,targetLen) for STORE_NAME + LOAD_CONST + RETURN_VALUE

## co_consts rule

For expressions with small-int literals: the FIRST constant encountered
in evaluation order is added to co_consts[0] even when LOAD_SMALL_INT
is used. Subsequent small-int constants are not added. This matches
CPython's co_consts layout exactly.

## New code

- `bytecode/gen_expr_lt.go`: exports `GenExprProlog`, `GenExprFirstEntry`,
  `GenExprSameLine` as wrappers over unexported linetable helpers.
- `compiler/gen_expr.go`: `compileGenExpr`, `genState` struct, `walk` method.
- `compiler/classify.go`: `modGenExpr`, `stmtGenExpr`, `genExprInfo`.
- `compiler/classify_ast.go`: `extractGenExprAssign`, `isGenExpr`.
- 6 new fixtures: 138-143.

## Out of scope for this version

- Float, large int, True, False, None literals in expressions.
- BoolOp (and/or), IfExp (ternary) with non-Name operands.
- UnaryOp (not) which requires TO_BOOL + UNARY_NOT.
- Parenthesized sub-expressions whose outer parens affect linetable ranges.
