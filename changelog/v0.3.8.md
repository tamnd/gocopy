# v0.3.8 — LOAD_GLOBAL reads in function bodies (2026-04-28)

Added support for reading module-level names (globals) inside function bodies,
enabling patterns like `if hue < ONE_SIXTH: return 0`.

## What changed

`isFuncBodyExpr` (classify_ast.go): the `*parser2.Name` case no longer
requires the name to be in `knownNames`. Any name with len <= 15 and
col <= 255 is accepted — unknown names are assumed to be module-level globals
that will be loaded via LOAD_GLOBAL.

`walkExpr` (func_body.go): the `*parser2.Name` case now checks whether the
name is in `fs.slots` (a local/param). If yes, emits LOAD_FAST_BORROW as
before. If no, emits LOAD_GLOBAL with oparg = `(nameIdx << 1) | 0` (bit 0 =
0, no NULL pushed) followed by 4 cache words.

LFLBLFLB guard (func_body.go): all LFLBLFLB fast-path checks — in BinOp,
Compare, Subscript, Tuple, Call args, and all condition blocks — now verify
that both names are actually in `fs.slots` (local) before using the
superinstruction. Previously the guard was `slot <= 15`; now it is
`isLocal && slot <= 15`. This prevents a global name from being silently
misidentified as slot 0.

Name-specific LOAD_FAST paths (simple assignment RHS, isIfAssign then-body,
ternary false branch) similarly check `isLocal` before emitting LOAD_FAST;
globals fall through to `walkExpr`.

## Fixtures

- `197_funcbody_global_read.py`: `if x < LIMIT: return x; return LIMIT`
- `198_funcbody_global_compare.py`: `if hue < ONE_SIXTH: return 0; return 1`
