# v0.1.9 - if/elif/else (2026-04-27)

Adds support for `if`/`elif`/`else` chains at module level. Each branch
condition must be a name; each branch body must be a single `name = small_int`
assignment (value 0-255).

## Supported patterns

```python
if cond:
    x = 1

if cond:
    x = 1
else:
    x = 2

if cond1:
    x = 1
elif cond2:
    x = 2
else:
    x = 3
```

The condition is evaluated with `TO_BOOL` and its result drives a
`POP_JUMP_IF_FALSE` into the next branch. `NOT_TAKEN` is emitted immediately
after each conditional jump, following CPython's branch prediction hint pattern.
The `elif` chain is represented in the AST as a nested `If` node inside the
`Orelse` of the parent; gocopy flattens that recursively.

## Opcodes added

| Opcode | Number | Cache words |
|--------|--------|-------------|
| TO_BOOL | 39 | 3 |
| POP_JUMP_IF_FALSE | 100 | 1 |
| NOT_TAKEN | 28 | 0 |
| LOAD_SMALL_INT | 94 | 0 |

(TO_BOOL was already introduced in v0.1.3 for `UNARY_NOT`; here it gains its
3-cache-word count in the opcode table.)

## New code

- `bytecode/ifelse.go`: `IfBranch`, `IfBranchLT`, `IfElseBytecode`, `IfElseLineTable`.
- `compiler/classify.go`: `modIfElse`, `stmtIfElse`, `ifElseBranch`, `ifElseClassify`.
- `compiler/classify_ast.go`: handles `*parser2.If` recursively (elif = nested If
  in Orelse).
- `compiler/compiler.go`: `compileIfElse`.
- 3 new fixtures: 131-133.

## Out of scope for this version

- Multi-statement branch bodies.
- Conditions that are not plain names.
- Branch bodies other than a single small-int assignment.
- `break` and `continue` inside loops (not yet at loops).
