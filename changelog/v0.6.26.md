# v0.6.26 — codegen module-level: closuredef (band C)

Date: 2026-04-29
Spec: notes/Spec/1500/1536_gocopy_v0626_codegen_closuredef.md
Roadmap: notes/Spec/1500/1523_gocopy_v06x_part2_roadmap.md (band C)
CPython parity: Python/codegen.c (3.14.0): module-level closure
lowering of `def f(x): def g(): return x; return g`.

The v0.6.26 release is the second sub-release in band C of the
v0.6.x part 2 roadmap. It teaches the codegen package the
modClosureDef shape:

- A single top-level `def f(x): def g(): return x; return g`
  definition where f, x, and g are 1..15-char ASCII identifiers,
  the outer body is exactly two stmts (inner def + `return g`),
  the inner body is exactly one `return x`, and there are no
  decorators, annotations, defaults, or variadic params anywhere.

This is the first codegen path that emits three nested
CodeObjects (module → outer → inner) and the first to exercise
the cell + free closure-variable machinery (LocalsKindArgCell on
the outer arg slot, LocalsKindFree on the inner). All three
CodeObjects are hand-built mirroring the classifier's
`compileClosure` byte-for-byte using the existing
`bytecode.ClosureOuter*` / `bytecode.ClosureInner*` /
`bytecode.FuncDefModule*` helpers — no IR/assembler routing yet.

Notable byte-equality details, all mirroring the classifier's
`compileClosure` byte-for-byte:

- **Inner `g`**: `RESUME 0` / `RETURN_CONST nil` style emission
  via `bytecode.ClosureInnerBytecode()`. ArgCount=0, StackSize=1,
  Flags=0x13 (CO_OPTIMIZED|CO_NEWLOCALS|CO_NESTED).
  LocalsPlusNames=[argName], LocalsPlusKinds=[LocalsKindFree].
  Name=innerFuncName; QualName=
  `<outerFuncName>.<locals>.<innerFuncName>`.
  FirstLineNo=innerDefLine.
- **Outer `f`**: emission via `bytecode.ClosureOuterBytecode()`.
  ArgCount=1, StackSize=2, Flags=0x03. LocalsPlusNames=[argName,
  innerFuncName], LocalsPlusKinds=[LocalsKindArgCell,
  LocalsKindLocal]. co_consts=[innerCode] (no None — all paths
  return explicitly). Name=QualName=outerFuncName.
  FirstLineNo=outerDefLine.
- **Module wrapper**: identical to modFuncDef — RESUME /
  LOAD_CONST 0 / MAKE_FUNCTION / STORE_NAME / LOAD_CONST 1 /
  RETURN_VALUE. co_consts=[outerCode, nil],
  co_names=[outerFuncName]. FirstLineNo=outerDefLine.

The 246/246 oracle stays byte-identical. The parallel-output
gate (`TestCodegenParity`) automatically lights up fixture 137
`closure_simple.py`.

After v0.6.26 the only remaining band-C release is v0.6.27
modFuncBodyExpr (general function body: assigns + return),
which closes the band.

## Added

- `compiler/codegen/visit_closuredef.go` — `closureDefModule`
  type plus `classifyClosureDefModule` + `buildClosureDefModule`.
  Mirrors `compiler/compiler.go::compileClosure` and
  `compiler/classify_ast.go::extractClosure` byte-for-byte.

## Changed

- `compiler/codegen/codegen.go` — `Build` recognizes
  modClosureDef after modFuncDef, ahead of the ErrUnsupported
  fallback. Doc comment updated.

- `compiler/codegen_parity_test.go` — `compileViaClassifier`
  picks up the modClosureDef classifier path so the parity gate
  validates the new codegen-owned fixture.

## Out of scope

- General function bodies (band C, v0.6.27).
- Nested closures more than one level deep.
- Multiple inner defs.
- Inner functions that take args.
- Decorators / annotations / defaults / vararg / kwarg.
- Routing inner/outer code objects through assemble.Assemble.
