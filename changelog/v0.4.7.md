# v0.4.7 — CPython struct.py compilation (2026-04-28)

Extends the compiler to handle a module body of one constant-literal list
assignment followed by from-import statements. Four gaps closed.

## What's new

### Multi-line small list with multiple elements per line

`extractLargeStringList` required each element on its own line. `struct.py`'s
`__all__` has 8 elements across lines 3–10, with 5 of them on line 3.
A new `extractMultiLineSmallList` handles lists of any line layout: it validates
all string elements, then scans forward for the closing `]`. It is tried as a
fallback when `extractLargeStringList` fails for a first-element-not-on-opening-line
list with fewer than 31 elements.

### New classification: `modClcThenImports`

A module body of exactly one constant-literal list assignment followed by one or
more from-import/import statements. Detected in `stmtsToClassification` after the
single-CLC check.

co_consts layout: `(first_elem, fromlist_1, …, fromlist_k, None, full_tuple)`.
Bytecode emits `BUILD_LIST 0 + LOAD_CONST tupleIdx + LIST_EXTEND 1 + STORE_NAME`
for the list, then each import statement, then `LOAD_CONST noneIdx + RETURN_VALUE`.

### Wildcard from-import (`from X import *`)

`cuCount()` now returns 5 for a wildcard entry (`len(Aliases)==1 && Name=="*"`).
`ImportBytecode` emits `CALL_INTRINSIC_1 2 + POP_TOP` for the wildcard path.

### `'*'` string interning

CPython pre-interns `'*'` at runtime startup, so it appears as
`TYPE_SHORT_ASCII_INTERNED` in marshalled output. The marshal writer now uses
`isInterned(s)` — which returns true for `isAllNameChars(s)` or `s == "*"` — in
place of the bare `isAllNameChars` call when deciding the marshal type for string
constants.

A new `ClcThenImportsLineTable` in `bytecode/import.go` generates the combined
linetable: RESUME prologue + LONG entry for the multi-line list build (3 CUs) +
SHORT0 STORE_NAME (1 CU) + per-import entries via the existing import-entry helpers.
