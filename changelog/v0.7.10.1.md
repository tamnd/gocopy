v0.7.10.1 opens the v0.7.10.x systematic series. It is a
**plumbing-only** release: open the new fixture directory,
teach the discovery paths to walk it, and ship the first 8
baseline fixtures. No visitor changes. No flowgraph changes.
The 1:1 CPython-port work begins in v0.7.10.2.

## Added

- `tests/fixtures/funcbody/` — new flat directory hosting
  funcbody-focused fixtures across the v0.7.10.x series.
- 8 baseline fixtures (`001..008`):
  - `001_args_zero.py` — `def f(): return 0`.
  - `002_args_zero_const_assign.py` — `def f(): x = 0; return x`.
  - `003_args_many_eight.py` — eight-parameter signature.
  - `004_name_param_only.py` — `def f(x): return x` (LOAD_FAST exit).
  - `005_name_local_assign_then_read.py` — `x = 1; return x`.
  - `006_args_two_left.py` — two params, return left.
  - `007_stmt_aug_sub.py` — `a -= 1; return a`.
  - `008_stmt_two_assigns.py` — two const assigns + return.
- `compiler/run_sh_test.go::TestRunShWalksFuncbody` — asserts
  `tests/run.sh` retains globs over both `tests/fixtures/*.py`
  and `tests/fixtures/funcbody/*.py`. Guards against regressions
  in run.sh refactors.

## Changed

- `tests/run.sh` — refactored the per-fixture diff into a
  `run_oracle()` helper and walks two passes: top-level
  `tests/fixtures/*.py` then `tests/fixtures/funcbody/*.py`.
  Total fixture count is now 254 (246 top-level + 8 funcbody).
- `compiler/visitor_parity_test.go::TestVisitorParity` —
  discovery extended to glob `tests/fixtures/funcbody/*.py`
  in addition to the top-level layer.

## Removed

- Nothing.

## Tests

- `TestVisitorParity`: `byte-identical=213 not-implemented=41
  mismatched=0 errored=0 skipped=0 (total=254)`. Up from
  205/246 at v0.7.10. All 8 new baselines route through the
  visitor.

- `go build ./...` clean. `go vet ./...` clean.
- `go test ./...` clean.

## Out of scope — moves to v0.7.10.2+

- Any visitor change. v0.7.10.2 collapses the per-shape `If`
  helpers into a single recursive `visitIfStmt` mirroring
  CPython's `codegen_if`.
- Any flowgraph change. v0.7.10.3 ports
  `insert_superinstructions`; v0.7.10.4 ports
  `optimize_load_fast`.
- Any of the not-yet-covered shapes from spec 1556 §3 (`pass`
  body, multi-target Assign, BoolOp, walrus, kw-only args, etc).
- The 1:1 CPython port `// MIRRORS:` headers — start landing
  in v0.7.10.2.
