# v0.5.9 — small-int literal assigns in modMixed (2026-04-28)

## What changed

### Small-int assigns in modMixed

The modMixed classifier now accepts module-level integer assignments where the
value is in range [0, 255]. These compile to LOAD_SMALL_INT (opcode 94) instead
of LOAD_CONST, with no entry added to co_consts.

Example: `IFLAG = 0`, `OFLAG = 1`, `CFLAG = 2` alongside function definitions.

Previously only foldedBinOp assigns (e.g. `ONE_THIRD = 1.0 / 3.0`) were accepted
in the assign section of modMixed. The check now also passes `int64` values in
[0, 255].

In `compileMixed`, the assign bytecode loop switches on value type: `int64`
emits LOAD_SMALL_INT; `foldedBinOp` emits LOAD_CONST from co_consts as before.

### Single-line CLC encoding fix

`MixedModuleLineTable` previously used the LONG linetable entry format for the
constLitColl (BUILD_LIST + LOAD_CONST + LIST_EXTEND) even when the entire list
was on a single line. CPython uses the compact ONE_LINE2 encoding in that case.
Fixed by checking `CLCCloseLine == CLCLine` and branching to `appendValueEntryN`
for the single-line case.
