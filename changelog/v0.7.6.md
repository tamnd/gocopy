# v0.7.6 — visit_If + per-branch implicit return + eliminate_empty_blocks (phase B opener)

Date: 2026-04-29
Spec: notes/Spec/1500/1548_gocopy_v076_visit_if.md
Architecture: notes/Spec/1500/1541_gocopy_v07_architecture.md
Roadmap: notes/Spec/1500/1542_gocopy_v07x_roadmap.md
CPython parity: Python/codegen.c (3.14.0) `codegen_visit_stmt_if`
(the if/elif/else statement visitor, ported as `visitIfStmt`),
`codegen_jump_if` (the cond → POP_JUMP_IF_{FALSE,TRUE} helper,
Name-cond branch ported as `emitJumpIfFalse`); Python/flowgraph.c
(3.14.0) `eliminate_empty_basic_blocks` (the new optimizer pass)
and `inline_small_or_no_lineno_blocks` (functionally mirrored
in-visitor by emitting per-branch implicit-return-None tails up
front rather than cloning a merge block post-CFG).

The sixth v0.7.x feature release. v0.7.5 closed phase A by
promoting every expression-RHS shape the v0.6 classifier owned;
v0.7.6 opens phase B with the first statement-level shape:
`if/elif/else` chains where every condition is a 1..15-char Name
and every body is a single small-int Assign. This is the first
release where the visitor emits a multi-block CFG (one entry
block, one body block per branch, optional kept-merge end block)
and the first to drive the optimize pipeline through a real
branching shape. The v0.6.22 modIfElse codegen classifier file
and Build dispatch arm are gone; the legacy
`compiler/compiler.go::compileIfElse` fallback stays alive for
shapes the visitor still rejects (multi-stmt body, non-Name cond,
non-small-int RHS) until v0.7.10 widens the visitor surface.

The 246/246 oracle stays byte-identical and the visitor's parity
count climbs from 135/246 to 138/246 (131_if_simple.py,
132_if_else.py, 133_if_elif_else.py).

## Added

- `compiler/codegen/visit_if_stmt.go` — `visitIfStmt`, the
  if/elif/else statement visitor, plus three helpers:
  `emitJumpIfFalse` (LOAD_NAME + TO_BOOL + POP_JUMP_IF_FALSE for
  a Name cond), `emitIfBranchAssign` (LOAD_SMALL_INT @ valLoc +
  STORE_NAME @ varLoc for a single small-int Assign, returns
  varLoc), and `emitImplicitReturnNone` (LOAD_CONST None +
  RETURN_VALUE at a given Loc). visitIfStmt allocates an
  endLabel only in the no-orelse case (with-orelse uses RETURN_
  VALUE per branch and never falls through to a merge), emits
  the cond head into the entry block, AddBlocks for the body and
  emits its NOT_TAKEN + Assign + implicit-return-None tail,
  recurses for elif (`*ast.If` in orelse) or emits the else
  branch directly, and for the no-orelse case emits the kept-
  merge end block at condLoc. Sets `u.tailEmitted = true` so
  visitModule skips its own trailing terminator emit.
- `compiler/optimize/eliminate_empty_blocks.go` —
  `eliminateEmptyBlocks`, mirroring CPython's
  `eliminate_empty_basic_blocks`. Walks empty labelled blocks,
  transfers the label to the first non-empty successor (or
  retargets jumps when the successor is also labelled), and
  removes the empty block. v0.7.6 visit_If shapes emit no empty
  bridging blocks, so the pass is a no-op for current fixtures;
  it lands now so future shapes that DO produce empty bridges
  have it ready.
- `compileUnit.tailEmitted` field — records that a statement
  visitor has already emitted the module's trailing implicit
  return (directly or once-per-branch). visitModule reads it
  before emitting its own LOAD_CONST None + RETURN_VALUE pair.

## Changed

- `compiler/codegen/visit_stmt.go::visitStmt` — adds an
  `*ast.If` arm that dispatches to `visitIfStmt`. `isLast=false`
  returns `ErrNotImplemented` (the v0.7.6 surface covers only
  top-level single-stmt-body if chains).
- `compiler/codegen/generate.go::visitModule` — when
  `u.tailEmitted` is true, skips the trailing
  `LOAD_CONST None + RETURN_VALUE` emit; `finalizeDeferred` is
  still called for any pending const-pool patches.
- `compiler/codegen/codegen.go::Build` — drops the
  `classifyIfElseModule` / `buildIfElseModule` dispatch arm.
  `Build` now starts at `classifyWhileModule`; only four
  statement-level shapes (modWhile, modFor, modFuncDef,
  modClosureDef) remain on the codegen classifier surface.
  Package doc updated to reflect v0.7.6 promotion.
- `compiler/optimize/optimize.go::Run` — adds
  `eliminateEmptyBlocks` to the pipeline before
  `inlineSmallExitBlocks`. Package doc updated to describe the
  new three-pass v0.7.6 pipeline.

## Removed

- `compiler/codegen/visit_if_else.go` — the v0.6.22 modIfElse
  classifier file (~280 LOC: `classifyIfElseModule`,
  `buildIfElseModule`, helper structs).
- `compiler/codegen/codegen_test.go` — the
  `TestBuildIfElseSimpleNoElse`, `TestBuildIfElseWithElse`,
  `TestBuildIfElseElifElse`, `TestBuildIfElseMultiVarFallsThrough`,
  and `TestBuildIfElseNonNameCondFallsThrough` tests that called
  `Build` directly to assert specific bytecode / fall-through for
  the deleted classifier shape. Oracle parity, visitor parity,
  and `TestCodegenParity` still cover the build-side behaviour
  end-to-end.

## Out of scope

- `visit_While`, `visit_For`. The first backward-jump shape;
  v0.7.7 lifts the `resolveJumps` panic.
- Compare / BoolOp / IfExp conds in `if`. The Name-only surface
  matches v0.6's modIfElse exactly; v0.7.10 generalises after
  function bodies are in place.
- Multi-stmt branch bodies. The v0.6 modIfElse classifier didn't
  accept these either; v0.7.10 generalises.
- The full optimize_cfg pass. v0.7.6 ships only the new
  `eliminate_empty_blocks` pass plus the pre-existing v0.7.5
  inline pass; the rest of CPython's `optimize_cfg` (DCE, jump
  threading, redundant-load elim, NOP removal) lands at v0.7.13.
- gopapy parser changes.
