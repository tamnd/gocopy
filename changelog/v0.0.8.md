## v0.0.8

`gocopy compile` accepts two more right-hand sides on a leading
`name = literal` assignment: the `...` literal and a plain-ASCII
bytes literal (single or double quoted, no escapes). The bytecode
shape is the same RESUME + LOAD_CONST + STORE_NAME + tail + LOAD_CONST
None + RETURN_VALUE that v0.0.7 emits; only the marshal layer needs
new entries to round-trip the new const types.

Two new const types reach the marshal stream:

- Python's `Ellipsis` singleton (the value of `...`) goes out as a
  single `TYPE_ELLIPSIS` byte (`0x2e`) with no `FLAG_REF`. CPython
  treats `Ellipsis` as a built-in singleton and never assigns it a
  ref slot.
- Bytes literals (`b"hi"`, `b""`) go through the existing TYPE_STRING
  bytestring path. Non-empty bytes get plain `TYPE_STRING` (no
  `FLAG_REF`); empty bytes always get `TYPE_STRING|FLAG_REF` and then
  dedup with the empty `localspluskinds` and `exctable` slots through
  `TYPE_REF`. Both rules already lived in `marshal.bytestring`; the
  new path just routes consts into it.

### Added

- `bytecode.Ellipsis` and `bytecode.EllipsisType`: the gocopy
  sentinel for Python's `Ellipsis`. Goes into `CodeObject.Consts` as
  any other literal value.
- Marshal support for `[]byte` and `bytecode.EllipsisType` in
  `emitObject`, plus matching `tupleKey` / `refCounter` recursion so
  the empty-bytes singleton dedups correctly.
- `tryParseAssign` accepts `...` and `b"..."` on the right-hand
  side. Identifier rules are unchanged from v0.0.7.
- Five new fixtures: `035_assign_ellipsis.py`,
  `036_assign_bytes.py`, `037_assign_empty_bytes.py`,
  `038_assign_bytes_then_pass.py`,
  `039_assign_ellipsis_then_pass.py`.

### Out of scope (deferred)

- Right-hand side integer / float / complex literals; `LOAD_SMALL_INT`
  and the `TYPE_INT` / `TYPE_FLOAT` / `TYPE_BINARY_FLOAT` marshal
  paths are still pending.
- Docstring + assignment combo (`"""doc"""\nx = None`); needs a new
  bytecode shape (RESUME + 2x LOAD_CONST/STORE_NAME) and a wider
  names tuple.
- Non-ASCII bytes (`b"\xff"`); needs the marshal layer to emit
  arbitrary bytes through TYPE_STRING (the current path is fine, but
  the parser still rejects backslashes).
- Multi-target assignment (`a = b = None`), augmented assignment
  (`x += 1`), and any expression on the RHS beyond a single literal.
- Assignment after a leading docstring or after another statement;
  v0.0.8 only accepts an assignment as the first statement.
- Wiring gopapy as the parser; still waiting on a gopapy v1.0.0.
