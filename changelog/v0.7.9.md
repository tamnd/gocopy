# v0.7.9 — closures (visit_ClosureDef + cell/free machinery)

Date: 2026-04-29
Spec: notes/Spec/1500/1552_gocopy_v079_visit_closure.md
Architecture: notes/Spec/1500/1541_gocopy_v07_architecture.md
Roadmap: notes/Spec/1500/1542_gocopy_v07x_roadmap.md
CPython parity: Python/codegen.c (3.14.0)
`codegen_visit_stmt_function_def` (the function-def statement
visitor, here invoked twice — outer at module scope, inner at
outer scope); `codegen_make_closure` (the closure-tuple build
sequence inlined into `visitClosureDef`); Python/compile.c
(3.14.0) `compiler_make_cell` and `compiler_copy_free_vars`
(the synthetic prologue ops for cell-promoted args and free-var
inits); Python/assemble.c (3.14.0) `_PyAssemble_MakeCodeObject`
(exercised on a doubly-nested unit for the first time).

v0.7.8 opened band C with the leaf single-arg single-return
function-def shape. v0.7.9 closes the closure step: the same
outer surface, but the body is `def g(): return arg; return g`
instead of `return arg`. The inner `def` captures `arg` from the
outer scope, forcing three new pieces of machinery into the
visitor:

1. **Doubly-nested compile units.** Module pushes outer; outer
   pushes inner. The v0.7.8 push/pop pair already supported any
   depth — this release just exercises depth 2 for the first time.
2. **Cell promotion at the outer scope.** `arg` becomes a
   `LocalsKindArgCell`, and the outer's bytecode opens with
   `MAKE_CELL 0` BEFORE its `RESUME 0`.
3. **Free-variable read at the inner scope.** `arg` becomes a
   `LocalsKindFree`, and the inner's bytecode opens with
   `COPY_FREE_VARS 1` BEFORE its `RESUME 0`, reads via
   `LOAD_DEREF 0`, and returns.

The v0.6.26 codegen-side classifier (`compiler/codegen/visit_closuredef.go`,
`classifyClosureDefModule` + `buildClosureDefModule`) is
**promoted off** the codegen surface. The legacy
`compiler/compiler.go::compileClosure` fallback stays alive for
shapes the visitor still rejects (multiple captured names,
multi-stmt bodies, decorators / annotations / defaults /
vararg / kwarg, etc.); the visitor catches the v0.6.26
classifier's exact slice byte-for-byte.

After v0.7.9 the codegen-classifier surface (`compiler/codegen/codegen.go::Build`)
is **empty**. Every shape that ever reached `codegen.Build` is
now visitor-owned; `Build` returns `ErrUnsupported`
unconditionally and the legacy `compiler.Compile` fallback path
takes over. The empty stub stays alive until the v0.7.26 funeral
so `runVisitorShadow` can keep `codegen.Build` as its first call
without a signature break.

The 246/246 oracle stays byte-identical and the visitor's parity
count climbs from 141/246 to 142/246 (137_closure_simple.py).

## Added

- `compiler/codegen/visit_closuredef_stmt.go` —
  `visitClosureDef`, the closure-def statement visitor.
  Pushes an outer `compileUnit` at `outerDefLine`; pushes an
  inner `compileUnit` at `innerDefLine` linked to the outer.
  Emits `COPY_FREE_VARS 1` at `Loc{}` (zero — the encoder's
  `IsZero` branch produces the NO_INFO entry the classifier
  hand-built via `entryHeader(codeNoInfo, 1)`), `RESUME 0` at
  `Loc{innerDefLine, innerDefLine, 0, 0}` (SHORT_FORM
  `[0x80, 0x00]`), `LOAD_DEREF 0` at the body's Name
  expression (ONE_LINE_*), `RETURN_VALUE 0` at the Return
  statement (SHORT_FORM), adds `nil` to the inner's Consts so
  `co_consts[0] = None`, then pops through `popChildUnit` with
  `ArgCount=0`, `Flags=CO_OPTIMIZED|CO_NEWLOCALS|CO_NESTED`
  (`0x13`), `LocalsPlusKinds=[LocalsKindFree]`,
  `QualName=outerName + ".<locals>." + innerName`.
  In the outer, emits `MAKE_CELL 0` at `Loc{}` (NO_INFO),
  `RESUME 0` at `Loc{outerDefLine, outerDefLine, 0, 0}`, then
  the 6-instruction LONG-merge run (`LOAD_FAST_BORROW 0` +
  `BUILD_TUPLE 1` + `LOAD_CONST <innerCode>` +
  `MAKE_FUNCTION 0` + `SET_FUNCTION_ATTRIBUTE 8` +
  `STORE_FAST 1`) all sharing one Loc spanning
  innerDefLine..innerRetLine, then `LOAD_FAST_BORROW 1`
  (loading the inner func object) and `RETURN_VALUE 0`.
  Pops the outer through `popChildUnit` with `ArgCount=1`,
  `Flags=CO_OPTIMIZED|CO_NEWLOCALS` (`0x03`),
  `LocalsPlusKinds=[LocalsKindArgCell, LocalsKindLocal]`. The
  outer's `co_consts` is `[innerCode]` — no trailing `None`
  because every path returns explicitly via `return g`. In the
  module, adds `outerCode` as `co_consts[0]`, adds the outer
  function name as `co_names[0]`, and emits `LOAD_CONST
  outerCode + MAKE_FUNCTION + STORE_NAME` at moduleOuterLoc;
  visitModule's trailing `LOAD_CONST None + RETURN_VALUE`
  shares the same Loc so the 5-CU run merges into one LONG
  line-table entry byte-identical to
  `bytecode.FuncDefModuleLineTable`.

## Changed

- `compiler/codegen/visit_stmt.go::visitStmt` — the
  `*ast.FunctionDef` arm now tries `visitClosureDef` first; on
  `ErrNotImplemented` it falls through to `visitFunctionDef`.
  The two visitors' AST predicates are disjoint
  (`len(Body) == 2` vs `len(Body) == 1`), so each visitor only
  sees its own shape.
- `compiler/codegen/codegen.go::Build` — drops the last
  remaining classifier dispatch arm (`classifyClosureDefModule`
  → `buildClosureDefModule`). The codegen-classifier surface is
  now empty: `Build` returns `ErrUnsupported` for every input.
  Package doc updated to reflect the empty surface and the
  funeral lifecycle (the stub stays alive until v0.7.26).

## Out of scope

- General function bodies — multi-stmt body with assigns +
  return + nested expressions (modFuncBodyExpr). v0.7.10.
- Multiple captured cells in a single closure.
- Inner functions with their own arguments.
- Multi-level closure nesting (closure-of-closure).
- Closures over names that are not the outer's first arg.
- Decorators / annotations / type-params / defaults /
  vararg / kwarg on either the outer or inner function.
- Symtable-driven cell/free promotion. The visitor hard-codes
  `LocalsKindArgCell` and `LocalsKindFree` based on AST shape;
  consulting `symtable.Scope.Symbols` for the cell/free
  decision lands at v0.7.10/v0.7.13 when the optimizer pipeline
  starts needing it.
- `STORE_DEREF`, `LOAD_FROM_DICT_OR_DEREF`. The roadmap
  mentions these alongside `LOAD_DEREF` and `MAKE_CELL`; the
  v0.7.9 surface only ever reads the captured cell (one
  `LOAD_DEREF`). Stores through cells (e.g.
  `def f(x): def g(): nonlocal x; x = 1`) and the `__class__`
  cell pathway land later when the visitor recognises wider
  closure shapes.
- gopapy parser changes.
