# v0.3.6 — is-None conditions in function bodies (2026-04-28)

Added support for `if x is None:` and `if x is not None:` conditions inside
function bodies, for both if-return and if-assign patterns.

## What changed

`compileFuncBodyExpr` now handles two new condition forms:

- `if x is None: return expr` — emits `LOAD_FAST_BORROW x` +
  `POP_JUMP_IF_NOT_NONE` + 1 cache word + `NOT_TAKEN`, then the then-branch.
- `if x is not None: return expr` — same structure with `POP_JUMP_IF_NONE`.

CPython does not use `COMPARE_OP` for these conditions; it uses dedicated
one-operand jump opcodes (`POP_JUMP_IF_NOT_NONE = 102`,
`POP_JUMP_IF_NONE = 101`), each with 1 cache word.

A related quirk: functions that use `is None` / `is not None` conditions do
not store integer return values in `co_consts`. CPython emits `LOAD_SMALL_INT`
directly and leaves `co_consts` as `(None,)`. The new `funcState.noneCheckFunc`
flag tracks this and skips the `co_consts` entry in `loadConst`.

The classifier was tightened to only accept `Is`/`IsNot` operators when the
comparator is the `None` constant; other identity checks (`is True`, `is 5`)
are deferred.

## Fixtures

- `191_funcbody_if_is_none_return.py`: `if x is None: return 0; return x`
- `192_funcbody_if_is_not_none_return.py`: `if x is not None: return x; return 0`
