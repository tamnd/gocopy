# v0.7.4 — visit_Compare + visit_BoolOp + visit_IfExp + first optimizer passes

Date: 2026-04-29
Spec: notes/Spec/1500/1546_gocopy_v074_visit_compare_boolop_ifexp.md
Architecture: notes/Spec/1500/1541_gocopy_v07_architecture.md
Roadmap: notes/Spec/1500/1542_gocopy_v07x_roadmap.md
CPython parity: Python/codegen.c (3.14.0): `codegen_visit_expr_compare`,
`codegen_visit_expr_boolop`, `codegen_visit_expr_ifexp`;
Python/flowgraph.c (3.14.0): `inline_small_or_no_lineno_blocks`,
`resolve_unconditional_jumps`, `_PyAssemble_ResolveJumps`.

The fourth v0.7.x feature release. v0.7.2 promoted the assignment
shapes; v0.7.4 promotes the three branching-RHS shapes the v0.6
classifier owned: `modCmpAssign` (single Compare with one comparator),
`modBoolOp` (two-operand `and` / `or`), and `modTernary` (three-Name
`<a> if <c> else <b>`). v0.7.3 was intentionally skipped — its work
was absorbed into v0.7.2.

These are the first IR consumers of the label primitives
(`ir.Seq.AllocLabel`, `ir.Seq.BindLabel`, `ir.Block.AddJump`) that have
existed since v0.6.3 with zero callers. To keep emit clean and let the
visitor grow, this release also lands the first two real flowgraph
optimizer passes — `inline_small_exit_blocks` and `resolve_jumps` —
and wires them into `optimize.Run`. The visitor emits label-driven
multi-block IR, the optimizer transforms it to byte-equivalent linear
IR, and the assembler keeps producing the exact same code object as
the v0.6 classifier did. The 246/246 oracle stays byte-identical and
the visitor's parity count climbs from 101/246 to 114/246.

## Added

- `compiler/optimize/inline_small_exit_blocks.go` — first real
  flowgraph pass. Walks every block; for blocks that end in
  `RETURN_VALUE`, have at most six instructions, and are reachable by
  a Label, finds predecessors that flow in via fall-through or an
  unconditional `JUMP_FORWARD`. If every predecessor is unconditional,
  duplicates the exit-block tail into each predecessor (dropping the
  consumed `JUMP_FORWARD`) and unbinds the label. Mirrors
  `Python/flowgraph.c::inline_small_or_no_lineno_blocks`. Used by the
  v0.7.4 IfExp shape; the BoolOp shape's merge survives because its
  predecessor flows in via a conditional `POP_JUMP_IF_FALSE` /
  `POP_JUMP_IF_TRUE`.
- `compiler/optimize/resolve_jumps.go` — second real flowgraph pass.
  Computes the per-block start code-unit offset, builds a
  `LabelID → byte offset` map, and rewrites every jump's `Arg` from
  the symbolic LabelID to the byte distance the assembler needs. Then
  linearises the seq down to a single block. Panics on a
  backward-pointing forward jump — a tripwire for v0.7.6 when
  `JUMP_BACKWARD` (loop) shapes start emitting through the visitor.
  Mirrors `Python/flowgraph.c::resolve_unconditional_jumps` and
  `_PyAssemble_ResolveJumps`.
- `compiler/codegen/visit_compare_stmt.go` — `emitCompareValue`
  lowers a single-Compare RHS (`<left> <cmp> <right>`, both Names) to
  `LOAD_NAME` / `LOAD_NAME` / `(COMPARE_OP | IS_OP | CONTAINS_OP)`.
  `emitBoolOpValue` lowers two-operand `and` / `or` to a 3-block
  shape: entry with `LOAD_NAME left` / `COPY 1` / `TO_BOOL 0` plus a
  `POP_JUMP_IF_FALSE` (And) or `POP_JUMP_IF_TRUE` (Or) into the merge
  label; right-branch with `NOT_TAKEN 0` / `POP_TOP 0` / `LOAD_NAME
  right`; merge bound to the end label. `emitIfExpValue` lowers
  `<true> if <cond> else <false>` (three Names) to a 4-block shape:
  entry with `LOAD_NAME cond` / `TO_BOOL 0` / `POP_JUMP_IF_FALSE` into
  falseLabel; true-branch with `NOT_TAKEN 0` / `LOAD_NAME true` plus a
  `JUMP_FORWARD` into endLabel; false-branch bound to falseLabel with
  `LOAD_NAME false`; merge bound to endLabel.
- `compiler/codegen/op_helpers.go::cmpOpFromAstOp` — maps the ten AST
  comparison op strings (Lt, LtE, Eq, NotEq, Gt, GtE, Is, IsNot, In,
  NotIn) to the `(Opcode, oparg)` pair CPython 3.14 emits. Mirrors
  `compiler/classify_ast.go::cmpOpFromOp` byte-for-byte.

## Changed

- `compiler/optimize/optimize.go::Run` — was the identity. Now runs
  `inlineSmallExitBlocks` followed by `resolveJumps`. Pure
  transformation on the InstrSeq; the visitor still owns IR
  generation and the assembler still consumes the resulting
  single-block flat seq.
- `compiler/codegen/visit_assign_stmt.go::emitAssignValue` — the
  `chained == false` switch gains three new arms: `*ast.Compare` →
  `emitCompareValue`, `*ast.BoolOp` → `emitBoolOpValue`, `*ast.IfExp`
  → `emitIfExpValue`. The `*ast.Constant` / `*ast.BinOp` /
  `*ast.UnaryOp` arms are unchanged.
- `compiler/codegen/generate.go::visitModule` — the trailing
  `LOAD_CONST None` / `RETURN_VALUE` now appends to
  `u.currentBlock()` (renamed `tailBlock`) rather than the entry
  block. The visitor's branching shapes leave a fresh merge block at
  the end of the seq, so the terminator must follow the current
  cursor.

## Removed

- `compiler/codegen/visit_cmp_assign.go`,
  `compiler/codegen/visit_boolop_assign.go`,
  `compiler/codegen/visit_ternary_assign.go` — the v0.6.14 / v0.6.15 /
  v0.6.16 classifier files for the three promoted shapes.
- `compiler/codegen/codegen.go::Build` — the
  `classifyCmpAssignModule`, `classifyBoolOpAssignModule`, and
  `classifyTernaryAssignModule` dispatch arms, plus the package
  doc-comment paragraphs that described those shapes.
- `compiler/codegen/codegen_test.go` — the
  `TestBuildCmpAssign*` / `TestBuildBoolOpAssign*` /
  `TestBuildTernaryAssign*` families that called `Build` directly to
  assert specific bytecode for the deleted classifier shapes. Oracle
  parity, visitor parity, and the surrounding tests still cover the
  build-side behavior.

## Out of scope

- Any AST node beyond what v0.7.2 already covered plus `Compare`,
  `BoolOp`, and `IfExp`. Collection / subscript / attribute / call /
  GenExp / If / While / For / FunctionDef shapes still ship through
  the v0.6 classifier and land at v0.7.5+.
- Multi-comparator chained Compare (`a < b < c`) and N-ary BoolOp
  (`a and b and c`); the visitor restricts to single-op Compare and
  two-operand BoolOp the same way the classifier did.
- The full CPython optimizer chain. v0.7.4 only ships
  `inline_small_exit_blocks` + `resolve_jumps`. The trailing-NOP
  fold, dead-code elimination, push/pop folding, and the
  full `optimize_cfg` pass land at v0.7.13+.
- Backward jumps (`JUMP_BACKWARD`). The resolveJumps pass panics on
  one as a tripwire for the v0.7.6 loop shapes.
- gopapy parser changes.
