# v0.1.17 - constants in function body expressions (2026-04-28)

Extends the function body compiler to allow constant values -- non-negative
integers (0-255), None, True, and False -- as operands and as return values.

## Supported patterns (new)

```python
def f(a):
    return a + 1

def f(a):
    return 1 + a

def f():
    return 42

def f(a):
    x = a + 2
    return x

def f():
    return None

def f():
    return True
```

## Architecture

`isFuncBodyExpr` in classify_ast.go now accepts `*parser2.Constant` nodes
for integer (0-255), None, True, and False values. `extractFuncBodyExpr` now
also accepts functions with zero parameters (the earlier `len(args.Args) == 0`
guard is removed).

`funcState` gains a `consts []any` slice that records constants in
first-occurrence order. `constIndex(v)` appends a new value or returns the
existing index. `buildConsts()` returns the slice, falling back to `(None,)`
when no constants appear.

`loadConst` in func_body.go emits LOAD_SMALL_INT for non-negative integers and
LOAD_CONST (index from co_consts) for None, True, and False, without emitting
a linetable entry.

`walkExpr` has a new `*parser2.Constant` case: calls `loadConst` then emits
a single linetable entry. This covers constants that appear inside BinOp
operands or assignment expressions.

Return-statement handling gets a special path for top-level constant returns:
when `st.expr` is a `*parser2.Constant`, the load instruction and RETURN_VALUE
share one combined 2-CU linetable entry covering the constant token range.
This matches CPython's behavior (verified against `python3.14 -m py_compile`).

## Opcodes

- `LOAD_SMALL_INT` (opcode 94): already in bytecode/opcode.go
- `LOAD_CONST` (opcode 82): already present

## New code

- `compiler/classify_ast.go`: `isFuncBodyExpr` Constant case; removed
  0-param guard in `extractFuncBodyExpr`
- `compiler/func_body.go`: `funcState.consts`; `constIndex`, `buildConsts`,
  `loadConst` methods; Constant case in `walkExpr`; combined-return path
- 6 new fixtures: 150-155

## Out of scope for this version

- Negative integer constants (CPython uses LOAD_CONST with a folded value;
  the gopapy AST presents them as UnaryOp(USub, Constant(n)))
- Float, complex, bytes, string constants in function bodies
- Constants as call arguments in function bodies
