# v0.7.10.2 — codegen_if 1:1 port

Date: 2026-04-30
Spec: notes/Spec/1500/1560_gocopy_v07102_codegen_if.md
Roadmap: notes/Spec/1500/1558_gocopy_v0710x_release_roadmap.md

v0.7.10.2 collapses the v0.7.10 per-shape If helpers
(`validateFuncBodyIf`, `validateFuncBodyTerminatingIf`,
`emitFuncBodyIf`, `emitFuncBodyTerminatingIf`) into a single
recursive `codegenIf` mirroring CPython 3.14
`Python/codegen.c::codegen_if`. The Compare-arm test
compilation is split out as `codegenJumpIf`, mirroring
`codegen_jump_if`. Validation collapses into one
`validateIfStmt(isLast)` predicate.

## Added

- `compiler/codegen/visit_if.go` — three new functions:
  - `codegenIf` — single recursive entry point for `*ast.If`
    inside a function body. Returns `(lastEndCol, lastEndLine,
    terminating, err)` so the caller can use the deepest
    Return's span for the outer LOAD_CONST/MAKE_FUNCTION/
    STORE_NAME Loc when the chain fully replaces the trailing
    function-body Return.
  - `codegenJumpIf` — Compare-arm test compilation, returns
    the false-branch label and the condLoc the caller binds
    to the NOT_TAKEN at the head of the then-block.
  - `validateIfStmt(ifs, isLast)` — recursive shape predicate
    consolidating both non-last and terminating shape checks.
- `// MIRRORS:` headers on `codegenIf` (Python/codegen.c:2042
  `codegen_if`) and `codegenJumpIf` (Python/codegen.c:1884
  `codegen_jump_if`) — first 1:1 port headers in the
  v0.7.10.x series.

## Changed

- `compiler/codegen/visit_funcbody_stmt.go::visitFuncBodyDef` —
  now calls `validateIfStmt(last, true)` for the
  terminating-If validation slot, `validateIfStmt(st, false)`
  for the non-last validation slot, and `codegenIf` for both
  emit slots. Removes per-shape dispatch.

## Removed

- `validateFuncBodyIf` (replaced by `validateIfStmt(_, false)`).
- `validateFuncBodyTerminatingIf` (replaced by
  `validateIfStmt(_, true)`).
- `emitFuncBodyIf` (replaced by `codegenIf`).
- `emitFuncBodyTerminatingIf` (replaced by `codegenIf`).

`visit_funcbody_stmt.go` shrinks by 215 LOC.

## Deviation from CPython

CPython's `codegen_jump_if` for the Compare arm emits
`<test>; TO_BOOL; POP_JUMP_IF_FALSE next` and a flowgraph
peephole later fuses TO_BOOL into the preceding COMPARE_OP by
setting bit 4 of its oparg. gocopy does not yet have that
fusion pass (it lands in v0.7.10.6 with `optimize_basic_block`);
`codegenJumpIf` therefore pre-fuses by emitting
`COMPARE_OP (cmp<<5)|mask|16; POP_JUMP_IF_FALSE next` directly.
Final bytecode is identical post-fusion. The deviation is
documented in `visit_if.go`'s package-level header comment and
collapses in v0.7.10.6.
