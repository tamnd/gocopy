# v0.1.18 - comparison operators in function bodies (2026-04-28)

Adds support for `<`, `<=`, `==`, `!=`, `>`, and `>=` in function body
expressions. The result of a comparison can be returned or used as an
operand in further expressions.

## Supported patterns (new)

```python
def f(a, b):
    return a < b

def f(a, b):
    return a == b

def f(a, b):
    return a != b
```

## Architecture

`isFuncBodyExpr` in classify_ast.go now accepts `*parser2.Compare` nodes with
a single operator that maps to COMPARE_OP. Chained comparisons (`a < b < c`)
are not yet supported.

`walkExpr` in func_body.go gains a `*parser2.Compare` case. The same
LOAD_FAST_BORROW_LOAD_FAST_BORROW optimisation that applies to BinOp also
applies here: when both operands are Name nodes with slots 0-15, one LFLBLFLB
instruction loads both in a single bytecode word before COMPARE_OP.

COMPARE_OP has one inline-cache word (2 CU total for the instruction + cache),
compared to BINARY_OP's five cache words (6 CU total). The linetable entry for
COMPARE_OP therefore covers 2 CU at the full operand span.
