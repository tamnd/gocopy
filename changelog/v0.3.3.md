# v0.3.3 - ternary return in function bodies (2026-04-28)

## What changed

Added support for `return val if cond else other` (ternary expression as
the return value) in function bodies compiled by `compileFuncBodyExpr`.

## Pattern

```python
def f(a, b):
    return a if a > b else b
```

## Bytecode structure

CPython evaluates the condition first, then the true branch, then the
false branch. The true branch ends with RETURN_VALUE, and the false
branch falls through to its own RETURN_VALUE. There is no JUMP_FORWARD.

```
evaluate condition (LFLBLFLB + COMPARE_OP(cond) + POP_JUMP_IF_FALSE + NOT_TAKEN)
evaluate true value (LOAD_FAST_BORROW / walkExpr)
RETURN_VALUE
[L1: false branch]
evaluate false value (LOAD_FAST for Name; walkExpr otherwise)
RETURN_VALUE
```

## LOAD_FAST vs LOAD_FAST_BORROW

CPython uses LOAD_FAST (opcode 84, "moving" load) for a simple Name
in the false branch, but LOAD_FAST_BORROW for the true branch and for
any non-Name expression in the false branch. This matches what gocopy
now emits.

## Linetable

All entries are SAME_LINE (no new-statement markers after the first)
because the ternary is a single expression. The false value's end column
serves as ternEnd for both RETURN_VALUE entries.
