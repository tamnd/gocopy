# v0.3.1 - elif chains in function bodies (2026-04-28)

## What changed

Extended the `isIfReturn` classifier to handle `elif` chains.

## Pattern

```python
def f(a, b):
    if a < b:
        return a
    elif a > b:
        return b
    return 0
```

CPython lowers `elif` to an `If` node inside the `orelse` of the parent
`If`. The compiler now flattens this recursively into consecutive
`isIfReturn` statements, each generating an independent
COMPARE_OP+POP_JUMP_IF_FALSE+NOT_TAKEN block with a forward jump.
