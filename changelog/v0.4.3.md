# v0.4.3 — large string list literals (n ≥ 31)

## What's new

Handle module-level assignments where the RHS is a list of 31 or more
plain-ASCII string constants with each element on its own line and the
closing `]` on its own line. CPython uses a different code sequence for
large lists (n ≥ 31):

- **List n ≤ 30**: `BUILD_LIST 0 + LOAD_CONST tuple + LIST_EXTEND 1` (v0.4.2)
- **List n ≥ 31**: `BUILD_LIST 0 + N×(LOAD_CONST elem_i + LIST_APPEND 1) + STORE_NAME`

The linetable for the n ≥ 31 case alternates between the element's line
(one LONG entry per LOAD_CONST) and the opening-bracket line (one LONG
entry per LIST_APPEND, spanning back to the `[` line). This mirrors
CPython's `kwlist` pattern in `keyword.py`.

co_consts layout: `(elem0, elem1, ..., elemN-1, None)` — elements in
order, None appended last (unlike the n ≤ 30 case which uses a tuple).

## New fixtures (221–223)

- 221 `x = ['False', ..., 'yield']` — 35-element keyword list
- 222 `x = ['a', ..., 'ee']` — 31-element list (boundary case)
- 223 `kwlist = ['False', ..., 'yield']` — same as 221 with longer name
