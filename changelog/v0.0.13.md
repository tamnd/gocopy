## v0.0.13

`gocopy compile` accepts N >= 2 sequential `name = literal` assignments
at module level, e.g. `x = 1\ny = 2\n`. Any combination of the literal
types already supported in v0.0.7-v0.0.12 (None, True, False, `...`,
string, bytes, integer, float, complex, negative) is accepted.

The consts tuple follows CPython's exact layout:

- The first assignment's value is always present in consts (as a
  phantom slot for LOAD_SMALL_INT, or the LOAD_CONST source otherwise).
- Subsequent small integers (0..255) are NOT added to consts; they are
  loaded via LOAD_SMALL_INT directly.
- Other values (large ints, floats, bools, strings, bytes, complex,
  None) are added to consts in encounter order, deduplicated.
- None is appended if not already present.
- The negated halves of negative literals are appended last.

Names are also deduplicated (if the same name is assigned twice, it
reuses the same slot in co_names).

### Added

- `modMultiAssign` classifier kind; `asgns []asgn` field on
  `classification` and `line int` field on `asgn`.
- Classifier loop allows consecutive assignments after the first.
- `compileMultiAssign` in `compiler/compiler.go`: builds consts/names/
  bytecode/linetable for N >= 2 assignments.
- `bytecode.AssignInfo` struct and `MultiAssignLineTable` in
  `bytecode/assign.go`.
- Seven new fixtures: `061_multi_assign_two_ints.py` through
  `067_multi_assign_small_then_neg.py`.
- Five `TestMultiAssign` sub-cases in `compiler/compiler_test.go`.

### Out of scope (deferred)

- Chained assignment (`x = y = 1`).
- Augmented assignment (`x += 1`).
- Annotated assignment (`x: int = 1`).
- Wiring gopapy as the parser.
