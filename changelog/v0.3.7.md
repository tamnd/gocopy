# v0.3.7 — float constants in function bodies (2026-04-28)

Added support for float literal constants inside function bodies, enabling
patterns like `if x < 0.0: x = 0.0` and `return 0.5`.

## What changed

`isFuncBodyExpr` (classify_ast.go): accepts `"float"` constants and lifts the
upper bound on `"int"` constants (previously 0–255 only; now any non-negative
integer is accepted — large values use LOAD_CONST instead of LOAD_SMALL_INT).

`loadConst` (func_body.go): new `"float"` case emits `LOAD_CONST idx` where
`idx` is the `constIndex(float64)` entry. Floats are always stored in
co_consts (no LOAD_SMALL_FLOAT exists in CPython 3.14).

`exprEndCol` (func_body.go): new `"float"` case calls `scanTokenEnd` to
compute the source end column from the literal text.

New helper `scanTokenEnd` (func_body.go): scans forward from `startCol` on
the current statement's source line, consuming numeric literal characters
(digits, `.`, `_`, `e`, `E`, `+`, `-`). Used by both `loadConst` and
`exprEndCol` to determine the end column of a float token.

## Fixtures

- `193_funcbody_return_float.py`: `def f(x): return 0.0`
- `194_funcbody_if_float_assign.py`: `if x < 0.0: x = 0.0` + return
- `195_funcbody_float_clamp.py`: double-sided clamp `[0.0, 1.0]`
- `196_funcbody_return_large_int.py`: `def f(x): return 256` (LOAD_CONST, not LOAD_SMALL_INT)
