# v0.2.3 - tuple expressions in function bodies (2026-04-28)

Adds tuple literals as valid function-body expressions, covering
`return a, b` and `return a, b, c` patterns.

## Supported patterns (new)

```python
def f(a, b):
    return a, b

def f(a, b, c):
    return a, b, c
```

## Architecture

`isFuncBodyExpr` gains a `*parser2.Tuple` case. Any tuple of one or
more elements qualifies when every element passes `isFuncBodyExpr`.

`walkExpr` gains a `*parser2.Tuple` case:

1. LFLBLFLB optimisation applies when the first two elements are Name
   nodes with slots 0-15 (same as BinOp/Compare).
2. Remaining elements are walked individually with walkExpr.
3. BUILD_TUPLE oparg=nElts follows; its linetable entry spans from the
   first element's start column to the last element's end column.

Depth tracking: each element added to the stack is tracked as it loads
(`trackDepth(i+1)` for the i-th element); LFLBLFLB path calls
`trackDepth(2)`. The final peak equals `nElts`.
