# v0.3.4 - LOAD_FAST for name assignments in function bodies (2026-04-28)

## What changed

Fixed a bug where simple name-to-name assignments (`b = a`) in function
bodies emitted LOAD_FAST_BORROW instead of LOAD_FAST. CPython uses the
"moving" load (LOAD_FAST, opcode 84) when the entire right-hand side is
a bare Name; borrowing loads (LOAD_FAST_BORROW) apply only when the name
appears as a sub-expression inside a larger expression.
