# v0.1.19 - augmented assignment in function bodies (2026-04-28)

Adds `+=`, `-=`, `*=`, `/=`, `//=`, `%=`, `**=`, `&=`, `|=`, `^=`, `<<=`,
and `>>=` inside function bodies. The target must already be a known local or
parameter; the RHS can be any expression accepted by isFuncBodyExpr.

## Supported patterns (new)

```python
def f(a):
    a += 1
    return a
```

## Architecture

`fbStmt` in classify.go gains two new fields: `isAugAssign bool` and
`augOp byte` (one of the NbInplace* constants). `extractFuncBodyExpr` in
classify_ast.go now handles `*parser2.AugAssign` nodes: the target must be a
Name already in `knownNames` (so you cannot augmented-assign to a fresh name),
and `augOpargFromOp` maps the operator string to the inplace oparg.

The compilation path in compileFuncBodyExpr handles `isAugAssign` statements
separately from plain assignments. When the RHS is a Name node and both target
and RHS slots fit in a nibble, LOAD_FAST_BORROW_LOAD_FAST_BORROW is emitted
for the two loads. Otherwise LOAD_FAST_BORROW(target) precedes a walkExpr call
for the RHS. In both cases BINARY_OP(NbInplace*) + five cache words follows,
then STORE_FAST(target_slot).

The linetable pattern for augmented assignment mirrors a regular binary
assignment: the target name column anchors the first-on-line entry and the
BINARY_OP entry spans the whole statement. STORE_FAST gets a one-CU entry for
the target name.
