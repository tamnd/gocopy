# v0.1.8 - function call assignments (2026-04-27)

Adds support for module-level assignments where the right-hand side is a
function call. The function and all positional arguments must be plain names.
No keyword arguments yet.

## Supported patterns

```python
x = f()         # call with no arguments
x = f(a)        # call with one positional argument
x = f(a, b)     # call with two positional arguments
```

CPython 3.14 uses a two-opcode call sequence: `PUSH_NULL` followed by
`LOAD_NAME` for the function, then `LOAD_NAME` for each argument, then
`CALL <argc>` (3 cache words). This differs from earlier CPython versions
that used `CALL_FUNCTION`.

## Opcodes added

| Opcode | Number | Cache words |
|--------|--------|-------------|
| PUSH_NULL | 33 | 0 |
| CALL | 52 | 3 |

## New code

- `bytecode/call.go`: `CallArg`, `CallAssignBytecode`, `CallAssignLineTable`.
- `compiler/classify.go`: `modCallAssign`, `stmtCallAssign`, `callAssign`.
- `compiler/classify_ast.go`: handles `*parser2.Call` with Name function and
  Name-only positional arguments (no keywords, no starargs).
- `compiler/compiler.go`: `compileCallAssign`.
- 3 new fixtures: 128-130.

## Out of scope for this version

- Keyword arguments (`f(a=b)`).
- Star-unpacking arguments (`f(*args)`).
- Calls without an assignment (pure expression statement).
- Method calls (`obj.method()`).
