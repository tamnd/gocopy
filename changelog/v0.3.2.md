# v0.3.2 - if-else-return in function bodies (2026-04-28)

## What changed

Extended the if-return classifier to handle `if/else` where both branches
end with `return`. The else branch becomes the fallthrough return.

## Pattern

```python
def f(a, b):
    if a < b:
        return a
    else:
        return b
```

CPython compiles this identically to `if a < b: return a; return b`.
The `else:` keyword is not present in the bytecode; the else branch's
return is the fallthrough instruction after the condition block.

Also supported: `if/elif/.../else` chains where every branch ends with
`return`.
