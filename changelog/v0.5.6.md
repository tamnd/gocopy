# v0.5.6 — implicit None return, scanEndCol rsc fix, int-const guard (2026-04-28)

## What changed

### Implicit None return (hsv_to_rgb)

Functions whose last statement is a bare `if cond: return expr` with no
`else` fall off the end and CPython emits `LOAD_CONST None + RETURN_VALUE`
attributed to the last `if`-condition's line and column span.  Three bugs
blocked byte-identical output for `hsv_to_rgb`:

1. **linetable line**: the implicit return was emitted with `emitSame`
   (inheriting the previous instruction's line — the `return` inside the
   `if` body), but CPython attributes it to the `if`-condition line.
   Fixed by switching to `emit` after `newStmt(lastSt.line)`.

2. **module bodyEndLine**: the outer module linetable records the function
   body's last source line.  For implicit-None functions the correct end
   line is the last *explicit* return's line (`thenLine`), not the `if`
   condition line.  Added `if g.hasImplicitNoneReturn && thenLine > bodyEndLine`
   guard.

3. **integer constant in co_consts**: CPython only places the *first*
   integer in `co_consts` when it is also the very *first* constant in the
   function (no float, string, or other constant precedes it).  The old
   guard (`!intConstSeen && !noneCheckFunc`) was missing this check, causing
   `6` to appear in `co_consts` for `hsv_to_rgb` (floats come first there).
   Added `len(fs.consts) == 0` to the condition.

### Right-child scanEndCol guard (fixtures 231, 233, 237, 238, 239)

`walkExpr` for a `BinOp` extends `rec` to include a closing `)` when the
right child is a parenthesized `BinOp`.  The v0.5.5 guard used
`exprCol(n.Right)` (the operator's column) and checked `line[rightCol] ==
'('`, which is wrong: gopapy sets `P.Col` to the *operator character*, not
the opening `(`.

The correct signal is `rsc` — the start column returned by
`walkExpr(n.Right)` — which is the leftmost *token* column.  If
`scanBackOpen(rsc) < rsc` then `(` immediately precedes the first token and
the whole right child is parenthesized.  Applied in both `walkExpr` and
`walkExprSkipFirstLocal`.

## New fixture

- `241_hsv_to_rgb.py` — CPython's `colorsys.hsv_to_rgb`; 6 `if`-return
