# v0.7.2 — visit_Assign + visit_BinOp + visit_UnaryOp + visit_AugAssign + int-fold

Date: 2026-04-29
Spec: notes/Spec/1500/1545_gocopy_v072_visit_assign_binop_unary.md
Architecture: notes/Spec/1500/1541_gocopy_v07_architecture.md
Roadmap: notes/Spec/1500/1542_gocopy_v07x_roadmap.md
CPython parity: Python/codegen.c (3.14.0): `codegen_visit_stmt_assign`,
`codegen_visit_stmt_augassign`, `codegen_visit_expr_binop`,
`codegen_visit_expr_unaryop`; Python/compile.c constant-folding
pass.

The second v0.7.x feature release. v0.7.1 promoted the no-op shapes
(empty / pass / docstring); v0.7.2 promotes every assignment shape
the v0.6 classifier owned: `modAssign` (with the `negLiteral` and
`foldedBinOp` sub-shapes), `modMultiAssign`, `modChainedAssign`,
`modAugAssign`, `modBinOpAssign`, and `modUnaryAssign`. The visitor
folds `BinOp(Const,op,Const)` and `UnaryOp(USub,numeric)` at emit
time using a phantom-slot + deferred-patch scheme that matches
CPython 3.14's post-CFG fold byte-for-byte. The 246/246 oracle stays
byte-identical and the visitor's parity count climbs from 28/246 to
101/246.

## Added

- `compiler/codegen/visit_assign_stmt.go` — `visitAssignStmt`
  lowers single-target and chained `<t0> = ... = <tN-1> = <value>`
  to IR, with the RHS dispatched through `emitAssignValue`.
  `emitAssignValue` accepts `*ast.Constant`, `*ast.BinOp`, and
  `*ast.UnaryOp` (BinOp/UnaryOp single-target only). `emitBinOpValue`
  folds `BinOp(Constant,op,Constant)` for foldable numeric operands
  via `foldBinOp` and emits `LOAD_NAME`/`LOAD_NAME`/`BINARY_OP` for
  `BinOp(Name,op,Name)`. `emitUnaryOpValue` folds
  `UnaryOp(USub, int|float Constant)` to a negative literal and
  emits `LOAD_NAME`/`UNARY_NEGATIVE`/`UNARY_INVERT` (or
  `LOAD_NAME`/`TO_BOOL`/`UNARY_NOT`) for the Name-operand cases.
  `visitAugAssignStmt` lowers `<name> <op>= <int>` for the 12
  in-place BinOp operators with a non-negative int RHS.
- `compiler/codegen/fold.go` — `numConstVal` and `foldBinOp` (the
  emit-time fold predicate). Lifted verbatim from
  `compiler/classify_ast.go` so the visitor can short-circuit
  `BinOp(Const,Const)` into a single LOAD opcode the same way the
  v0.6 classifier did. Retires when v0.7.16's optimize_cfg pass
  lands.
- `compileUnit.phantomDone bool` and
  `compileUnit.deferredPatches []deferredPatch` — track the
  first-const-emit phantom slot and queue LOAD_CONST instructions
  whose final `Arg` must be rewritten in `finalizeDeferred` once
  None has consumed its post-phantom slot. `emitDeferredConstLoad`
  appends a placeholder LOAD_CONST and registers a patch.
- `compiler/codegen/op_helpers.go` — moves `constantToValue`,
  `binOpargFromOp`, and `augOpargFromOp` (the v0.6 classifier's
  shared op helpers) into a single visitor-pipeline-owned file. The
  gen-expr classifier and the new visitor both call into them.
- `checkBodyShape(body)` in `compiler/codegen/generate.go` —
  rejects body shapes the visitor does not own yet (a no-op
  statement before a real one; a bare AugAssign without a preceding
  Assign), returning `ErrNotImplemented` so `runVisitorShadow`
  falls back to the classifier path.

## Changed

- `compiler/codegen/visit_const.go::visitConstant` —
  `emitConstLoad` now applies the phantom-slot rule (claim
  `co_consts[0]` for the first const-emit even when `LOAD_SMALL_INT`
  replaces the LOAD_CONST) and emits `LOAD_SMALL_INT` for int64 in
  `0..255`. Mirrors the LOAD_SMALL_INT specialization in
  Python/compile.c.
- `compiler/codegen/visit_stmt.go::visitStmt` — adds `*ast.Assign`
  and `*ast.AugAssign` arms (dispatching to the new visitor file)
  and threads `isLast` so `*ast.Pass` and `*ast.ExprStmt` no-op
  tails still merge into the trailing `LOAD_CONST None` /
  `RETURN_VALUE` Loc the way `visit_module.go` did at v0.7.1.
- `compiler/codegen/generate.go::visitModule` — iterates the body
  through `visitStmt` once `checkBodyShape` accepts it; calls
  `finalizeDeferred()` after `addConst(nil)` so None occupies
  `co_consts[1]` (post-phantom) and folded results land at
  `co_consts[2..]`.
- `compiler/codegen/visit_gen_expr.go::buildGenExprModule` — the
  StackSize floor drops from `max(2, walkerDepth)` to
  `max(1, walkerDepth)`. With the dedicated `unary_assign`
  classifier gone, single-name UnaryOp shapes (`x = -a`, `x = ~a`)
  fall through into gen-expr, and the previous floor was
  over-conservative for those trivial 1-slot cases.

## Removed

- `compiler/codegen/visit_assign.go`,
  `compiler/codegen/visit_multi_assign.go`,
  `compiler/codegen/visit_chained_assign.go`,
  `compiler/codegen/visit_aug_assign.go`,
  `compiler/codegen/visit_binop_assign.go`,
  `compiler/codegen/visit_unary_assign.go` — the v0.6 classifier
  files for the six promoted shapes. Build no longer dispatches
  them.
- `compiler/codegen/visit_module.go` — the `noOpStmt` type and
  `locOf` helper had no remaining consumers after the six classifier
  files retired.
- `compiler/codegen/codegen.go::Build` — the six dispatch arms
  (`classifyAssignModule`, `classifyMultiAssignModule`,
  `classifyChainedAssignModule`, `classifyAugAssignModule`,
  `classifyBinOpAssignModule`, `classifyUnaryAssignModule`) and
  the package doc-comment paragraphs that described them.
- `compiler/codegen/codegen_test.go` — the `TestBuildAssign*`,
  `TestBuildMultiAssign*`, `TestBuildChainedAssign*`,
  `TestBuildAugAssign*`, and the two `TestBuildUnaryAssignNot*`
  cases that called `Build` directly to assert specific bytecode
  for the deleted classifier shapes. Oracle parity, visitor parity,
  and the cmp/boolop/ternary/etc tests still cover the build-side
  behavior.

## Out of scope

- Any AST node beyond `Constant`, `Name`-Load, `Pass`, `ExprStmt`,
  `Assign`, `AugAssign`, `BinOp`, `UnaryOp`, and `Module`. `If`,
  `While`, `FunctionDef`, etc. land at v0.7.3+.
- `Name` Store / Del contexts (Store only via `STORE_NAME` from
  Assign / AugAssign). Standalone Name-Store land at v0.7.3+.
- Compare / BoolOp / IfExp / collection / subscript / attribute /
  call / GenExp shapes. Their classifier paths remain.
- A general optimizer pass (`optimize.Run` is still identity); the
  visitor emits the post-CFG-fold pattern directly for the
  promoted shapes.
- gopapy parser changes.
