## v0.0.9

`gocopy compile` accepts a non-negative integer literal on the
right-hand side of a leading `name = literal` assignment. Decimal,
hex (`0x`), octal (`0o`), and binary (`0b`) literals are all
recognised, with optional underscore digit separators. Values must
fit in `int32` range [0, 2^31-1]; larger integers are deferred.

CPython's lowering splits on the value:

- **0..255**  -  `LOAD_SMALL_INT <val>` with the value embedded in the
  oparg. `consts = (int_val, None)`, `noneIdx = 1`.
- **256..2^31-1**  -  `LOAD_CONST 0` (the integer at const index 0).
  Same `consts = (int_val, None)`.

Both paths share `STORE_NAME <name>`, the same no-op tail shape
from v0.0.7, and the same `AssignLineTable` line-table builder
(the value's column range still comes from the scanner, not the
value itself).

Marshal emits `TYPE_INT` (0x69) with a 32-bit little-endian payload.
`FLAG_REF` follows CPython 3.14's immortal-object rule: integers
in the small-int cache range [−5, 256] are immortal and always get
`FLAG_REF`; larger integers get it only if they appear more than
once in the walk (which doesn't happen in the assignment case, so
they come out as plain `TYPE_INT`).

### Added

- `LOAD_SMALL_INT Opcode = 94` constant in `bytecode`.
- `bytecode.AssignSmallIntBytecode(val byte, tailStmts int)` for
  the 0..255 case.
- `marshal.emitObject` handles `int64` → `TYPE_INT` with the
  immortal-range FLAG_REF rule. `tupleKey` and `refCounter.tuple`
  updated to track `int64` consts.
- `parseIntLiteral` and `parseBaseLiteral` in `compiler/classify.go`:
  parse decimal/hex/oct/bin integer literals (with underscores) into
  `int64`, rejecting floats, complex, and values > MaxInt32.
- `tryParseAssign` tries `parseIntLiteral` before the
  string/bytes branch.
- `compiler.Compile` dispatches to `AssignSmallIntBytecode` for
  0..255 and `AssignBytecode(1, ...)` for 256+, both with
  `consts = (int_val, None)`.
- Eight new fixtures: `040_assign_int_zero.py` through
  `047_assign_int_hex_large.py`.

### Out of scope (deferred)

- Negative integer literals (unary minus not yet parsed).
- Integer values > 2^31-1 (need TYPE_LONG marshal path).
- Float and complex literals (need TYPE_BINARY_FLOAT / TYPE_COMPLEX).
- Docstring + assignment combo.
- Wiring gopapy as the parser; still waiting on a gopapy v1.0.0.
