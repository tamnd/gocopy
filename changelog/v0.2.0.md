# v0.2.0 - function calls in function bodies (2026-04-28)

Adds `return f(args...)` and `x = f(args...)` inside function bodies
where the callee is a global name (builtin or module-level function)
and every argument is a valid func-body expression.

## Supported patterns (new)

```python
def f(a):
    return len(a)

def f(a, b):
    return max(a, b)

def f():
    return int()

def f(a):
    return range(a, 10)

def f(a):
    b = len(a)
    return b
```

## Architecture

`isFuncBodyExpr` gains a `*parser2.Call` case. A call qualifies when
the function expression is a Name that is not a known local or param
(i.e., a global), there are no keyword arguments, and every positional
argument passes `isFuncBodyExpr`.

`walkExpr` in func_body.go gains a `*parser2.Call` case. The call emits:

1. LOAD_GLOBAL oparg = (name_index << 1) | 1 (the low bit requests the
   NULL push that CPython 3.14 merges into LOAD_GLOBAL); 4 cache words.
2. Arguments. If there are exactly two Name arguments whose slot
   numbers fit in 4 bits, LOAD_FAST_BORROW_LOAD_FAST_BORROW is used
   (the same LFLBLFLB optimisation as BinOp and Compare). Otherwise
   each argument is walked with walkExpr in order.
3. CALL oparg = nargs; 3 cache words.

The linetable pattern mirrors the module-level call pattern:
LOAD_GLOBAL + caches get one entry spanning the function name token;
each argument gets its own entry (or the LFLBLFLB pair gets one entry
for the first name); CALL + caches get one entry spanning from the
function name through the closing `)`. RETURN_VALUE gets a separate
entry.

`funcState` gains a `names []string` field with `nameIndex` and
`buildNames` helpers mirroring `constIndex`/`buildConsts`. The inner
code object's `co_names` is now wired to `fs.buildNames()` instead of
the hardcoded empty slice.

The `LOAD_GLOBAL` opcode (92) and its 4-word cache entry are added to
`bytecode/opcode.go`.

`scanCallEnd` is a new helper that scans the source line from a given
column forward to find the first `)`, returning col+1 past it. This
finds the closing paren of a call regardless of whether the last
argument's AST end column aligns with it.

## New code

- `bytecode/opcode.go`: LOAD_GLOBAL = 92, CacheSize[92] = 4
- `compiler/classify_ast.go`: *parser2.Call case in isFuncBodyExpr
- `compiler/func_body.go`: names field in funcState; nameIndex,
  buildNames, scanCallEnd methods; *parser2.Call case in walkExpr
The ambient-depth tracking for nested calls: before evaluating each arg,
maxDepth is temporarily saved and reset to zero. After the arg's walkExpr
runs, its internal peak (argPeak) is combined with the ambient depth
(2 + args already loaded) and fed back to trackDepth. This ensures
`str(len(a))` records co_stacksize=5, not 3.
