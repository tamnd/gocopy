# v0.0.16

`gocopy compile` accepts all twelve inplace augmented assignment operators,
not just `+=`. v0.0.15 introduced the shape; this rung adds the remaining
eleven: `-=`, `*=`, `//=`, `%=`, `**=`, `&=`, `|=`, `^=`, `>>=`, `<<=`, `/=`.

The bytecode shape is identical to v0.0.15: LOAD_NAME, LOAD augVal,
BINARY_OP oparg, 5 inline-cache words, STORE_NAME. Only the BINARY_OP
oparg differs per operator (NB_INPLACE_* enum). 3-char operators (`//=`,
`**=`, `>>=`, `<<=`) shift augValStart one column right compared to 2-char
operators.

## Added

- `NbInplaceAnd`, `NbInplaceFloorDivide`, `NbInplaceLshift`,
  `NbInplaceMultiply`, `NbInplaceRemainder`, `NbInplaceOr`,
  `NbInplacePower`, `NbInplaceRshift`, `NbInplaceSubtract`,
  `NbInplaceTrueDivide`, `NbInplaceXor` constants in `bytecode/opcode.go`.
- `oparg byte` parameter added to `bytecode.AugAssignBytecode`.
- `tryParseAugAssign` in `compiler/classify.go` extended to recognise all
  twelve operators (3-char checked before 2-char to avoid ambiguity).
- `augOparg byte` field on `rawStmt` and `classification`.
- `cls.augOparg` forwarded to `AugAssignBytecode` in `compiler/compiler.go`.
- Eleven new fixtures: `078`-`088`.
- Eleven new sub-cases in `TestAugAssign`.

## Deferred

- Augmented assignment with non-integer RHS (`x += 1.0`, `x += y`).
- Annotated assignment (`x: int = 1`)  -  deferred to v0.1.2 (AST path).
- Wiring gopapy/v2 as the parser.
