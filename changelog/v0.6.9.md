# v0.6.9 — codegen module-level: basic assignment

Date: 2026-04-29
Spec: notes/Spec/1500/1518_gocopy_v069_codegen_assign.md
CPython parity: Python/codegen.c (3.14.0): module-level
`compiler_mod` → `compiler_visit_stmt` for `Assign` nodes whose
RHS is a non-folded simple constant.

The v0.6.9 release teaches the codegen package the next module-
level shape after the v0.6.8 docstring: `<name> = <const>`
followed by zero or more no-op statements, where `<const>` is a
non-folded simple literal (None, True, False, int, float, complex,
str, bytes, Ellipsis) and `<name>` is 1..15 ASCII chars at column
0. The classifier still owns the constant-folded sub-shapes
(negative literals, `1 + 2`-style folds), the multi-target
assignment shapes (modMultiAssign, modChainedAssign), and every
other module-level shape.

The 246/246 oracle stays byte-identical. The parallel-output gate
(`TestCodegenParity`) automatically lights up every fixture
matching the new subset.

## Note on roadmap deviation

Spec 1508 originally allocated v0.6.9 to the `optimize_load_fast`
+ superinstructions + fold pass. With codegen still owning only
empty/no-ops/docstring after v0.6.8, the optimizer would have
nothing to optimize — no `LOAD_FAST` to borrow, no superinstruction
candidates, no constants to fold. The optimizer pair (`optimize_cfg`
+ load-fast/super/fold) slides into a later sub-release once
codegen owns shapes whose CFGs are non-trivial. Spec slots from
1519 onward retain their roadmap allocations.

## Added

- `compiler/codegen/visit_assign.go` — `assignModule` type plus
  `classifyAssignModule` and `buildAssignModule`. The builder emits
  `RESUME 0; LOAD_CONST 0` (or `LOAD_SMALL_INT <val>` for int64 in
  0..255); `STORE_NAME 0`; `(N-1)x NOP`; `LOAD_CONST <noneIdx>`;
  `RETURN_VALUE`. The trailing `LOAD_CONST` + `RETURN_VALUE` pair
  shares the *name's* Loc when there is no tail (so the encoder
  merges `STORE_NAME` + the trailing pair into one 3-unit SHORT0
  entry); when there is a tail, the trailing pair shares the last
  tail stmt's Loc. The bytes byte-equal `bytecode.AssignBytecode` /
  `bytecode.AssignSmallIntBytecode` + `bytecode.AssignLineTable`.

- `classifyTailNoOp` factored out of the no-op / docstring /
  assign classifiers — one helper, one predicate.

- `constantToValue` mirrors the classifier helper of the same
  name: turns an `*ast.Constant` into its Go-typed value for the
  kinds codegen owns.

## Changed

- `compiler/codegen/codegen.go` — `Build` recognizes the
  modAssign shape after the docstring shape and ahead of the
  ErrUnsupported fallback.

- `compiler/codegen_parity_test.go` — `compileViaClassifier` picks
  up the modAssign classifier path so the parity gate validates
  every newly-codegen-owned fixture against the classifier
  reference.

## Tests

- 246/246 fixtures byte-identical (`tests/run.sh`).
- `compiler/codegen/codegen_test.go` covers `x = 1` (LOAD_SMALL_INT),
  `x = 1000` (LOAD_CONST general path), `x = None` (consts
  collapse), `name = "hi"` followed by a two-stmt no-op tail, and
  the negative-literal rejection case.
- `compiler/codegen_parity_test.go` automatically validates every
  modAssign fixture matching the supported subset.

## Out of scope

- modAssign with `negLiteral` (`x = -1`) — needs the constant-fold
  pass.
- modAssign with `foldedBinOp` (`x = 1 + 2`) — needs the
  constant-fold pass.
- modMultiAssign, modChainedAssign, modAugAssign — later
  sub-releases.
- modBinOpAssign, modUnaryAssign, modCmpAssign, modBoolOp — later.
- Function definitions, classes, comprehensions — later.
- Optimizer passes — later.
