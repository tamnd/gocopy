# v0.7.10.7 ‚Äî extended args (PosOnly / KwOnly / *args / **kwargs) in funcbody

Date: 2026-04-30
Spec: notes/Spec/1500/1565_gocopy_v07107_extended_args.md
Roadmap: notes/Spec/1500/1558_gocopy_v0710x_release_roadmap.md

v0.7.10.7 opens `visitFuncBodyDef` to function signatures that use
positional-only, keyword-only, `*args`, and `**kwargs` parameters.
The visitor still gates out defaults and parameter annotations
(those land at v0.7.10.8 / v0.7.10.10). Body emit shape is
unchanged ‚Äî the work is purely in the function unit's metadata
(`co_argcount`, `co_posonlyargcount`, `co_kwonlyargcount`,
`co_flags`, `co_varnames`, `co_localspluskinds`).

Bringing fixture 020 (`def f(a, b, /, c, d, *args, e, f, **kwargs)`)
online surfaced two latent symtable bugs that no shipped fixture
exercised before:

  - `bindArgs` appended params in the AST source order
    (`PosOnly + Args + Vararg + KwOnly + Kwarg`). CPython's
    `co_varnames` puts KwOnly BEFORE Vararg/Kwarg. Reordered to
    match.
  - `LocalsPlusKinds` returned `LocalsKindArg` (FastLocal | FastArg
    = 0x26) for every param. CPython differentiates by arg kind:
    posonly = 0x22, regular = 0x26, kwonly = 0x24,
    `*args` = 0x2a, `**kwargs` = 0x2c. The fix derives the
    arg-kind bits from each param's slot position in `Params`,
    using the `PosOnlyCount` / `ArgCount` / `KwOnlyCount` /
    `HasVararg` / `HasKwarg` already on Scope.

## Added

- `tests/fixtures/funcbody/016_args_vararg.py` ‚Äî
  `def f(*args): return args`. Pins `CO_VARARGS`, `ArgCount=0`,
  single-slot LocalsPlusNames.
- `tests/fixtures/funcbody/017_args_kwarg.py` ‚Äî
  `def f(**kwargs): return kwargs`. Pins `CO_VARKEYWORDS`.
- `tests/fixtures/funcbody/018_args_posonly.py` ‚Äî
  `def f(a, /, b): return a`. Pins `PosOnlyArgCount=1`,
  `ArgCount=2`.
- `tests/fixtures/funcbody/019_args_kwonly.py` ‚Äî
  `def f(a, *, b): return b`. Pins `KwOnlyArgCount=1`,
  `ArgCount=1`.
- `tests/fixtures/funcbody/020_args_full.py` ‚Äî
  `def f(a, b, /, c, d, *args, e, f, **kwargs): return args`. Pins
  the full extended layout. Exercises both the bindArgs reorder and
  the LocalsPlusKinds per-position kind bits.

## Changed

- `compiler/codegen/visit_funcbody_stmt.go` ‚Äî gates loosened to
  accept PosOnly / KwOnly / Vararg / Kwarg. Defaults and non-nil
  KwOnlyDef still reject. Validation extended over all named-param
  slots (name length 1..15, col ‚â§ 255, no annotation). Slot-count
  cap remains 15.  `assemble.Options` populated:
  `ArgCount = PosOnly + Args`, `PosOnlyArgCount`, `KwOnlyArgCount`,
  flags include `CO_VARARGS` / `CO_VARKEYWORDS` when present.
- `compiler/symtable/build.go::bindArgs` ‚Äî append order corrected
  to `PosOnly + Args + KwOnly + Vararg + Kwarg`.
- `compiler/symtable/scope.go::LocalsPlusKinds` ‚Äî per-position
  arg-kind derivation. Backwards-compatible: every pre-v0.7.10.7
  fixture used regular positional args only, which still map to
  `LocalsKindArg` (0x26).
- `CHANGELOG.md` ‚Äî v0.7.10.7 entry added.
