# v0.1.13 - closures (2026-04-27)

Adds support for a simple closure at module level: an outer function with
one argument defines an inner function with no arguments that returns the
outer argument as a free variable, then the outer function returns the inner
function.

## Supported pattern

```python
def f(x):
    def g():
        return x
    return g
```

This is the first version that uses cell variables, free variables, and the
MAKE_CELL/COPY_FREE_VARS synthetic instructions that CPython emits before
RESUME for functions involved in closures.

CPython's lowering for the outer function:
- `MAKE_CELL 0` promotes x from a plain argument slot to a cell object (this
  instruction runs before RESUME and has no source location, so it gets a
  NO_INFO linetable entry).
- The inner function is made with `LOAD_FAST_BORROW 0` (load x cell),
  `BUILD_TUPLE 1` (closure tuple), `LOAD_CONST 0` (g code object),
  `MAKE_FUNCTION 0`, `SET_FUNCTION_ATTRIBUTE 8` (set `__closure__`), and
  `STORE_FAST 1` (bind g).
- `LOAD_FAST_BORROW 1` + `RETURN_VALUE` returns g.

CPython's lowering for the inner function:
- `COPY_FREE_VARS 1` copies the cell from the closure into the frame (also
  a synthetic instruction, NO_INFO entry).
- `LOAD_DEREF 0` loads the free variable x through the cell.
- `RETURN_VALUE` returns it.

The outer function's co_consts is `(g_code,)` only - no None, because all
code paths return explicitly. The inner function's co_consts is `(None,)`.

localspluskinds encoding for closure variables:
- outer f arg x: 0x66 (CO_FAST_ARG | CO_FAST_CELL - arg promoted to cell)
- outer f local g: 0x20 (CO_FAST_LOCAL)
- inner g free x: 0x80 (CO_FAST_FREE)

The inner function carries co_flags = 0x13 (CO_OPTIMIZED | CO_NEWLOCALS |
CO_NESTED). The outer function carries co_flags = 0x3 (CO_OPTIMIZED |
CO_NEWLOCALS), same as before since it is defined at module level.

## New code

- `bytecode/closure.go`: `ClosureOuterBytecode`, `ClosureInnerBytecode`,
  `ClosureOuterLineTable`, `ClosureInnerLineTable`.
- `bytecode/linetable.go`: `codeNoInfo = 15` constant for NO_INFO entries
  (synthetic instructions with no source location).
- `compiler/classify.go`: `modClosureDef`, `stmtClosureDef`, `closureDef`.
- `compiler/classify_ast.go`: `extractClosure`; `extractFuncDef` now
  handles a 2-statement body (inner def + return inner).
- `compiler/compiler.go`: `compileClosure`.
- 1 new fixture: 137.

## Out of scope for this version

- Inner functions that take arguments.
- Closures that capture more than one free variable.
- Nested closures (closure inside a closure).
- Outer functions that return something other than the inner function.
- Closures that modify the captured variable (`nonlocal x`).
