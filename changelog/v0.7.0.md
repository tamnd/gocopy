# v0.7.0 — visitor pipeline scaffolding (driver pivot)

Date: 2026-04-29
Spec: notes/Spec/1500/1543_gocopy_v070_driver_pivot.md
Architecture: notes/Spec/1500/1541_gocopy_v07_architecture.md
Roadmap: notes/Spec/1500/1542_gocopy_v07x_roadmap.md
CPython parity: Python/compile.c (3.14.0): top-level driver,
`compiler_unit`, `_PyCompile_CodeGen`. Python/codegen.c: `compiler`
struct, scope-stack discipline.

The v0.7.0 release opens the v0.7.x line and starts the
architectural rebuild specified in 1541. v0.6.x shipped 21
sub-releases of "codegen owns one classifier shape per release"
(modEmpty..modClosureDef) and ratcheted the oracle from 246/246
to 246/246. The infrastructure those releases were supposed to
build on top of — `compiler/symtable`, `compiler/ir`,
`compiler/flowgraph`, `compiler/assemble` — was landed in
v0.6.1..v0.6.5 and never consumed by codegen. The codegen package
itself became a relocated shape classifier (23 visit_*.go files
of ad-hoc pattern matching), not the visitor-driven pipeline 1506
specified.

v0.7.x is the rebuild. v0.7.0 is plumbing only: the visitor
pipeline is wired in but every AST node returns
ErrNotImplemented. The classifier path stays authoritative. v0.7.1
starts populating visit_<node> arms.

## Added

- `compiler/codegen/generate.go` — `codegen.Generate(*ast.Module,
  *symtable.Scope, GenerateOptions) (*ir.InstrSeq, error)`,
  the new entrypoint for the v0.7.x visitor pipeline. Returns
  `ErrNotImplemented` for every input at v0.7.0. Mirrors CPython's
  `_PyCompile_CodeGen`.
- `compiler/codegen/generate.go` — `compileUnit` struct, the
  per-frame state every code object lowers to. Mirrors CPython's
  `compiler_unit`.
- `compiler/codegen/generate.go` — `ErrNotImplemented`, distinct
  from the legacy `ErrUnsupported` so callers can tell the two
  transition paths apart.
- `compiler/codegen/generate.go` — `GenerateOptions` (Filename,
  Name, QualName, FirstLineNo). Kept separate from the legacy
  `Options` struct so the legacy path can be deleted at v0.7.26
  without a signature break.
- `compiler/optimize/optimize.go` — new package; `Run(seq) seq`
  identity at v0.7.0. The seven optimizer passes (optimize_cfg,
  optimize_load_fast, insert_superinstructions, fold_constants,
  resolve_unconditional_jumps,
  add_checks_for_loads_of_uninitialized_variables,
  trim_unused_consts) land one per release across v0.7.13..v0.7.19.
- `compiler/visitor.go` — `runVisitorShadow` helper that drives
  Generate → optimize.Run → assemble.Assemble. Returns nil on any
  error including ErrNotImplemented.
- `compiler/visitor_parity_test.go` — `TestVisitorParity`. Walks
  every fixture, runs the visitor pipeline, reports counts:
  byte-identical / not-implemented / mismatched / errored / skipped.
  At v0.7.0: `byte-identical=0 not-implemented=246 mismatched=0`.
- `compiler/cpython_compileall_test.go` — `TestCompileallLib`.
  Opt-in via `GOCOPY_CPYTHON_LIB`. Walks a CPython 3.14 Lib/
  directory and reports the same counts at stdlib scale. Skipped
  when the env var is unset.
- `compiler/codegen/generate_test.go` — unit tests for the new
  entrypoint and the compileUnit constructor.
- `compiler/optimize/optimize_test.go` — identity tests for `Run`.

## Changed

- `compiler/compiler.go` — `Compile` calls `runVisitorShadow` after
  `symtable.Build` succeeds. The shadow's CodeObject is discarded;
  the legacy classifier path stays authoritative. At v0.7.0 the
  shadow always returns nil.

## Tests

- 246/246 fixtures byte-identical (`bash tests/run.sh`).
- `TestCodegenParity` continues to pass (unchanged behavior).
- `TestVisitorParity` passes; logs the v0.7.0 steady state.
- `TestCompileallLib` skips when `GOCOPY_CPYTHON_LIB` is unset.
- `go test ./...` clean. `go vet ./...` clean.

## Out of scope

- Any visit_<node> implementation (v0.7.1+).
- Any optimizer pass (v0.7.13+).
- Promotion of any classifier shape (the shadow is diagnostic only).
- Rename / deletion of `codegen.Build`, `codegen.Options`,
  `codegen.ErrUnsupported`, or any of the v0.6 visit_*.go shape
  files. Those stay in place for the entire v0.7.0..v0.7.25
  transition and are deleted at v0.7.26 (classifier funeral).
- gopapy parser changes.
