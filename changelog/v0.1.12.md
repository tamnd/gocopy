# v0.1.12 - simple function definitions (2026-04-27)

Adds support for defining a function with a single argument at module level,
where the body is a single `return arg` statement. This is the first version
that produces nested code objects.

## Supported pattern

```python
def f(x):
    return x
```

The module code object holds the function's code object in its `co_consts`
tuple. This required fixing a marshal bug: nested code objects must emit
`TYPE_CODE` (0x63) without `FLAG_REF`, while only the top-level module code
gets `TYPE_CODE | FLAG_REF` (0xe3).

The function code object has `co_flags = 0x3` (CO_OPTIMIZED | CO_NEWLOCALS).
Each argument entry in `co_localspluskinds` is 0x26. The module linetable
uses a LONG entry with a non-zero `end_line_delta` to cover the multi-line
function definition.

## Opcodes added

| Opcode | Number | Cache words |
|--------|--------|-------------|
| MAKE_FUNCTION | 23 | 0 |
| LOAD_FAST_BORROW | 86 | 0 |

## New code

- `bytecode/funcdef.go`: `FuncDefModuleBytecode`, `FuncDefModuleLineTable`,
  `FuncReturnArgBytecode`, `FuncReturnArgLineTable`.
- `marshal/writer.go`: nested `*CodeObject` in co_consts emits `TYPE_CODE`
  (0x63, no FLAG_REF). Previously all code objects got FLAG_REF, which broke
  the byte-identical check for any module with a nested function.
- `compiler/classify.go`: `modFuncDef`, `stmtFuncDef`, `funcDefClassify`.
- `compiler/classify_ast.go`: `extractFuncDef`; handles `*parser2.FunctionDef`
  with no decorators, no return annotation, a single argument, and a single
  `return arg` body where the returned name matches the argument name.
- `compiler/compiler.go`: `compileFuncDef`.
- 1 new fixture: 136.

## Out of scope for this version

- Multiple arguments.
- Default argument values.
- Decorators and return annotations.
- Body with more than one statement.
- Nested functions and closures (coming in v0.1.13).
