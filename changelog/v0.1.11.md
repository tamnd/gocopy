# v0.1.11 - for loops (2026-04-27)

Adds support for a simple `for loopVar in iter: bodyVar = val` loop at module
level. Both the loop variable and the iterable must be names. The body must be
a single `name = small_int` assignment (value 0-255). No `break`, `continue`,
or `else` clause.

## Supported pattern

```python
for i in items:
    x = 1
```

CPython 3.14 uses a four-opcode for-loop sequence: `GET_ITER` to produce the
iterator, `FOR_ITER` to advance it and jump past the loop on exhaustion,
`STORE_NAME` for the loop variable, the body, and `END_FOR` + `POP_ITER` at
the bottom. `END_FOR` and `POP_ITER` are new in 3.14; earlier versions used
a single `JUMP_BACKWARD` back to `FOR_ITER`.

## Opcodes added

| Opcode | Number | Cache words |
|--------|--------|-------------|
| GET_ITER | 16 | 0 |
| FOR_ITER | 70 | 1 |
| END_FOR | 9 | 0 |
| POP_ITER | 30 | 0 |

## New code

- `bytecode/forloop.go`: `ForAssignBytecode`, `ForAssignLineTable`.
- `compiler/classify.go`: `modFor`, `stmtFor`, `forAssign`.
- `compiler/classify_ast.go`: handles `*parser2.For` with a single-assign body,
  a Name target, and a Name iter.
- `compiler/compiler.go`: `compileFor`.
- 1 new fixture: 135.

## Out of scope for this version

- Multi-statement loop bodies.
- Tuple unpacking in the loop target (`for a, b in items`).
- `break`, `continue`, and `else`.
- Iterable that is not a plain name.
