# v0.6.20 — codegen module-level: call assign

Date: 2026-04-29
Spec: notes/Spec/1500/1530_gocopy_v0620_codegen_call_assign.md
Roadmap: notes/Spec/1500/1523_gocopy_v06x_part2_roadmap.md (band A)
CPython parity: Python/codegen.c (3.14.0): module-level
`compiler_call`, driven by a top-level `Assign` whose value
is a `Call`.

The v0.6.20 release is the seventh sub-release in band A of
the v0.6.x part 2 roadmap. It teaches the codegen package
the modCallAssign shape:

- `<target> = <func>(<arg0>, ..., <argN-1>)` where target,
  func and every positional arg are Names of 1..15 ASCII
  chars on a single source line. N >= 0 (no-args is OK).
  Keyword arguments and unpacking are out of scope —
  classifier rejects them too.

This is the first codegen path to emit `PUSH_NULL` (the
non-method calling convention) and `CALL` with 3 inline-
cache words. The line table groups LOAD_NAME func +
PUSH_NULL into one 2-unit entry, then one entry per arg
LOAD_NAME, then a 4-unit CALL+caches entry, then the 3-unit
target run.

co.StackSize = 2 + N (LOAD_NAME func + PUSH_NULL = 2 slots,
each arg adds 1).

The 246/246 oracle stays byte-identical. The parallel-output
gate (`TestCodegenParity`) automatically lights up fixtures
128 (`x = f()`), 129 (`x = f(a)`), and 130 (`x = f(a, b)`).

## Added

- `compiler/codegen/visit_call_assign.go` —
  `callAssignModule` type plus `classifyCallAssignModule` +
  `buildCallAssignModule`. Single-statement only so the
  parity gate stays byte-equal with the classifier's
  `compileCallAssign`.

## Changed

- `compiler/codegen/codegen.go` — `Build` recognizes
  modCallAssign after modAttrStore, ahead of the
  ErrUnsupported fallback.

- `compiler/codegen_parity_test.go` — `compileViaClassifier`
  picks up the modCallAssign classifier path so the parity
  gate validates the new codegen-owned fixtures against the
  classifier reference.

## Out of scope

- Method calls (`x = obj.f(args)`) — classifier rejects too.
- Keyword arguments (`x = f(k=v)`) and unpacking
  (`*args` / `**kwargs`).
- Multi-line calls — classifier rejects too.
- modGenExpr — v0.6.21 (closes band A).
- Function definitions, classes, comprehensions, try/except
  — bands B and C of the part 2 roadmap.
- Optimizer passes, classifier funeral, perf — closing
  band.
